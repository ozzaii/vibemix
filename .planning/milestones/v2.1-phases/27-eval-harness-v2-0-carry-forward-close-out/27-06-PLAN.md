<!-- PyInstaller target-arch flag (verified 2026-05 PyInstaller stable) -->
- `pyinstaller --target-arch arm64 ...` produces an arm64-only bundle (binary `lipo -archs` returns "arm64")
- `pyinstaller --target-arch x86_64 ...` produces an x86_64-only bundle
- `--target-arch universal2` IS technically possible BUT requires universal2 Python interpreter + universal2 wheels for EVERY binary extension (numpy, scipy, sounddevice, pyobjc, av, livekit, mido) — most not available for Python 3.12. Per RESEARCH §Critical Correction "RECOMMENDED PATH": target-triple split is lower-risk.

<!-- Tauri externalBin target-triple convention (verified v2.tauri.app/develop/sidecar) -->
```json5
// tauri.conf.json5
{
  "bundle": {
    "externalBin": [
      "../resources/vibemix-sidecar"  // Tauri auto-appends the target triple at install time
                                       // → vibemix-sidecar-aarch64-apple-darwin on Apple Silicon
                                       // → vibemix-sidecar-x86_64-apple-darwin on Intel Mac
                                       // → vibemix-sidecar-x86_64-pc-windows-msvc.exe on Windows
    ]
  }
}
```

Target triples (per Tauri docs):
- macOS Apple Silicon: `aarch64-apple-darwin`
- macOS Intel: `x86_64-apple-darwin`
- Windows MSVC: `x86_64-pc-windows-msvc.exe`  (Windows already works fine; not REC-09 scope)
- Linux: not supported per CLAUDE.md "Platforms: macOS + Windows in v1. Linux explicitly excluded"

GitHub Actions runner availability (per RESEARCH §Environment Availability):
- `macos-14`: Apple Silicon (arm64) — DEFAULT for arm64 builds
- `macos-13`: Intel — for x86_64 builds (still available May 2026 per GitHub docs)
- `macos-14-large`: Intel under Apple Silicon hardware via Rosetta (alternative if macos-13 is deprecated)
