<objective>
Close the v2.0 LATENCY-14 WASAPI device-change carry-forward (Pitfall P70). Currently, when a user plugs in headphones or switches audio devices mid-session on Windows, vibemix crashes — the WASAPI loopback stream points at a stale device handle. This plan subscribes to the Windows COM `IMMNotificationClient::OnDefaultDeviceChanged` event via comtypes and soft-restarts the stream WITHOUT blocking the COM callback (Microsoft's hard requirement per https://learn.microsoft.com/en-us/windows/win32/api/mmdeviceapi/nn-mmdeviceapi-immnotificationclient).

Per RESEARCH §Code Examples "LATENCY-14 — IMMNotificationClient stub": the pattern is callback-signals-event + worker-thread-restarts-stream. ANY synchronous work in OnDefaultDeviceChanged → Windows kills the audio service.

Per CONTEXT decision LATENCY-14: macOS gets a graceful stub — no comtypes import on macOS, no Windows-only crash.

Per Assumption A6: verify comtypes is already a transitive of pyaudiowpatch BEFORE adding explicit; if absent, add `'comtypes>=1.4; sys_platform == "win32"'` to dev deps.

Output:
- Extended `src/vibemix/platform/_audio_windows.py` with `_DeviceChangeListener` COM class + `WindowsLoopbackAudio` worker thread pattern
- `pyproject.toml` adds comtypes IF not transitive (verify step in task)
- `tests/runtime_closeouts/test_wasapi_default_device_change.py` mocks COM + asserts callback < 1ms + soft-restart triggered

<!-- Microsoft IMMNotificationClient interface (verified via WebSearch + learn.microsoft.com) -->

The interface has 5 callback methods; ALL MUST be non-blocking + return S_OK (0):
- `OnDefaultDeviceChanged(flow, role, default_device_id)` — fires when default playback/capture device changes
- `OnDeviceAdded(device_id)` — new device plugged in
- `OnDeviceRemoved(device_id)` — device removed
- `OnDeviceStateChanged(device_id, new_state)` — active/disabled/unplugged transition
- `OnPropertyValueChanged(device_id, key)` — device property changed (often noisy; usually ignored)

comtypes COMObject pattern (verified via comtypes docs):
```python
from comtypes import COMObject, GUID
from comtypes.client import CreateObject
from ctypes import HRESULT, c_int, c_wchar_p

# IMMNotificationClient IID per Windows SDK:
_IID_IMMNotificationClient = "{7991EEC9-7E89-4D85-8390-6C703CEC60C0}"

class IMMNotificationClient(IUnknown):  # auto-generated via comtypes gen
    _iid_ = GUID(_IID_IMMNotificationClient)

class _DeviceChangeListener(COMObject):
    _com_interfaces_ = [IMMNotificationClient]

    def __init__(self, restart_event: threading.Event):
        super().__init__()
        self._restart_event = restart_event

    def OnDefaultDeviceChanged(self, flow, role, default_device_id):
        # NON-BLOCKING per Microsoft docs — signal only, work elsewhere
        self._restart_event.set()
        return 0  # S_OK

    def OnDeviceAdded(self, device_id): return 0
    def OnDeviceRemoved(self, device_id): return 0
    def OnDeviceStateChanged(self, device_id, new_state): return 0
    def OnPropertyValueChanged(self, device_id, key): return 0
```

Registration pattern (subscribe the listener to the device enumerator):
```python
# Use IMMDeviceEnumerator::RegisterEndpointNotificationCallback
enumerator = CreateObject(_CLSID_MMDeviceEnumerator, interface=IMMDeviceEnumerator)
self._listener = _DeviceChangeListener(self._restart_event)
enumerator.RegisterEndpointNotificationCallback(self._listener)
# ... on shutdown: enumerator.UnregisterEndpointNotificationCallback(self._listener)
```

Worker thread pattern (the actual stream restart):
```python
def _restart_worker(self):
    while not self._stop_event.is_set():
        if self._restart_event.wait(timeout=1.0):
            self._restart_event.clear()
            try:
                self._restart_stream_safely()
            except Exception as e:
                print(f"[wasapi restart err] {e}", file=sys.stderr)
                # do not propagate — keep worker alive

def _restart_stream_safely(self):
    # Close current stream gracefully
    if self._stream is not None:
        self._stream.close()
    # Re-resolve default device + open new stream
    # ... existing assert_wasapi_loopback_rate logic ...
    self._stream = self._open_loopback_stream()
```
