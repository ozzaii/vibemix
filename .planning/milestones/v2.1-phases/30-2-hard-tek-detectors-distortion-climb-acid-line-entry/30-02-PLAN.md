## DSP design

### `dominant_freq_in_band(samples, sr, low_hz=200, high_hz=800) -> float`
- Hanning-windowed rfft.
- Restrict to `[low_hz, high_hz]`.
- Return frequency of bin with max magnitude.
- Out-of-band guard: if in-band energy < 1% of total → return 0.0 (anti-hallucination).
- Silence → 0.0.

### `band_resonance_q(samples, sr, low_hz=200, high_hz=800) -> float`
- Hanning-windowed rfft.
- In-band magnitudes only.
- Return `peak_magnitude / (mean_magnitude + 1e-12)`.
- Silence → 0.0.
- High-Q resonance → > 8.0; flat noise → ≈ 1.0.

## Detector design

### State
- `last_event_at: float = 0.0`
- `freq_history: deque[(t, freq)]` — maxlen 16 (covers ~1.5s at 100ms tick or more at any sane cadence).
- `q_history: deque[(t, q)]` — same length.
- We don't need a separate sweep-direction state — slope sign tells direction.

### `detect(state, audio_buf, now)`
1. Silence gate: `state.rms < LOW_RMS` → return None.
2. `audio_buf is None` → return None.
3. Cooldown: `now - last_event_at < 8.0` → return None.
4. Snapshot 1.5s window: `audio_buf.snapshot(int(sr * 1.5))`.
5. Compute `freq = dominant_freq_in_band(samples, sr, 200, 800)`.
6. Compute `q = band_resonance_q(samples, sr, 200, 800)`.
7. If `freq <= 0`: return None (silence/out-of-band).
8. Append `(now, freq)` to `freq_history`, `(now, q)` to `q_history`.
9. Trim to entries within `now - 3.0` window (3s sweep evaluation window).
10. Require `len(freq_history) >= 8` (need enough points for slope).
11. Compute sweep slope: linear regression of `log2(freq)` vs `t`. Slope is octaves/sec.
12. Require `abs(slope) > 0.2` (CONTEXT: 0.2 oct/sec).
13. Resonance-rise: require `min(early_q) < 3.0` AND `max(late_q) > 8.0` where early = first half, late = second half of `q_history`.
14. Time-window: require span `last_t - first_t >= 1.5`.
15. ALL conditions met → fire:
    - `formant_hz = median(freq for _, freq in freq_history)` (sweep midpoint).
    - `resonance_q = max(q for _, q in q_history)` (peak Q).
    - `Event("ACID_LINE_ENTRY", state, extra={"formant_hz": round(formant_hz, 1), "resonance_q": round(resonance_q, 2)})`
    - `last_event_at = now`
    - Clear histories.
    - Return Event.

## Synth helpers

To synthesize a TB-303 sweep test fixture, generate per-tick samples:
- For each tick at time `t_i`: `freq_i = 250 * 2**((t_i - t0) * sweep_octaves)`, where `sweep_octaves = 1.0 / 1.5` for 1 oct/sec.
- Magnitude profile: narrow sinusoid + small white-noise floor. As Q rises, reduce noise / increase peak.
- Each fixture snapshot returns the trailing 1.5s window samples reflecting the instantaneous freq.

Easier: drive detector with mocked `dominant_freq_in_band` results via a stub FakeAudioBuf returning pre-baked signals. We synthesize an actual sweep signal that gives `dominant_freq_in_band` the expected value at each tick.

## Risks / mitigations
- **Risk:** Synthesized sweep doesn't pass slope threshold due to FFT bin quantization. **Mitigation:** use 1 oct/sec sweep (well above 0.2 threshold).
- **Risk:** Q proxy too noisy on synthesized signal. **Mitigation:** Use clean sine + white noise mix to control Q deterministically.
