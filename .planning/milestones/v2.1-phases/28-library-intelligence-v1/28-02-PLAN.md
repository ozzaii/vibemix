<objective>
Build the vector persistence layer + storage-agnostic `LibraryStore` facade. Two backends with one shared top-K path — sqlite-vec on Mac (and Win where its extension loads), numpy fallback elsewhere. The critical contract: **identical Python `cosine_topk()` runs on both** so Mac/Win produce bit-identical top-K rank orders (Pitfall P55 mitigation).

Includes Wave 0 sqlite-vec ARM64 Win probe (RESEARCH Assumption A2): the module-level `open_store()` tries `sqlite_vec.load(db)` first; on `OperationalError` / `ImportError` it falls through to numpy with a structured log line. This makes the Win-ARM64 wheel-availability question survivable at runtime — not a hard install blocker.

Purpose: every downstream plan (search, grounding, similarity) calls `store.search(qvec, k)` — they don't care which backend underpinned it.

Output:
- `src/vibemix/library/index_sqlite_vec.py` (Mac/Win-x64 primary)
- `src/vibemix/library/index_numpy.py` (fallback)
- `src/vibemix/library/store.py` (facade)
- 1000-vector parity fixture + 50-query fixture
- Parity tests that prove Mac+Win bit-identity

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Sidecar → user file system (`~/.cache/vibemix/library.db`, `library_vectors.npy`, `library_ids.json`) | All writes local; no remote IO in this plan. |
| Backend selection probe | Catches `sqlite_vec.load` failure to choose backend — risk of accidental Linux-only sidecar bypass. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-02-01 | Tampering | `library.db` blob storage | accept | User-owned `~/.cache/`. SHA256 collision needed to substitute a track's vector — computationally infeasible. |
| T-28-02-02 | DoS | Malformed npy file on disk | mitigate | `np.load(allow_pickle=False)` blocks pickle exec; load errors caught + backend re-initialized to empty (log + continue). |
| T-28-02-03 | Information Disclosure | track_id list in `library_ids.json` | accept | Local file, user-owned; no remote sync. POSIX 0644 acceptable. |
| T-28-02-04 | Tampering | `sqlite_vec.load` probe — host could ship a malicious extension as `sqlite_vec`? | accept | sqlite-vec is pinned via pyproject `==0.1.9` (lockfile-protected). Tauri bundle path is sandboxed — extension loaded from the locked PyPI wheel. |
| T-28-02-05 | Repudiation | Backend choice not logged | mitigate | `open_store()` emits `-> library store: backend={chosen} reason={probe_result}` to stdout; sidecar log captures. |
</threat_model>
