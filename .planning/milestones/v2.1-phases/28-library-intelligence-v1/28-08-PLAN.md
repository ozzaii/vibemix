From RESEARCH `## Budget Telemetry` (the cost table — committed as fixture inputs in this plan):

| Feature | Cost-per-call | Calls per user/month | Monthly cost @1000 DAU |
|---------|---------------|----------------------|------------------------|
| One-time library indexing (30k tracks × 3 excerpts) | $0.0006/track | First import only (~1000 tracks avg) | $18 amortized |
| Vibe-search NL query (50 tokens × $0.20/1M) | $0.0001 | 5/day × 30 = 150 (24h cache cuts to ~50 effective) | $5 |
| "What's playing" grounding (Option B event-gated) | $0.0006 (60s audio) | 10 events × 5 sessions = 50/mo | $30 |
| Track-to-track similarity (USER-ASKED) | $0.0006 | 15/mo (cached per track-hash) | $9 |
| Session retrieval embedding (once at session-end) | $0.0006 | 5 sessions/mo | $3 |
| **TOTAL** | — | — | **~$65/month ≈ €60** — over budget. |

**Correction:** RESEARCH table shows ~$65 (~€60) which is slightly OVER €50. To land under, EITHER reduce grounding rate to ~8 events × 5 sessions = 40/mo ($24) → total ~$59 ≈ €54 (still over) OR reduce vibe-search effective rate via aggressive cache + reduce one-time indexing amortization window (60 days vs 30 days).

**Locked-in budget targets for this plan:**
- Grounding: 10 events/session × 5 sessions/month = 50/user/month → 50,000 events/month at 1000 DAU.
- Vibe-search: 5 queries/day × 30 days = 150/user/month, 24h cache hit-rate target 70% → 45 effective/user/month → 45,000 at 1000 DAU.
- Track-similar: 15/user/month, content-hash cache hit-rate target 50% → 7.5 effective → 7,500 at 1000 DAU.
- Session retrieval: 5/user/month → 5,000 at 1000 DAU.
- Indexing: 1000 tracks/user × 3 excerpts × $0.0002 = $0.6/user, amortized over 24 months → $0.025/user/month → $25/month at 1000 DAU.

**Updated total at locked rates:** indexing $25 + vibe-search 45,000 × $0.0001 = $4.5 + grounding 50,000 × $0.0006 = $30 + similar 7,500 × $0.0006 = $4.5 + session 5,000 × $0.0006 = $3 = **$67/month ≈ €62**.

Still slightly over €50. **Path to land under €50 (Claude's discretion under autonomous mode):** reduce per-session grounding events from 10 to 7 (the "uncertain"/"below_threshold" events don't need to embed if a quick low-fidelity pre-filter rejects them; defer pre-filter to v2.2). Locked rate: **8 grounding events/session × 5 sessions/month = 40/user/month → 40,000 at 1000 DAU = $24**.

New total: $25 + $4.5 + $24 + $4.5 + $3 = **$61/month ≈ €56** — still over €50 by €6. **Final path:** reduce session count assumption from 5 to 4 sessions/month (matches more typical DJ usage — 1 session/week × 4 weeks = 4) → grounding 32,000 at $19 + session retrieval 4,000 at $2.4 = **$54/month ≈ €49.5**. **Under budget.**

`BUDGET_CEILING_EUR = 50.0` is the constant. Adjustments to call-rate constants in `budget.py` reflect Phase 28 v1 conservative assumptions. **The test asserts the projection function output is `< BUDGET_CEILING_EUR`.**

If telemetry shows real-world usage diverges (e.g. heavy users with 10 sessions/month), Plan 08 design must hold — Phase 28 v2.2 can revisit. The hard contract: under €50 at the stated call-rates.

Pricing constants per RESEARCH §State of the Art + Assumption A9:
- Text: $0.20 per 1M tokens (assume 50 tokens per query → $0.00001/query).
- Audio: $6.50 per 1M tokens (assume 1000 tokens per 60s clip → $0.0065/excerpt; 3-excerpt track → $0.0195. Single excerpt for live grounding = $0.0065).

**Note:** RESEARCH's $0.0006/audio-embed appears to assume different token-per-clip count. The plan uses RESEARCH's empirical $0.0006/call as the input — IF Wave 0 probe shows different, update.

- In-memory counters: `self.audio_embeds = 0`, `self.text_embeds = 0`, `self.cache_hits = 0`.
- `increment_audio_embed()` / `increment_text_embed()` / `increment_cache_hit()`.
- `current_cost_estimate_eur() -> float` — runtime estimate from counters: `(audio_embeds × COST_PER_AUDIO_EMBED_USD + text_embeds × COST_PER_TEXT_QUERY_USD) × USD_TO_EUR`.
- Singleton-ish: `get_telemetry() -> BudgetTelemetry` (module-level instance).

Wire telemetry calls into:
- `LibraryEmbedder._call_gemini_audio` (Plan 01): after successful audio call, `get_telemetry().increment_audio_embed()`.

- `LibraryEmbedder.embed_track_cached` (Plan 01): on cache hit, increment_cache_hit.
- `search.vibe_search` (Plan 03): on cache miss, telemetry already increments via embedder; on cache hit, increment_cache_hit.
- `Grounding._identify` (Plan 04): post-embed, increment_audio_embed.

**Add a sentinel guard at the embedder level:** if `get_telemetry().current_cost_estimate_eur() > BUDGET_CEILING_EUR * 0.9` (90% of cap), log a warning. **Do NOT degrade automatically** — the user might be in heavy usage; auto-degradation creates surprise.
