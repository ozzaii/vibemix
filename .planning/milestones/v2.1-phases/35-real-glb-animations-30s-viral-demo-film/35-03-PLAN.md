# Plan 35-03 — README Hero Hash Sync Gate

**REQ-IDs:** ASSETS-07
**Pitfall:** P68

## Goal

`scripts/check_readme_hero_hash.py` + `.github/workflows/readme-hero-sync.yml`. Parses the README hero block comment (`<!-- vibemix:hero-start sha256=... -->`), hashes `docs/assets/demo.mp4`, fails on drift.

`PLACEHOLDER` sentinel: when `sha256=PLACEHOLDER` is present, the script treats the asset as not-yet-shipped (Kaan-action pending) and exits 0. This lets us land the gate before the real asset.

## Tasks

1. Write `scripts/check_readme_hero_hash.py`:
   - Parse README for the hero block comment.
   - Extract `sha256=<value>` from comment.
   - If value == `PLACEHOLDER` → exit 0 with note ("asset pending Kaan-action").
   - Else compute SHA256 of `docs/assets/demo.mp4`, compare. Drift → exit 1.
   - If asset missing AND value != PLACEHOLDER → exit 1.
2. Write `.github/workflows/readme-hero-sync.yml` — runs on PR + push to main + nightly schedule.
3. Write `tests/repo/test_readme_hero_hash_sync.py`:
   - `test_placeholder_sentinel_passes` — current README state.
   - `test_drift_detected` — write tmp README with wrong hash → exit 1.
   - `test_asset_missing_with_real_hash_fails` — tmp README with non-PLACEHOLDER hash but no asset → exit 1.
   - `test_matching_hash_passes` — tmp asset + matching hash → exit 0.
