<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer→open_debrief_window | session_dir from JS — untrusted. Rust validates via validate_under_root before any FS or process operation. |
| WebviewWindow→sidecar via ws | Plan 02 sidecar bound to 127.0.0.1:8766 — same trust model as main sidecar's 8765. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-29-04-01 | Tampering | session_dir contains `..`, absolute path outside recordings root | mitigate | validate_under_root canonicalizes + checks starts_with — Test 3 covers. Defense-in-depth: capability validator regex from Plan 00 already restricts shape. |
| T-29-04-02 | Elevation of Privilege | Renderer spawns multiple debrief sidecars to exhaust resources | mitigate | Focus-existing logic — Test 2 covers. Maximum one debrief sidecar process at a time. |
| T-29-04-03 | Denial of Service | Sidecar crashes silently, window stays blank | mitigate | Crash watcher emits `sidecar-debrief-crashed` event → Plan 05 error banner; Test 2 covers. |
| T-29-04-04 | Tampering | session_dir argument passed to sidecar contains shell metacharacters | mitigate | tauri-plugin-shell `Command::args` does NOT invoke a shell — passes argv directly to execve. No metacharacter expansion. Capability validator constrains shape anyway. |
| T-29-04-05 | Information Disclosure | Window event payload contains sensitive info | accept | `{exit_code, reason}` only — no session data. |
| T-29-04-06 | Spoofing | Another process binds 127.0.0.1:8766 before our sidecar | accept | Plan 02 sidecar exits with `port_in_use` error; Rust detects exit, emits crash event; user retries. Low likelihood (8766 not a common port). |
| T-29-04-07 | Tampering | Race between close handler and crash watcher | mitigate | `guard.take()` is idempotent (Option semantics); two takers each get one Some or None. Test 3 covers. |
</threat_model>
