# Plan 31-04 — Reaction layer + emote_parser

**REQ:** MASCOT-24

## Goal
Parse `[emote:NAME]` tags from Gemini response text; fire matching reaction clip on the priority-80 ReactionLayer with 2.5s timeout + cancel-aware.

## Contract
Python:
- `parse_emote_tags(text: str) -> list[str]` — extracts `[emote:NAME]` patterns.
- Whitelist: `wave / point_left / point_right / fist_pump / nod / headbang / surprised`. Unknown tags silently dropped (anti-slop — caller logs).
- Returns ordered list (first-seen order); duplicates preserved.
- Tags stripped from output text via `strip_emote_tags(text) -> tuple[clean_text, intents]`.

Frontend:
- `ReactionLayer.fire(intent, now_ms)` — schedules priority-80 transition with 2.5s timeout.
- Cancel-aware: `cancel()` flushes via PriorityStack cancel-priority 999 sentinel.
- Same intent fired twice in quick succession → second is queued (FIFO), not dropped.

## Verification
- pytest: 10 cases covering whitelist, unknown tags, multiple tags, malformed brackets, trailing/leading text preservation.
- vitest: ReactionLayer.fire schedules at priority 80 with timeout 2500.
