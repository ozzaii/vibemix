# Plan 31-01 — PriorityStack + CrossfadePolicy

**REQ:** MASCOT-20, MASCOT-21
**Pitfalls:** P62 (single-mixer race), P72 (cancel-priority 999)

## Goal
Implement the priority-stack manager and crossfade policy that arbitrates 4 simultaneous layer transitions with 100ms stagger and cancel-priority 999 flush.

## Files
- NEW: `tauri/ui/src/mascot/priority-stack.ts`
- NEW: `tauri/ui/src/mascot/priority-stack.test.ts`
- NEW: `tauri/ui/src/mascot/crossfade-policy.ts`
- NEW: `tauri/ui/src/mascot/crossfade-policy.test.ts`

## Contract
- `PriorityStack` class — pure (no wall-clock, time injected via `now`).
- 4 named channels: `base` (50), `emotion` (60), `anticipation` (70), `reaction` (80).
- API: `play(layer, clip, opts)`, `cancel(layer, {flush})`, `cancelAll()`, `resolve()`.
- Cancel flag `{flush: true}` = priority 999 sentinel — flushes ALL pending queues except base.
- `CrossfadePolicy.transition(from, to, opts)` → `{stagger_ms, fade_in_ms, fade_out_ms}`.
- 100ms stagger: when multiple layers fire in same frame, highest priority t=0, next t=100ms, etc.
- Defaults: fade_in 200ms, fade_out 150ms.

## Verification
- vitest perf: 100 simultaneous transitions, p99 frame-time < 22ms.
- Cancel-flush: queue all 4 layers, fire cancel, assert all queues empty + base untouched.
- 100ms stagger: 4 simultaneous play() calls, assert delays 0/100/200/300.
