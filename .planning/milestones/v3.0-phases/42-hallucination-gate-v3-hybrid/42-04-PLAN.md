(a) `scripts/release/check_gate.sh`:
- SPDX header + comment block citing Plan 42-04, GATE-06, hybrid regime contract.
- `set -euo pipefail`.
- Configurables (via env, with defaults):
  - `EVAL_RUNS_DIR="${EVAL_RUNS_DIR:-.planning/eval-runs}"`
  - `THRESHOLD_LOCK="${THRESHOLD_LOCK:-eval/THRESHOLD-LOCK.md}"`
  - `EAR_TEST_GATE="${EAR_TEST_GATE:-scripts/release/check_ear_test.sh}"`
  - `MIN_CONSECUTIVE_GREEN=7`
- Logic:
  1. **Parse locked thresholds.** Use python one-liner: `f1_min=$(uv run python -c "from scripts.eval.threshold_lock import parse_threshold_lock_frontmatter; from pathlib import Path; print(parse_threshold_lock_frontmatter(Path('${THRESHOLD_LOCK}'))['thresholds']['f1_min'])")` — and same for substance_min, cited_cosine_min, bypass_max.
  2. **Glob last 7 nightly run dirs.** `ls -1dt ${EVAL_RUNS_DIR}/*/` 2>/dev/null | head -n ${MIN_CONSECUTIVE_GREEN}. If fewer than 7 dirs exist: exit 1 with structured stderr "BLOCKED_BY=nightly: only N consecutive nightly runs (need 7)".
  3. **For each of the 7 dirs**, parse `eval_report.json` via jq:
     - extract aggregate (or per-session min) f1, substance, cited_cosine, bypass
     - assert: f1 >= f1_min AND substance >= substance_min AND cited_cosine >= cited_cosine_min AND bypass <= bypass_max
     - if ANY metric fails, accumulate `failed_runs+=("$dir: $metric=$value (lock=$lock)")`
  4. **Invoke ear-test gate.** `bash "${EAR_TEST_GATE}"` — capture exit code. If nonzero: append `ear_test_failed=true`.
  5. **Verdict.**
     - If `${#failed_runs[@]} -eq 0` AND `ear_test_failed=false`: print `"  PASS check_gate: 7/7 nightly green + ear-test green"`; exit 0.
     - Else: print `"  FAIL check_gate:"` followed by `BLOCKED_BY:` lines naming each failed input; exit 1.
- GitHub Actions annotations on failure (mirror `check_no_hardcoded_model.sh` pattern).
- `chmod +x` the script after writing (`Bash: chmod +x scripts/release/check_gate.sh`).
