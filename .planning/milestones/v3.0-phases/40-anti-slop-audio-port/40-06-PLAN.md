<objective>
Extend `blackhole_probe.py` to emit structured `audio.probe.{detected,missing,cta_fired}` events (per AUDIO-07 + locked decision Q4) so the install-wizard funnel becomes observable. Today the probe returns a plain dict; the wizard polls and decides UI state, but there's no audit trail of which path users took (probe-passed-on-first-try vs probe-passed-after-cta vs gave-up). Adding emit_event hooks creates that trail without coupling the probe to a specific IPC channel.

**Hallucination class closed:** None — this is install-funnel instrumentation, not Gemini grounding.

**Pitfall 5 defense:** RESEARCH §Common Pitfalls 5 documents a real race — on fresh boot, sounddevice can return the device list before CoreAudio finishes enumeration, so a Mac with BlackHole installed reads as `installed=False` on the first probe. The wizard fires the install CTA, the user clicks, and only on the next probe does BlackHole appear. This plan adds a single 1.5s-sleep + retry inside the probe (per RESEARCH recommendation) so the CTA doesn't fire spuriously.

**Sink locked (Q4):** Events emit to local stdout + the existing `events.jsonl` via the `VoiceRecorder.log_event` pattern. No new IPC channel. The wizard Rust shell can already subscribe to `events.jsonl` tail-reading if it wants the events; the Python sidecar emits them. This keeps the probe stateless from the wizard's POV.

Purpose: AUDIO-07 instrumentation + fresh-boot race defense, ready for Kaan's fresh-Mac walk discharge (documented as KAAN-ACTION runbook).

class BlackHoleProbeResult(TypedDict):
    installed: bool
    device_name: str | None

def probe_blackhole() -> BlackHoleProbeResult:
    for entry in _query_devices():
        name = entry.get("name")
        if isinstance(name, str) and "BlackHole" in name:
            return {"installed": True, "device_name": name}
    return {"installed": False, "device_name": None}

Target post-Plan-40-06:
```python
from typing import Any, Callable, TypedDict

EmitEvent = Callable[[str, dict], None]

def probe_blackhole(
    emit_event: EmitEvent | None = None,
    *,
    retry_on_missing: bool = True,
) -> BlackHoleProbeResult:
    """Probe + (optional) structured-event emit + Pitfall 5 fresh-boot-race retry."""
    result = _probe_once()
    if not result["installed"] and retry_on_missing:
        # Pitfall 5 — CoreAudio race on fresh boot. Single retry after 1.5s.
        time.sleep(1.5)
        result = _probe_once()
    if emit_event is not None:
        if result["installed"]:
            emit_event("audio.probe.detected", {"device_name": result["device_name"]})
        else:
            emit_event("audio.probe.missing", {"device_name": None})
    return result


def emit_cta_fired(emit_event: EmitEvent, cta: str = "blackhole_install_link_opened") -> None:
    """Caller invokes this when the wizard dispatches the install URL CTA."""
    emit_event("audio.probe.cta_fired", {"cta": cta, "url": BLACKHOLE_INSTALL_URL})
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| sounddevice query → probe | Untrusted device-list from CoreAudio; trusted by macOS but app-published audio devices can have arbitrary names |
| emit_event callback → events.jsonl | Wizard or test code provides the callback; probe assumes it may raise (try/except wrap) |
| probe outcome → wizard CTA decision | Pitfall 5 fresh-boot race could fire CTA on a Mac that does have BlackHole — mitigated by retry |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-06-01 | DoS | emit_event raising | mitigate | `try/except Exception: pass` wrapper around every emit call. Test `test_emit_failure_does_not_crash_probe` pins this. |
| T-40-06-02 | Spoofing | Device with "BlackHole" in name but not the real driver | accept | macOS device-name namespace is uncontrolled. The probe's substring match is conservative but spoofable. Accepted because the spoof gains the attacker nothing (they bypass the install-CTA only). |
| T-40-06-03 | Timing | Pitfall 5 fresh-boot race | mitigate | Single 1.5s retry per RESEARCH recommendation. Test `test_pitfall_5_retry_succeeds_on_second_try` pins the success path; `test_pitfall_5_retry_still_missing` pins the actually-not-installed path. |
| T-40-06-04 | Repudiation | No audit trail of which probe path users took | mitigate | `audio.probe.{detected,missing,cta_fired}` events provide full install-funnel trail. Kaan's fresh-Mac walk discharge checklist requires events.jsonl artifact. |
</threat_model>
