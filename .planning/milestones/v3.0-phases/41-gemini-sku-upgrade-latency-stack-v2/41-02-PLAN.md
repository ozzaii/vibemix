<debounce_design>
<!-- Pitfall 2: callback storm during heavy mixing. Per CONTEXT.md: 5-10s debounce. -->

Debounce pattern (asyncio.call_later cancel-and-reschedule):

The EvidenceRegistry holds a single `_pending_refresh_handle: asyncio.TimerHandle | None`. On mutate():
1. If `_on_mutation` is None, return (cache wiring optional).
2. Cancel `_pending_refresh_handle` if set.
3. Reschedule `loop.call_later(5.0, lambda: asyncio.create_task(_on_mutation()))`.
4. Track `_last_refresh_at: float` — if (now - last_refresh) < 30s, push the scheduled refresh to (30 - elapsed) instead of 5.0 (min-interval guard).

This means: in a 60-event burst over 1 minute, the refresh fires at most 2 times (once after the burst settles 5s post last event; once 30s later if mutations resume).
</debounce_design>
