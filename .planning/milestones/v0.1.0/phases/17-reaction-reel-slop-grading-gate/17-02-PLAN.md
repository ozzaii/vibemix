<objective>
Build the blind-grading CLI that 4 raters will run during the actual gate evaluation. The script reads a Phase 15 recordings dir, extracts the AI's reactions from events.jsonl, anonymizes them via SHA8, shuffles them deterministically per rater, plays each voice clip, prompts the rater for the 10-field grade per the rubric, and writes a resumable JSONL.

Purpose: Without this script, raters either grade in chronological order (biased by recency, mode trajectory, set energy curve) or grade with metadata visible (genre/mode/skill leaks into "would a real friend say this" judgment). The script enforces blind randomized grading.

Output: One Python CLI (`scripts/reaction_reel/grade.py`) + comprehensive unit tests covering the anonymization, shuffle, resume, and schema invariants. No frontend, no IPC, no Tauri changes.
</objective>

Per-rater deterministic shuffle seed:

    random.Random(seed_int).shuffle(reactions)

Grade record written to <rater>.jsonl (one JSON object per line):
    {
      "reaction_id": <string, sha8>,
      "score": <int 1-5>,
      "rater": <string in {kaan, francesco, dj1, dj2}>,
      "grounded": <bool>,
      "timely": <bool>,
      "unique": <bool>,
      "personality_fit": <bool>,
      "slop_flag": <string in {none, late, generic, hallucination, repetition, cringe}>,
      "comment": <string, may be empty>,
      "would_clip": <bool>,
      "graded_at_iso": <string, ISO-8601 wall clock — operational field, not in rubric>
    }

grades.key.json schema:
    {
      "<sha8>": {"t": <float>, "text": <string>, "latency_s": <float>}
    }
    # The mapping is written ONCE on first invocation, BEFORE any grading prompts.
    # If it already exists, grade.py reads it and verifies the reaction set hasn't changed
    # (events.jsonl extension after grading started is rejected with a clear error).

Resume semantics:
    - On every launch, grade.py reads <rater>.jsonl if it exists.
    - Already-graded reaction_ids are skipped silently.
    - Shuffle order is regenerated from the same seed — already-graded entries are still
      in the shuffled sequence at the same positions; the script advances past them.
    - This guarantees: (a) deterministic order, (b) no double-grading, (c) resume-anywhere.

    # The script does NOT auto-detect slop — that's the rater's call.

    - Ctrl-C exits cleanly — partial line never written (only completed records hit disk).
    - Special "skip" command at the score prompt: typing "skip" defers the reaction
      (does NOT write a record; resume-next-launch will encounter it again).

      - `extract_reactions(session_dir: Path) -> list[dict]`: opens events.jsonl, line-by-line json.loads, filters kind=="ai_text", returns list sorted by `t`.
      - `compute_reaction_id(session_dir_name: str, reaction: dict) -> str`: SHA1 of f"{session_dir_name}|{reaction['t']}|{reaction['text']}", first 8 hex chars.
      - `compute_shuffle_seed(rater: str, session_dir_name: str) -> int`: SHA1 of f"{rater}|{session_dir_name}", first 8 bytes big-endian int.
      - `compute_shuffle_order(rater: str, session_dir_name: str, reactions: list[dict]) -> list[dict]`: deterministic shuffle via `random.Random(seed).shuffle(copy)`.
      - `load_grades_key(grades_dir: Path) -> dict | None`: reads grades.key.json if present; returns None if missing.
      - `write_grades_key(grades_dir: Path, reactions: list[dict]) -> None`: atomic write (tmp + os.replace) of {sha8: {t, text, latency_s}} for every reaction. mode=0o700 on the grades dir, mirroring VoiceRecorder.
      - `verify_grades_key_matches(existing: dict, reactions: list[dict]) -> None`: raises SystemExit with a clear stderr message if the set of sha8s computed from `reactions` is not a subset of the existing key's keys. (Subset, not equal — events.jsonl may have additional non-ai_text entries that don't generate reactions; we only care about ai_text reactions matching the persisted key.)
      - `load_already_graded(rater_jsonl: Path) -> set[str]`: reads <rater>.jsonl line-by-line, returns set of reaction_ids already in the file.
      - `append_grade_atomic(rater_jsonl: Path, record: dict) -> None`: serialize one JSON object + newline, append to file via fsync + open(mode="a"). For atomicity across interruption: write to a tmp file first (single record), then append the validated line and fsync. (Simple approach: open in "a" mode, write the json + "\n", flush, fsync. Partial writes are not possible at the single-line level on POSIX append + flush + fsync. Document this in the function docstring.)
      - `play_clip(voice_wav: Path, t: float, window_s: float = CONTEXT_WINDOW_S) -> None`: extracts the ±window_s segment around t and plays it. v1 implementation: on darwin, use `subprocess.run(["afplay", "-t", str(window_s * 2), <wav>])` with a precomputed temp WAV cut via `wave` stdlib (read voice.wav, slice frames [int((t-window_s)*sr), int((t+window_s)*sr)], write tmp WAV). On non-darwin, log a warning and skip playback (rater grades by transcript only). This is acceptable for v1 per the CONTEXT "darwin-first" implicit scope.
      - `prompt_grade_record(reaction_id: str, rater: str, reaction: dict) -> dict | None`: runs the terminal prompts in order. Returns a dict matching the 11-field schema. Returns None if user types "skip" at the score prompt.
      - `grade_session(session_dir: Path, rater: str) -> int`: top-level orchestration. Returns exit code.
      - `main(argv: list[str] | None = None) -> int`: argparse wrapper. Validates rater enum. Calls grade_session. Returns exit code.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| events.jsonl → grader script | User-controlled file content (text field of ai_text events comes from Gemini output; could in principle contain shell-meta or unicode tricks). Script must treat text strictly as data, never as code. |
| Rater terminal input → JSONL writer | Rater types comments; the comment field must be JSON-escaped, not shell-interpolated. |
| <session_dir> argv → filesystem | Path-traversal class via the positional argument. |
| Voice WAV → audio playback | Passing raw filename to `subprocess.run(["afplay", str(path)])`. Use list-form argv (no shell=True) so path is not interpreted by a shell. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-02-01 | Tampering | grades.key.json after grading starts | mitigate | `verify_grades_key_matches` rejects any extracted reaction not in the persisted key; raises SystemExit with a clear message rather than silently re-grading. |
| T-17-02-02 | Information Disclosure | rater grade file via shared filesystem | accept | Recordings live under app_data_dir() with 0o700 from Plan 15-02. The grades/ subdir inherits 0o700 (mkdir mode=0o700). Local-only attack surface; raters are trusted parties. |
| T-17-02-03 | Tampering | shell injection via session_dir argv | mitigate | Path passed via pathlib.Path constructor; all subprocess.run calls use list-form argv (NO shell=True); ai_text `text` field never reaches subprocess. |
| T-17-02-04 | Denial of Service | malformed events.jsonl line crashes the grader | mitigate | extract_reactions catches per-line JSONDecodeError and logs at WARNING (silently skipping is too quiet; the rater must know which line was malformed); proceeds with remaining lines. |
| T-17-02-05 | Repudiation | rater claims they didn't grade a low score | mitigate | Every grade record includes graded_at_iso operational field with the wall-clock timestamp. Combined with the grader's argv-passed rater identity, the JSONL is a per-rater audit log. |
