    # vibemix — Windows Installer (Inno Setup 6)

    Compiles `dist\vibemix\` (the PyInstaller `--onedir` output from
    `scripts/build_sidecar.py` — or `cargo tauri build`'s release tree) into
    `vibemix-installer.msi`.

    ## Local compile (Kaan on Windows)

    Prereq: Inno Setup 6.4.x installed (https://jrsoftware.org/isinfo.php).

    1. Build the sidecar:
       ```
       uv run python scripts/build_sidecar.py --spec vibemix-core.windows.spec
       ```
    2. Download VC++ Redist into `installer/windows/redist/`:
       ```
       curl -L -o installer/windows/redist/vc_redist.x64.exe https://aka.ms/vs/17/release/vc_redist.x64.exe
       ```
    3. Compile the installer:
       ```
       iscc /DAppVersion=0.1.0 /DSourceDir=..\..\dist\vibemix /DOutputDir=..\..\output installer\windows\vibemix-installer.iss
       ```
    4. Output: `output\vibemix-installer.msi`

    ## CI compile (Plan 18-05)

    `.github/workflows/release.yml` runs the same `iscc` command after
    SignPath-signing every nested .exe / .dll in the source dir + sets
    `/Ssignpath="<signpath-cli-cmd>"` to wire SignTool into the build.

    ## What the .iss does

    Bullet list of the 7 [Setup] decisions per CONTEXT D-Area-3.

    ## SourceDir layout (what iscc consumes)

    ```
    dist\vibemix\
      vibemix.exe              ← Tauri wrapper or PyInstaller entrypoint
      _internal\               ← PyInstaller frozen Python tree
        python312.dll
        ...

          vibemix-core-x86_64-pc-windows-msvc.exe
          _internal\           ← nested PyInstaller tree
    ```

    All nested .exe / .dll must be SignPath-signed BEFORE iscc runs per CONTEXT D-Area-3.
