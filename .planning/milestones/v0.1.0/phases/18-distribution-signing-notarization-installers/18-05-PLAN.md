<objective>
Ship the **release matrix driver** — the GitHub Actions workflow that takes
a `v*` git tag, walks it through the entire Phase 18 chain (build → sign →
notarize → MSI → verify → publish), and lands a Gatekeeper-accepted DMG +
SignPath-signed MSI in a GitHub Release plus a signed updater manifest at
api.altidus.world. This is the load-bearing "release-day" gate; it consumes
the four Wave 1/Wave 2 plans (Plan 18-01 verify_binary.py, Plan 18-02
sign_macos.sh, Plan 18-03 vibemix-installer.iss, Plan 18-04 updater config
+ keys) and binds them into a single executable workflow.

This plan delivers DIST-07 **partially** — the workflow exists and runs end-
to-end in mock-signing mode on PRs; full fresh-machine rehearsal (DIST-07's
"verify a clean install on a fresh non-dev VM") is owned by Phase 20. The
boundary is intentional per CONTEXT §domain: "Out of scope for autonomous
execution: ... Fresh-machine install rehearsal (owned by Phase 20)."

Plan 18-05 closes **DIST-01, DIST-02, DIST-03, DIST-06, DIST-07-partial,
VERIFY-04** by wiring every Wave 1/2 artifact into the CI pipeline.

Purpose:
- Make releases one command: `git tag v0.1.0 && git push --tags`. Everything
  else runs in CI.
- Make every release verifiable + signed end-to-end. Zero unsigned artifacts
  shipped; zero unverified artifacts published; zero leaked secrets in logs.
- Make PR validation cheap. Mock-signing on PRs proves the workflow shape
  without requiring secret access (forks can validate the CI YAML without
  Kaan handing out Apple Developer ID credentials).

Output:
- `.github/workflows/release.yml` — the 2-OS, 5-stage matrix
- `.github/workflows/README.md` — secrets inventory + dry-run docs
- `scripts/dist/sign_manifest.sh` — manifest signing helper (called by
  release.yml; standalone-runnable for Kaan's local rehearsals)
- `docs/release-process.md` — end-to-end runbook
- POC files diff-untouched; zero new deps
</objective>

    ```yaml
    name: Release

    on:
      push:
        tags:
          - 'v*'
      workflow_dispatch:
        inputs:
          dry_run:
            description: "Build + verify only; skip sign + publish"
            type: boolean
            default: true

    permissions:
      contents: write   # Required for softprops/action-gh-release to create the release
      id-token: write   # Optional — for OIDC if used in the future

    env:
      CARGO_TERM_COLOR: always
      PYTHON_VERSION: "3.12"
      NODE_VERSION: "20"

    jobs:
      detect-signing-mode:
        name: Detect signing mode
        runs-on: ubuntu-latest
        outputs:
          signing_available: ${{ steps.check.outputs.signing_available }}
        steps:
          - id: check
            run: |
              set +x  # never echo secret-presence to logs
              if [[ -n "${{ secrets.APPLE_DEVELOPER_ID_P12_BASE64 }}" && -n "${{ secrets.SIGNPATH_API_TOKEN }}" && -n "${{ secrets.TAURI_UPDATER_PRIVATE_KEY }}" ]]; then
                echo "signing_available=true" >> "$GITHUB_OUTPUT"
                echo "Signing secrets present — full release mode."
              else
                echo "signing_available=false" >> "$GITHUB_OUTPUT"
                echo "Signing secrets absent — mock-signing dry-run mode."
              fi

      placeholder-pubkey-gate:
        name: Reject placeholder updater pubkey
        runs-on: ubuntu-latest
        if: startsWith(github.ref, 'refs/tags/v')
        steps:
          - uses: actions/checkout@v4
          - name: Verify pubkey is not placeholder
            run: |
              set -e
              if grep -q 'TAURI_UPDATER_PLACEHOLDER\|dW50cnVzdGVkIGNvbW1lbnQ6IFRBVVJJX1VQREFURVJfUExBQ0VIT0xERVI=' tauri/src-tauri/tauri.conf.json5; then
                echo "::error::tauri.conf.json5 still contains the TAURI_UPDATER_PLACEHOLDER pubkey." >&2
                echo "::error::Generate a real keypair per tauri/src-tauri/keys/README.md and replace the placeholder before tagging." >&2
                exit 1
              fi
              echo "Pubkey placeholder gate clean."

      build-macos:
        name: Build macOS (Apple Silicon)
        runs-on: macos-14
        needs: [detect-signing-mode, placeholder-pubkey-gate]
        if: always() && needs.placeholder-pubkey-gate.result != 'failure'
        env:
          SIGNING_AVAILABLE: ${{ needs.detect-signing-mode.outputs.signing_available }}
        steps:
          - uses: actions/checkout@v4

          - uses: actions/setup-node@v4
            with:
              node-version: ${{ env.NODE_VERSION }}

          - uses: actions/setup-python@v5
            with:
              python-version: ${{ env.PYTHON_VERSION }}

          - uses: astral-sh/setup-uv@v3

          - uses: dtolnay/rust-toolchain@stable
            with:
              targets: aarch64-apple-darwin

          - name: Install macOS build deps
            run: |
              brew install create-dmg

          # Stage 1 — Build sidecar + Tauri bundle.
          - name: Build PyInstaller sidecar
            run: |
              uv sync --frozen
              uv run python scripts/build_sidecar.py --spec vibemix-core.macos.spec

          - name: Build Tauri app
            working-directory: tauri
            run: |
              npm ci
              npm run build
              cd src-tauri
              cargo tauri build

          # Stage 2 + 3 — Sign + DMG + Notarize + Staple (full mode only).
          - name: Sign + notarize + DMG
            if: env.SIGNING_AVAILABLE == 'true'
            env:
              APPLE_DEVELOPER_ID_P12_BASE64: ${{ secrets.APPLE_DEVELOPER_ID_P12_BASE64 }}
              APPLE_DEVELOPER_ID_PASSWORD: ${{ secrets.APPLE_DEVELOPER_ID_PASSWORD }}
              APPLE_DEVELOPER_ID: ${{ secrets.APPLE_DEVELOPER_ID }}
              APPLE_TEAM_ID: ${{ secrets.APPLE_TEAM_ID }}
              APPLE_API_KEY_ID: ${{ secrets.APPLE_API_KEY_ID }}
              APPLE_API_KEY_ISSUER: ${{ secrets.APPLE_API_KEY_ISSUER }}
              APPLE_API_KEY_P8: ${{ secrets.APPLE_API_KEY_P8 }}
              APPLE_DEVELOPER_ID_KEYCHAIN_PASSWORD: ${{ secrets.APPLE_DEVELOPER_ID_KEYCHAIN_PASSWORD }}
              CI: "true"
            run: |
              set +x   # never echo secret env vars
              chmod +x scripts/dist/sign_macos.sh
              ./scripts/dist/sign_macos.sh tauri/src-tauri/target/release/bundle/macos/vibemix.app
              set -x

          # Stage 4 — Verify (always runs, even in mock-signing).
          - name: Verify binary — AIza scan + key-pattern grep
            run: |
              if [[ "$SIGNING_AVAILABLE" == "true" ]]; then
                TARGET="dist/vibemix-$(jq -r .version tauri/src-tauri/tauri.conf.json5 2>/dev/null || echo unknown).dmg"
                if [[ ! -f "$TARGET" ]]; then
                  TARGET=$(ls dist/vibemix-*.dmg 2>/dev/null | head -1 || echo "")
                fi
                if [[ -z "$TARGET" || ! -f "$TARGET" ]]; then
                  TARGET="tauri/src-tauri/target/release/bundle/macos/vibemix.app"
                fi
              else
                TARGET="tauri/src-tauri/target/release/bundle/macos/vibemix.app"
              fi
              echo "Verifying: $TARGET"
              uv run python -m scripts.dist.verify_binary "$TARGET" --report dist/verify-report-macos.json

          - name: Upload macOS artifacts (mock or signed)
            uses: actions/upload-artifact@v4
            with:
              name: macos-artifacts
              path: |
                dist/vibemix-*.dmg
                tauri/src-tauri/target/release/bundle/macos/vibemix.app
                dist/verify-report-macos.json
              if-no-files-found: warn
              retention-days: 7

      build-windows:
        name: Build Windows
        runs-on: windows-latest
        needs: [detect-signing-mode, placeholder-pubkey-gate]
        if: always() && needs.placeholder-pubkey-gate.result != 'failure'
        env:
          SIGNING_AVAILABLE: ${{ needs.detect-signing-mode.outputs.signing_available }}

            shell: pwsh

              npm run build
              cd src-tauri
              cargo tauri build

          - name: Submit signing request to SignPath
            if: env.SIGNING_AVAILABLE == 'true'
            uses: signpath/github-action-submit-signing-request@v1.2.0
            with:
              api-token: ${{ secrets.SIGNPATH_API_TOKEN }}
              organization-id: ${{ secrets.SIGNPATH_ORGANIZATION_ID }}
              project-slug: ${{ secrets.SIGNPATH_PROJECT_SLUG }}
              signing-policy-slug: ${{ secrets.SIGNPATH_SIGNING_POLICY_SLUG }}
              artifact-configuration-slug: vibemix-binaries
              github-artifact-id: ${{ steps.upload-unsigned.outputs.artifact-id }}
              wait-for-completion: true

            if: env.SIGNING_AVAILABLE == 'true' || github.event_name == 'workflow_dispatch'

            run: |
              $version = "0.1.0"   # TODO Plan 19: extract from tag name
              & "${env:ProgramFiles(x86)}\Inno Setup 6\iscc.exe" `
                "/DAppVersion=$version" `
                "/DSourceDir=..\..\dist\vibemix" `
                "/DOutputDir=..\..\output" `
                "/Ssignpath=${env:SIGNPATH_SIGNTOOL_CMD}" `

          # Stage 4 — Verify the signed MSI (always — mock mode verifies the unsigned tree).
          - name: Verify binary — AIza scan + key-pattern grep
            shell: pwsh
            run: |
              $target = if ($env:SIGNING_AVAILABLE -eq "true" -and (Test-Path "output/vibemix-installer.msi")) { "output/vibemix-installer.msi" } else { "dist/vibemix/vibemix.exe" }
              Write-Host "Verifying: $target"
              uv run python -m scripts.dist.verify_binary $target --report dist/verify-report-windows.json

          - name: Upload Windows artifacts
            uses: actions/upload-artifact@v4
            with:
              name: windows-artifacts
              path: |
                output/vibemix-installer.msi
                dist/vibemix/vibemix.exe
                dist/verify-report-windows.json
              if-no-files-found: warn
              retention-days: 7

      release-publish:
        name: Publish to GitHub Releases + sign manifest
        runs-on: ubuntu-latest
        needs: [detect-signing-mode, build-macos, build-windows]
        if: startsWith(github.ref, 'refs/tags/v') && needs.detect-signing-mode.outputs.signing_available == 'true'
        steps:
          - uses: actions/checkout@v4

          - uses: actions/setup-node@v4
            with:
              node-version: ${{ env.NODE_VERSION }}

          - name: Download all artifacts
            uses: actions/download-artifact@v4
            with:
              path: release-artifacts/

          - name: Sign updater manifest (latest.json)
            env:
              TAURI_UPDATER_PRIVATE_KEY: ${{ secrets.TAURI_UPDATER_PRIVATE_KEY }}
              TAURI_UPDATER_KEY_PASSWORD: ${{ secrets.TAURI_UPDATER_KEY_PASSWORD }}
            run: |
              set +x   # never echo private key value
              chmod +x scripts/dist/sign_manifest.sh
              ./scripts/dist/sign_manifest.sh \
                --version "${GITHUB_REF#refs/tags/v}" \
                --macos-url "https://github.com/${{ github.repository }}/releases/download/${{ github.ref_name }}/vibemix-${{ github.ref_name }}-aarch64-apple-darwin.dmg" \
                --windows-url "https://github.com/${{ github.repository }}/releases/download/${{ github.ref_name }}/vibemix-installer.msi" \
                --notes "Release ${{ github.ref_name }}" \
                --output release-artifacts/latest.json
              set -x

          - name: POST signed manifest to api.altidus.world
            env:
              BRAVOH_MANIFEST_UPLOAD_TOKEN: ${{ secrets.BRAVOH_MANIFEST_UPLOAD_TOKEN }}

    #!/usr/bin/env bash
    # SPDX-License-Identifier: Apache-2.0
    #
    # Phase 18 Plan 18-05 — sign the Tauri updater manifest (latest.json).
    # Called by .github/workflows/release.yml after the DMG + MSI are
    # signed + uploaded to the GitHub Release. Outputs a JSON manifest
    # ready for the updater plugin to consume per docs/updater.md.
    #
    # Required env vars:
    #   TAURI_UPDATER_PRIVATE_KEY      — base64 of the .key file contents
    #   TAURI_UPDATER_KEY_PASSWORD     — passphrase chosen at generation
    #
    # Flags:
    #   --version <semver>         — e.g. 0.1.0
    #   --macos-url <url>          — full URL to the signed DMG on the GitHub Release
    #   --windows-url <url>        — full URL to the signed MSI
    #   --notes <text>             — release notes (one-liner)
    #   --output <path>            — write the signed manifest to this path
    #
    # The script:
    #   1. Decodes TAURI_UPDATER_PRIVATE_KEY into a temp .key file (set +x).
    #   2. Calls `npx @tauri-apps/cli signer sign --private-key ... --password ... <payload>` once per platform.
    #   3. Assembles the multi-platform manifest:
    #        {
    #          "version": "0.1.0",
    #          "notes": "...",
    #          "pub_date": "...",
    #          "platforms": {
    #            "darwin-aarch64": { "url": "...", "signature": "..." },
    #            "windows-x86_64": { "url": "...", "signature": "..." }
    #          }
    #        }
    #   4. Writes the manifest to --output.
    #   5. Cleans up the temp .key file via trap.

    set -euo pipefail
    set +x   # private key must NEVER hit logs

    VERSION=""
    MACOS_URL=""
    WINDOWS_URL=""
    NOTES=""
    OUTPUT=""

    while [[ $# -gt 0 ]]; do
      case "$1" in
        --version) VERSION="$2"; shift 2 ;;
        --macos-url) MACOS_URL="$2"; shift 2 ;;
        --windows-url) WINDOWS_URL="$2"; shift 2 ;;
        --notes) NOTES="$2"; shift 2 ;;
        --output) OUTPUT="$2"; shift 2 ;;
        *) echo "Unknown arg: $1" >&2; exit 2 ;;
      esac
    done

    : "${TAURI_UPDATER_PRIVATE_KEY:?TAURI_UPDATER_PRIVATE_KEY required}"
    : "${TAURI_UPDATER_KEY_PASSWORD:?TAURI_UPDATER_KEY_PASSWORD required}"
    : "${VERSION:?--version required}"
    : "${OUTPUT:?--output required}"

    # Materialise the private key into a temp file with a strict trap.
    KEY_DIR=$(mktemp -d)
    trap 'rm -rf "$KEY_DIR"' EXIT INT TERM
    KEY_PATH="$KEY_DIR/vibemix_updater.key"
    echo "$TAURI_UPDATER_PRIVATE_KEY" | base64 --decode > "$KEY_PATH"
    chmod 600 "$KEY_PATH"

    sign_payload() {
      local url="$1"
      # `tauri signer sign` signs the file at URL; for HTTP URLs the CLI
      # downloads first. We use the URL form so the manifest signature
      # matches the bytes the updater plugin will fetch.
      npx --yes @tauri-apps/cli signer sign \
        --private-key-path "$KEY_PATH" \
        --password "$TAURI_UPDATER_KEY_PASSWORD" \
        "$url"
    }

    MACOS_SIG=$(sign_payload "$MACOS_URL" 2>/dev/null || echo "")
    WINDOWS_SIG=$(sign_payload "$WINDOWS_URL" 2>/dev/null || echo "")

    PUB_DATE=$(date -u +"%Y-%m-%dT%H:%M:%SZ")

    cat > "$OUTPUT" <<EOF
    {
      "version": "$VERSION",
      "notes": "$NOTES",
      "pub_date": "$PUB_DATE",
      "platforms": {
        "darwin-aarch64": {
          "url": "$MACOS_URL",
          "signature": "$MACOS_SIG"
        },
        "windows-x86_64": {
          "url": "$WINDOWS_URL",
          "signature": "$WINDOWS_SIG"
        }
      }
    }
    EOF

    echo "Signed manifest written: $OUTPUT"
    ```
