<objective>
Wire Kaan's voice into Gemini as a **literal 2nd multimodal Part** (per AUDIO-01 + locked decision Q1 in CONTEXT.md). Today `KAAN_SPOKE` is a signal-only event — the AI knows Kaan spoke but never hears what he said. v4 (`cohost_v4.py:2253-2310, 1791-1813`) attaches a 12s ring of resampled mic PCM as a second `audio/wav` Part. This plan ports that path verbatim into `src/vibemix/`.

**Hallucination class closed:** "AI invents what Kaan said" — when Kaan asks something verbal, Gemini can answer to the literal words instead of reacting to the abstract fact-of-speech.

Purpose: Anti-slop grounding — replace the KAAN_SPOKE signal-only contract with KAAN_SPOKE + literal mic audio Part.

Output:
- New `mic_audio_buf` instance (16kHz int16 12s ring) wired into the existing mic callback.
- Zero-fill-during-AI-talk to break the self-triggered KAAN_SPOKE feedback loop (load-bearing IP per RESEARCH Pitfall 1).
- `DJCoHostAgent.llm_node` Part 2 attachment gated by `kaan_spoke_recent` (4s window) + signal-presence check.
- Tests pinning both ring behavior and Part-count semantics.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| sounddevice mic callback → mic_audio_buf | Untrusted external audio (room noise / AI feedback / Kaan voice) crosses into the int16 ring shared with `llm_node` |
| mic_audio_buf → Gemini Part 2 | Mic-captured audio leaves the machine via genai client — privacy boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-01-01 | Spoofing | `_mic_callback_factory` AI-talk gate | mitigate | `mic._current_gain() == MIC_GAIN_AT_AI_TALK` zero-fill at callback time prevents AI's own voice (mic-captured feedback) from masquerading as Kaan's voice in the mic Part (RESEARCH Pitfall 1; self-triggered KAAN_SPOKE loop is a real observed failure mode). |
| T-40-01-02 | Information Disclosure | mic Part 2 transmission | accept | Mic audio leaves the machine only when KAAN_SPOKE-recent AND mic ring has signal. User explicitly invoked the co-host. Existing `print_security_banner` (`vibemix.runtime.sec_check`) covers privacy disclosure; no new posture in this plan. |
| T-40-01-03 | DoS | sounddevice callback exception | mitigate | All new code inside the callback runs under `try/except Exception: pass` — audio thread never crashes even on `resample_poly` malformed input (v4:2293-2294 pattern). |
| T-40-01-04 | Tampering | mic_audio_buf state | mitigate | Pre-allocated zero-alloc `AudioBuffer` ring with `threading.Lock` is the same proven primitive used for `clean_audio_buf`; no new concurrency surface introduced. |
