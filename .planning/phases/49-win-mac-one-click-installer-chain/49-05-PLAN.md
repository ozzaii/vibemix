<interfaces>
v3.0 SHIP-04 scaffold defines a 5-row matrix at `scripts/dist/install_vm_matrix.json`. Phase 49 extends it with a 60s gate. Per memory `project_one_click_install_hard_req`, the 60s ceiling is a HARD requirement — never relax.

Per PITFALLS.md, CI runners are stateless + cannot freely spin up Tart VMs (macOS Apple Developer cert + hardware constraints). So:
- On PR / branch builds: run with `--simulate` (reads stub ms from JSON; passes if all stubs < 60s)
- On tag builds: run without `--simulate` (spawns real Tart VMs)

The real-VM run is gated on Kaan-action §INSTALL-VM-RUN. Until that discharges, even tag builds run in simulate mode but emit a WARNING in the workflow log. Verifier writes a report to `dist/vm-matrix-runs/<UTC>/report.json`.

Step-0 worktree-sync invariant per memory `feedback_worktree_must_sync_main_first`.
</interfaces>
