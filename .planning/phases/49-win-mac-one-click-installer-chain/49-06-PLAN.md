<interfaces>
Phase 47/48 established the sibling-script pattern: instead of widening `scripts/launch/check_no_ai_slop.py` pinned target paths (which would re-scope the contract every phase), each phase that needs anti-slop on a new surface creates a sibling-script that IMPORTS the blocklist constant. See `scripts/audit/check_no_slop_opp.py` (Phase 48) as the reference implementation.

The parent script's `AI_SLOP_BLOCKLIST` is a Python list of literal forbidden tokens. It is exported (importable) so sibling scripts can apply the same blocklist to different file globs.

Uninstall data residency: vibemix recordings live at `~/Library/Application Support/vibemix/recordings/` (Mac) / `%APPDATA%\vibemix\recordings\` (Win). Debriefs at sibling `debriefs/` path. Ghost calibration at `ghost_calibration.json`. These are user-owned data; uninstall preserves by default per INSTALL-07.

Step-0 worktree-sync invariant per memory `feedback_worktree_must_sync_main_first`.
</interfaces>
</context>

    | unlock | (rewrite) | Forbidden in product copy |
