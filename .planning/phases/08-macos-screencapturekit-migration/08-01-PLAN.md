<objective>
Wave 1 ships the ScreenCaptureKit migration for macOS:

1. Replace the `mss`-based screen-capture pipeline inside `src/vibemix/platform/_screen_macos.py` with `pyobjc-framework-ScreenCaptureKit`'s `SCStream` callback API. Keep `Quartz.CGWindowListCopyWindowInfo` for window enumeration (NOT deprecated; D-Enumeration API).
2. Use `SCContentFilter.initWithDesktopIndependentWindow:` targeting the picked SCWindow only — no full-screen fallback in shipping code (D-Privacy gate + Success Criterion 4 + P13 prevention).
3. Bridge SCStream's Objective-C delegate callback (runs on a dispatch queue) into asyncio via `loop.run_in_executor` plus a thread-safe ring — same shape as Phase 7's winsdk bridge pattern.
4. Preserve the Phase 1 `ScreenBackend` Protocol shape exactly (`is_available()`, `find_window_bounds()`, `capture()`) + Phase 3's `latest()` + `run_capture_loop()` surface. The selector in `__init__.py` is untouched.

Purpose: macOS 15.0+ forward-compat. `mss` internally calls `CGWindowListCreateImageFromArray`, obsoleted in macOS 15.0. ScreenCaptureKit (available since 12.3) is the supported path.

Constraint: No live ScreenCaptureKit tests in this phase — Phase 16 (hallucination verification) + Phase 20 (fresh-machine rehearsal) are the authoritative live gates. Wave 1 is mocked tests only: a fake SCStream delegate callback producer that hands the code path a synthetic CMSampleBuffer / CVPixelBuffer.

Output: rewritten `_screen_macos.py` + updated `tests/test_screen_macos.py` + `pyproject.toml` delta (add `pyobjc-framework-ScreenCaptureKit`, remove or scope `mss` per Claude's Discretion below).

Discretion (resolved here, documented for executor):
- **Frame-delivery mechanism:** threading-safe ring (single-slot `_ScreenBuffer` already exists — extend with a `latest_frame` slot that the delegate pushes into; asyncio consumer reads via `latest()`). Simpler than `asyncio.Queue` because there's no backpressure semantic — we only ever care about the most recent frame at 1Hz.
- **`mss` disposition:** scope to win32 only — `"mss>=10.2.0 ; sys_platform == 'win32'"` in `pyproject.toml`. `ScreenWindows` still uses it; `_screen_macos.py` removes the `import mss` block entirely.
- **`pyobjc-framework-ScreenCaptureKit`:** add as a separate darwin-marker dep (alongside `pyobjc-framework-Quartz`). pyobjc ships per-framework wheels — verified via `pip index versions pyobjc-framework-ScreenCaptureKit` at execute time.

This is a TDD plan: each impl task pairs a RED test commit with a GREEN impl commit (Phase 7 style: `test(08-01):` then `feat(08-01):`).

Phase 7 async-bridge pattern (the model for Wave 1's SCStream callback bridge), from src/vibemix/platform/_track_windows.py:
```python
async def run_poll_loop(self, stop_event: asyncio.Event) -> None:
    loop = asyncio.get_running_loop()
    while not stop_event.is_set():
        try:
            full = await loop.run_in_executor(None, self._poll_smtc_sync)
            if full:
                self._state.update(full)
        except Exception as e:
            print(f"[track poll err] {e}", file=sys.stderr)
        await asyncio.sleep(1.0)
```

ScreenCaptureKit surface (pyobjc bindings — the keys the impl will call):
- `ScreenCaptureKit.SCShareableContent.getShareableContentWithCompletionHandler_(...)` — fires permission prompt on first call; returns SCWindow / SCDisplay / SCRunningApplication lists.
- `ScreenCaptureKit.SCContentFilter.alloc().initWithDesktopIndependentWindow_(sc_window)` — window-targeted filter.
- `ScreenCaptureKit.SCStreamConfiguration.alloc().init()` — `width`, `height`, `pixelFormat`, `minimumFrameInterval`, `queueDepth` setters.
- `ScreenCaptureKit.SCStream.alloc().initWithFilter:configuration:delegate:(filter, config, delegate)` — instantiate.
- `addStreamOutput:type:sampleHandlerQueue:error:` — register the delegate for `.screen` type on a dispatch queue.
- `startCaptureWithCompletionHandler:` / `stopCaptureWithCompletionHandler:` — async start/stop.
- Delegate callback: `stream_didOutputSampleBuffer_ofType_(stream, sample_buffer, output_type)` — yields a `CMSampleBuffer`. Convert via `CoreMedia.CMSampleBufferGetImageBuffer(sb)` → `CVPixelBuffer` → lock base address → numpy view → PIL.
- SCWindow has `.windowID()` → matches the CGWindowID returned by `kCGWindowNumber` in CGWindowListCopyWindowInfo. Use this to map a Quartz-enumerated window into an SCWindow for the filter.

CMSampleBuffer → JPEG path (the conversion the delegate executes, synchronously inside the dispatch queue before pushing to the ring):
1. `CMSampleBufferGetImageBuffer(sample_buffer)` → CVPixelBuffer.
2. `CVPixelBufferLockBaseAddress(pixel_buffer, kCVPixelBufferLock_ReadOnly)`.
3. Read `CVPixelBufferGetWidth/Height/BytesPerRow/BaseAddress`.
4. Wrap in numpy `frombuffer` (BGRA stride-aware), strip alpha → RGB.
5. `Image.fromarray(...)`; `img.thumbnail((max_width, max_height))`; `img.save(buf, format="JPEG", quality=jpeg_quality)`.
6. `CVPixelBufferUnlockBaseAddress(...)`.
7. Push `(jpeg_bytes, w, h)` into `_ScreenBuffer` via existing `push()`.
