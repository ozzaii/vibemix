# Phase 14 — UI Design Contract

> Migration-style phase. The visual contract is already locked in `mocks/vibemix-direction-final.html` (CDJ Whisper v5). v5 tokens already live in `tauri/ui/src/tokens.css` (commit `0615344`) behind a backward-compat shim. This document is the **per-consumer migration contract** the planner and executor read while dismantling the shim and refactoring components surface-by-surface.

> Generated by gsd-ui-researcher. Verified by gsd-ui-checker + gsd-ui-auditor between cycles; final approval lives in the per-surface entries of `14-POLISH-LOG.md`.

## Design System

| Property            | Value                                                                                          |
|---------------------|------------------------------------------------------------------------------------------------|
| Tool                | none (no shadcn — vanilla TS, pure-function components, project-local `registerStyle()`)       |
| Preset              | not applicable                                                                                 |
| Component library   | none (project-local pure-function components — see `tauri/ui/src/{wizard,session,settings,mascot}/components/`) |
| Icon library        | inline SVG only (vendored under `tauri/ui/src/{wizard,session}/icons/`); no `lucide`, no `@heroicons` |
| Display + body font | **Saira** (variable wdth + wght) — vendored WOFF2 in this phase (replaces `@import` from `fonts.googleapis.com`) |

| Surface order       | wizard → live session UI → settings drawer → mascot overlay window chrome                      |

## Spacing Scale

The v5 token scale is the single source of truth. Both axes already live in `tokens.css`. Component plans MUST consume the `--sp-1..8` axis; legacy `--sp-xs..3xl` aliases are part of the shim and are deleted in the final subtractive commit of this phase.

| Token   | Value | v5 Usage                                                       |
|---------|-------|----------------------------------------------------------------|
| `--sp-1`| 4px   | Icon gaps, LED-to-label gap, inline `gap` inside dense rows    |
| `--sp-2`| 8px   | Compact element spacing, button vertical padding bottom edge   |
| `--sp-3`| 12px  | Top/bottom strip padding (session top-strip, bottom-strip)     |
| `--sp-4`| 16px  | Default element spacing, panel inner padding, CTA row top gap  |
| `--sp-5`| 24px  | Section padding, meter cluster gap, drawer group gap           |
| `--sp-6`| 40px  | Section horizontal padding, surface grid gap, drawer x-padding |
| `--sp-7`| 64px  | Page-level vertical breaks, brief padding-top                  |
| `--sp-8`| 96px  | Major hero spacing, brief padding-top on top section           |

Exceptions:
- Display-window inner padding `14px 22px 16px` (mock-verbatim — keep as literal in component CSS, NOT promoted to a token)
- Border-anim `padding: 1px` (mask trick — keep literal)
- LED dot `width/height 5–9px` (semantic; the dot is the unit, not a spacing token)
- Knurled tray-mac height `28px` (mock-verbatim)
- Mascot overlay window aspect `3/4` (mock-verbatim; pixel size set per Tauri overlay window, not on the inner panel)

**Legacy spacing aliases (kept until the shim-delete commit, then DELETED):**

```
--sp-xs (4)  --sp-sm (8)  --sp-md (16)  --sp-lg (24)  --sp-xl (32)  --sp-2xl (48)  --sp-3xl (64)
```

`--sp-xl: 32px` and `--sp-2xl: 48px` have no v5 equivalent — components currently using them migrate to `--sp-5` (24) or `--sp-6` (40) at the author's discretion; if 32/48 is load-bearing, inline the literal in the component CSS string with a `/* mock-verbatim */` comment.

## Glass + Glow + Atmospherics Contract

### Glass panel anatomy (load-bearing)

Every glass surface (session panel, settings drawer body, wizard primary panel, mascot overlay window chrome, spec panels) follows the same composite:

```
background: var(--glass-N);                                       /* N = 1 | 2 | 3 */
backdrop-filter: var(--blur-glass[-light|-display]);
-webkit-backdrop-filter: var(--blur-glass[-light|-display]);      /* mandatory — Safari/WebKit */
border: 1px solid var(--glass-edge);
border-radius: var(--rad-md);                                     /* 6px default; 8px for mascot window; 2px for inline chips */
box-shadow:
  inset 0 1px 0 var(--glass-top),                                 /* subtle top sheen */
  inset 0 -1px 0 rgba(0,0,0,0.65),                                /* hairline bottom shadow */
  0 36px 80px rgba(0,0,0,0.8),                                    /* deep drop shadow */
  0 0 0 1px rgba(255,255,255,0.018);                              /* almost-imperceptible outer rim */
position: relative;                                               /* required for .border-anim absolute child */
overflow: hidden;                                                 /* required for border-anim mask */
```

Panel size tier → glass + blur token mapping:

| Tier              | Glass token  | Blur token            | Shadow weight                                            |
|-------------------|--------------|-----------------------|----------------------------------------------------------|
| Primary panel     | `--glass-1`  | `--blur-glass` (32px sat 140%) | `0 36px 80px rgba(0,0,0,0.8)`                            |
| Secondary panel   | `--glass-2`  | `--blur-glass-light` (16px sat 120%) | `0 16px 36px rgba(0,0,0,0.55)`                           |
| Recessed display  | `--glass-3`  | `--blur-glass-display` (6px sat 105%)| `inset 0 2px 6px rgba(0,0,0,0.9), inset 0 0 18px rgba(255,138,61,0.035)` |

### Animated border placement (load-bearing)

Drop `<div class="border-anim"></div>` as the **FIRST CHILD** of every:
- Wizard primary glass panel (`.wizard-grid` primary column root)
- Live session glass panel (`.session`)
- Settings drawer glass body (slide-over root, NOT the backdrop)
- Mascot overlay window chrome (`.mascot-window` — the Tauri overlay window's outermost HTML element, NOT the Three.js canvas)

Parent rules:
- Parent must be `position: relative` AND `overflow: hidden`.
- `border-anim` is `z-index: 4`; component content lives at `z-index: 1+`.
- Default duration: `var(--motion-border-sweep)` = 22s.
- Modifier `.slow` → 32s (use on mascot overlay window).
- Modifier `.rev` → reverse direction (pair with `.slow` on the mascot window so primary session + mascot don't sync).

**One sweep per surface** — never nest two `.border-anim` instances in the same visible region. The drawer slide-over and the live session beneath it MAY both have their own sweep, since the drawer occludes the session.

### Backdrop-filter performance fallback (load-bearing)

Two-tier degradation, BOTH shipped in this phase:

1. **`prefers-reduced-motion: reduce` override**
   ```css
   @media (prefers-reduced-motion: reduce) {
     :root {
       --blur-glass: blur(16px);             /* drops saturate */
       --blur-glass-light: blur(8px);
       --blur-glass-display: blur(4px);
       --motion-border-sweep: 0s;            /* freeze the sweep */
     }
     .border-anim { animation: none; opacity: 0.6; }
   }
   ```

2. **Runtime toggle** — Settings → Performance group → "Lighter blur" switch writes `data-blur-perf="on"` on `<html>`:
   ```css
   html[data-blur-perf="on"] {
     --blur-glass: blur(16px);
     --blur-glass-light: blur(8px);
     --blur-glass-display: blur(4px);
   }
   ```
   The runtime toggle does NOT freeze `.border-anim` (animation cost is conic-gradient repaint, not backdrop-filter; cheap on every GPU we ship to).

Acceptance: both code-paths exercised on Kaan's M-series Mac AND one Windows non-dev machine before phase close (the Windows rehearsal can borrow Phase 20's clean VM if needed). Per-machine FPS measurement is deferred to Phase 16; Phase 14 just proves the toggles work.

### Body background contract (load-bearing — already in tokens.css)

The body radial-gradient stack at `tokens.css:259–270` is mock-verbatim. Component plans MUST NOT add their own radial gradients to chrome surfaces. If a surface needs ambient color, it leans on the glass `backdrop-filter: saturate()` pulling color through from the body.

The body `::before` multi-layer film-grain SVG noise blob at `tokens.css:274–285` is also mock-verbatim and required.

### Glass-streak utility

`.vmx-glass-streak` (already in tokens.css:341–361) — a faint diagonal sheen, opt-in per surface. Drop as a sibling of `.border-anim` inside a glass panel. Modifier `.bottom` rotates -8°. Use sparingly — one streak per visible panel max.
