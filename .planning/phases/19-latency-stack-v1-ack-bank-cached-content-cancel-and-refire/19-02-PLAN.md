<objective>
Trim the per-turn prompt for ack-eligible events so Gemini's TTFT drops by the
≥500ms target gated in CONTEXT D-08, and shorten the multimodal payload (audio
Part window + optional screen Part) for the same event classes.

Purpose: Pitfall 9 (prompt bloat) closes here. Diet is event-type-aware so PHASE
and TRACK_CHANGE — the two events where Gemini truly needs the full 18s audio
window + the screen Part to ground a substantive reaction — keep their full
payload. The four ack-eligible event classes (HEARTBEAT, MIX_MOVE, LAYER_ARRIVAL,
KAAN_SPOKE) shrink to a 6s audio window + no screen Part, and the prompt-text
side gets a compressed template that drops history clauses + the evidence-corpus
footer footnote redundancy. This is the cheapest of the four mitigations
(pure-text edit, no API surface change, no LiveKit interaction) and lands the
TTFT win that the ack bank (Plan 19-04) and the cache (Plan 19-03) build on.

Output: `AICoach.build_prompt` extended with `diet: bool = False` kwarg + the
two token-cap constants + ACK_ELIGIBLE_EVENTS set, `DJCoHostAgent.llm_node`
calls the diet path on ack-eligible events and varies `snapshot_wav` window +
screen-Part skip accordingly, two test files pinning the new behavior without
breaking the v4-byte-identical golden tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Event.type → diet dispatch | Event.type string enters AICoach.build_prompt and DJCoHostAgent.llm_node; an unknown type silently falls through to "diet=False" — safe default but masks bugs. ValueError raised on invalid diet=True dispatch. |
| Compact evidence_line → Gemini prompt | The 5-field compact evidence_line drops phase_history / set_arc / track_history fields; Gemini loses some priors on ack-eligible events. The audio Part is the safety net (Gemini still hears 6s of mix). |
| Audio Part window | snapshot_wav window shrinks 18s→6s on diet — if AudioBuffer is undersized at session-warmup, the 6s window may include leading silence. AudioBuffer ring already guards against overrun (per Phase 2 docs); accept the warmup edge case. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-19-02-01 | Information Disclosure | compact evidence_line | accept | Compact evidence intentionally omits 5 history fields on ack-eligible events. The information loss is the design — it buys TTFT — and Gemini's audio Part recovers grounding. Document via test_diet_true_drops_evidence_corpus_footer. |

| T-19-02-03 | Denial of Service | snapshot_wav window | accept | Shrinking the audio window REDUCES API payload — net DoS-resistance improves. No mitigation required. |
| T-19-02-04 | Repudiation | invocation log | mitigate | recorder.log_event("llm_invoke", ...) gains diet + audio_seconds fields → Phase 16 ear-test can correlate Gemini reaction quality to the diet dispatch (durable record per turn). |
