<objective>
Build the Gemini context-caching layer that shaves 500-1500ms off TTFT per call
by reusing a server-side cached prefix containing the static prompt header
(system instruction + persona + citation grammar block from Phase 18).

Purpose: Pitfall 11 (cache-floor under-pad) closes here. The hard rule from
CONTEXT D-08 is that the system instruction must be padded above the 1024-token
Gemini cache floor BEFORE the cache is created — Gemini rejects cache creation
under-floor with a non-obvious 400 error. This plan builds the create-and-pad
path, the 4-minute refresh manager, AND the invalidate hook that Plan 19-01's
CancelGate fires on cancel-and-refire (a stale cache could carry conversational
context from the cancelled turn into the refire — invalidate guarantees the
refire starts clean).

Output: `src/vibemix/agent/cache.py` defining `GeminiContextCache` with
create/current_name/invalidate/refresh_loop, two cache-floor + lifecycle test
files, `DJCoHostAgent` constructor extended with optional `cache` arg + llm_node
threading `cached_content` into `GenerateContentConfig`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GeminiContextCache → google.genai.aio.caches | Cache create/delete crosses to Google's cloud cache store. The cache contains the SYSTEM INSTRUCTION (persona + grammar) — not user data, not Kaan's audio — but a third party (someone observing the cache name) could enumerate vibemix sessions. |
| dj_cohost.invalidate_cache → cache | The invalidate method is a public surface on DJCoHostAgent; any caller (Plan 19-01 CancelGate, Plan 19-04 ack-fire path, future Settings UI) can blow away the cache and force a recreate. No rate-limit on invalidate by design — Plan 19-01's 8s cooldown already throttles the upstream cancel rate. |
| Cache name → log line | Cache name is printed via stderr + written to events.jsonl. The name is a Google opaque token; no PII. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-19-03-02 | Denial of Service | cache.create failure | mitigate | refresh_loop catches create exceptions and KEEPS the old cache; cache-cold path exists in dj_cohost (cache.current_name() returns None → self._gen_cfg fallback). System never goes hard-down on cache outage. |
