TTFTMeter (NEW — vibemix.runtime.ttft):

```python
class TTFTMeter:
    def __init__(self, *, window: int = 8, default_ms: float = 1500.0,
                 time_fn: Callable[[], float] = time.monotonic) -> None: ...
    def record_event_fired(self, now: float | None = None) -> None: ...
    def record_first_chunk(self, now: float | None = None) -> None: ...
    def rolling_avg_ms(self) -> float: ...
```

- `default_ms` (1500.0) is the sentinel returned when the meter has 0 samples — chosen
  so the first event after warmup passes the AckBank TTFT gate (>800ms) and an ack
  fires while the user is still waiting on the FIRST Gemini response of the session.
- `window` (8) bounds the rolling average to the last 8 successful event→first-chunk
  pairs. record_first_chunk is a no-op if no event_fired is currently pending.

AckBank wire-in pattern in coach_loop (per 19-VERIFICATION recommendation):

```python
cancel_cooldown_active = (
    cancel_gate.last_cancel_at != 0.0
    and (time.monotonic() - cancel_gate.last_cancel_at) < CANCEL_COOLDOWN_S
)
fire, reason = ack_bank.should_fire(
    rolling_ttft_avg_ms=ttft_meter.rolling_avg_ms(),
    last_ack_at=trigger_state.get("last_ack_at"),
    last_response_at=trigger_state.get("last_response_at"),
    cancel_cooldown_active=cancel_cooldown_active,
)
if fire:
    bucket, pcm, idx = ack_bank.pick_for_event(ev)
    playback.push(pcm.tobytes())
    trigger_state["last_ack_at"] = time.monotonic()
    recorder.log_event("ack_fire", bucket=bucket, sample_index=idx, reason=reason)
# Fall through to set_next_event + generate_reply per existing flow.
```

CancelGate wire-in (the in-flight handle is now retained in trigger_state):

```python
in_flight_ev = trigger_state.get("in_flight_ev")
in_flight_handle = trigger_state.get("in_flight_handle")
if in_flight_ev is not None and in_flight_handle is not None:
    if cancel_gate.try_cancel(in_flight_handle, ev, in_flight_ev):
        await agent.invalidate_cache()
        trigger_state["in_flight"] = False
        # then proceed to fire ev as the new in-flight
    else:
        continue  # in-flight wins; drop incoming
```

Note: the legacy `single in-flight enforcement` in coach.py runs the body synchronously via
`await asyncio.wait_for(handle.wait_for_playout())`, so today's loop NEVER iterates a second
event while one is in-flight (the loop awaits playout before returning to the top). The wiring
keeps the existing await-based single-in-flight contract — CancelGate becomes the seam to
preempt mid-await ONLY when the planning future builds a fan-in path. For Phase 19 today
the cancel branch is reachable via the Plan 19-04 ack path PRE-fire-event collision detection
(when a still-in-flight handle exists at the top of the next tick — possible when the previous
wait_for_playout raised TimeoutError or an exception cleared in_flight=False but left handle
referenced). The plumbing is the deliverable; the asynchronous fan-in arrives in v2.x.

Concrete Phase-19 in-flight-cancel surface: `allow_interruptions=False` becomes
`allow_interruptions=True` so SpeechHandle.interrupt(force=True) (the chokepoint inside
CancelGate) can actually preempt the playout. The existing wait_for(20s timeout) safeguards
us against a truly stuck handle.
