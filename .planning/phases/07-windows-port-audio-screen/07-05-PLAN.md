    Section 1 — **What this doc is for**: explain that Phases 1-7 ship a cross-platform Python package, Phases 11 + 18 ship a real Tauri + MSI installer for end users. This doc is for running vibemix on Windows from source, today.

    Section 2 — **Prerequisites**:
    - Windows 10 build 19041+ or Windows 11 (WASAPI loopback + winsdk SMTC requirements).
    - Python 3.12.x from python.org (NOT the Microsoft Store version — known PATH issues with PyInstaller).
    - uv (Astral) — `winget install astral-sh.uv` or PowerShell installer one-liner.
    - A DJ controller (DDJ-FLX4 for v1; Phase 9 expands).
    - Default playback device set to 48000 Hz, 16-bit (Control Panel → Sound → Properties → Advanced → Default Format).

    Section 3 — **Install steps** (paste-able PowerShell block):
    - `git clone https://github.com/ozzaii/vibemix.git` + `cd vibemix`
    - `uv sync` — picks up Windows-only deps automatically via sys_platform markers
    - `uv run python Scripts/pywin32_postinstall.py -install` — some Windows builds need this; if path doesn't exist, locate via `uv run python -c "import win32api; print(win32api.__file__)"`
    - `Set-Content .env "GEMINI_API_KEY=your_key_here"` at repo root

    Section 4 — **Sample-rate calibration**: Control Panel → Sound → Playback → device Properties → Advanced → Default Format = 48000 Hz, 16-bit, Stereo. This is the Windows equivalent of macOS Audio MIDI Setup. Phase 7's `AudioWindows.assert_wasapi_loopback_rate` will refuse to start if mismatched.

    Section 5 — **Run**:
    - Sanity check: `uv run python -c "import vibemix.platform; print(vibemix.platform.AudioImpl.__name__)"` → expected `AudioWindows`.
    - Run: `uv run python -m vibemix` (Phase 11 ships the full entry point; today runs the smoke main).

    Section 6 — **DJ controller setup**: plug DDJ-FLX4 → Windows auto-installs Pioneer driver → vibemix's `MidiWindows` finds it via substring "DDJ-FLX4" in port name. Phase 9 expands controller library.

    Section 7 — **Troubleshooting**:
    - "WASAPI loopback device not found" → check default playback device is set + not muted; reboot may help after audio-driver install.
    - "SampleRateMismatchError" → repeat Section 4 with 48000 Hz.
    - "pywin32 ImportError" → run `pywin32_postinstall.py -install` from Section 3.
    - "winsdk ImportError" → `uv pip install winsdk` manually if marker didn't fire (rare; report as issue).
    - SMTC returns no title → expected for djay Pro on Windows; not all DJ apps expose to SMTC. Phase 7 documented limitation; Phase 9/11 may add app-specific scrapers.

    Section 8 — **What's deferred**: Tauri shell + auto-install (Phase 11), MSI installer with SignPath signing (Phase 18), CI matrix on `windows-latest` (Phase 20).

    Format: standard Markdown, code blocks with `powershell` language hint where applicable, no emojis.

    Per CONTEXT Acceptance Criteria — "docs/windows-setup.md deployment doc written".

    Avoid: marketing language. Avoid claiming Phase 7 is "shipped to end users" — it's shipped to Phase 20 CI + Kaan's dev box. Avoid covering Phase 11/18 territory.
