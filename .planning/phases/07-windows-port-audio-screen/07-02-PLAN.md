<objective>
Implement `AudioWindows` ‚Äî the Windows-side AudioBackend impl using **PyAudioWPatch** for WASAPI loopback capture (no virtual cable needed ‚Äî captures whatever's routed to the default playback device) plus standard PyAudio output and mic streams.

Purpose: deliver the v1 Windows win ‚Äî Windows users plug-and-play, no BlackHole equivalent. Match macOS feature set: 48kHz int16 capture buffer fed to AudioBuffer; 24kHz int16 AI voice output to user-selected device; 48kHz stereo passthrough (`PASSTHROUGH_GAIN=0.0` currently disabled, same as macOS); float32 mic input.

Constraint: `import pyaudiowpatch` lives ONLY inside method bodies, NOT at module top ‚Äî so the file imports cleanly on macOS for mocked testing. Mirror the macOS sample-rate guard from Phase 2 (`assert_device_sample_rate`) with a Windows-specific actionable message: "Open Control Panel ‚Üí Sound ‚Üí Properties ‚Üí Advanced ‚Üí set Default Format to 48000 Hz, 16-bit".

PyAudioWPatch API surface (https://pypi.org/project/PyAudioWPatch/):
- `pyaudiowpatch.PyAudio()` ‚Üí instance; `.get_default_wasapi_loopback_device()` returns dict with `'index'`, `'name'`, `'defaultSampleRate'`, `'maxInputChannels'`.

- `.open(...)` for streams ‚Äî params: `format` (paFloat32 / paInt16), `channels`, `rate`, `input=True/output=True`, `input_device_index`, `frames_per_buffer`, `stream_callback`. Returns a Stream object with `.start_stream()`, `.stop_stream()`, `.close()`, `.is_active()`.
- WASAPI loopback: pass the **loopback** index from `get_default_wasapi_loopback_device()` as `input_device_index` and set `input=True`. PyAudioWPatch's WASAPI loopback device indices are distinct from regular WASAPI render devices.
- Constants: `paInt16` = 8, `paFloat32` = 1.
