Step 1 — Add two `#[tauri::command]` functions to `lib.rs`:

  ```rust
  fn resolve_recordings_root(app: &AppHandle) -> Result<PathBuf, String> {
      let base = app.path().app_data_dir().map_err(|e| format!("app_data_dir: {e}"))?;
      Ok(base.join("recordings"))
  }

  fn validate_under_root(candidate: &Path, root: &Path) -> Result<PathBuf, String> {
      let canon_root = root.canonicalize().map_err(|e| format!("root canon: {e}"))?;
      let canon_target = candidate.canonicalize().map_err(|e| format!("target canon: {e}"))?;
      if !canon_target.starts_with(&canon_root) { return Err("path_traversal_rejected".into()); }
      if canon_target == canon_root { return Err("path_traversal_rejected".into()); }
      Ok(canon_target)
  }

  #[tauri::command]
  async fn reveal_in_os(app: AppHandle, session_dir: String) -> Result<(), String> {
      let root = resolve_recordings_root(&app)?;
      let candidate = root.join(&session_dir);
      let safe = validate_under_root(&candidate, &root)?;
      // macOS: `open -R <path>` selects in Finder. Windows: `explorer /select,<path>` selects in Explorer.
      // Linux is OUT OF SCOPE for vibemix v2.0 (CLAUDE.md §Constraints — Platforms: macOS + Windows in v1).
      #[cfg(target_os = "macos")]
      { Command::new("open").args(["-R", safe.to_str().ok_or("path utf8")?]).status().map_err(|e| format!("open: {e}"))?; }
      #[cfg(target_os = "windows")]
      { Command::new("explorer").args(["/select,", safe.to_str().ok_or("path utf8")?]).status().map_err(|e| format!("explorer: {e}"))?; }
      Ok(())
  }

  #[tauri::command]
  async fn open_input_wav(app: AppHandle, session_dir: String) -> Result<(), String> {
      let root = resolve_recordings_root(&app)?;
      let candidate = root.join(&session_dir).join("input.wav");
      let safe = validate_under_root(&candidate, &root)?;
      // tauri-plugin-shell open() launches the OS-default handler for the file (audio app).
      app.shell().open(safe.to_string_lossy().to_string(), None).map_err(|e| format!("shell.open: {e}"))?;
      Ok(())
  }
  ```

  Use `std::process::Command` for the shell-out. Import `use tauri::Manager;` if not already imported (for `app.path()`).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| UI invoke → Rust command | session_dir is untrusted input from the UI process |
| Rust command → OS shell-out | `open -R` / `explorer /select,` invoked with validated path only |
| capability allowlist | shell:allow-open scoped to recordings-root glob — defense in depth on top of validate_under_root |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-03-01 | Tampering | reveal_in_os session_dir param | mitigate | validate_under_root canonicalizes both root + candidate, rejects symlink escapes via canonicalize() following symlinks. Rust unit tests in test_reveal_in_os_command.rs are the gate. |
| T-15-03-02 | Elevation of privilege | shell-out via Command::new("open"/"explorer") | mitigate | Args are passed as separate elements (not concatenated into a shell string), so shell metacharacters in session_dir cannot inject. The validate gate also rejects ../ traversal. |
| T-15-03-03 | Information disclosure | reveal_in_os opens Finder to a path | accept | The path is the user's own recordings directory; Finder showing it is the intended UX. No PII leak — the user is local-only. |
| T-15-03-04 | Tampering | open_input_wav opens arbitrary file in default audio app | mitigate | validate_under_root + the path is hardcoded to `<safe>/input.wav` — never accepts a free-form filename, only a session_dir basename. |
| T-15-03-05 | Denial of service | malformed session_dir causing canonicalize to fail | mitigate | Errors surface as Result::Err to the UI; client logs to console.error. No panic, no app crash. |
</threat_model>
