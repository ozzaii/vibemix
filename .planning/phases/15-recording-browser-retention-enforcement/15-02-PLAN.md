<objective>
Close the periodic-sweep + events.jsonl gaps for ROADMAP success criterion #4 ("Retention policy auto-prunes recordings older than the limit on sidecar startup AND every 6h; events.jsonl logs `retention_pruned: count=N, bytes=M`").

Phase 15 already ships boot + session-close + settings-change sweep triggers. This plan adds:
1. A periodic asyncio sweep task that fires every 6h while the sidecar is alive.
2. An `events.jsonl` line per pruned sweep with the exact shape from CONTEXT.md §"Retention Enforcement".
3. The total-bytes-pruned figure surfaced from `run_retention_sweep` (currently it returns names only — bytes are summed before deletion via a stat pass per entry).

Purpose: the audit in Plan 15-01 confirmed both gaps exist (zero matches for `21600`/`hours=6`/`sweep_loop` and zero matches for `retention_pruned` in `src/vibemix/`). This plan closes both gaps inside the SessionLoop runtime that already owns the boot + close sweeps.

Output:
- Periodic 6h sweep task spawned in `SessionLoop.run()` (cooperative cancellation via `_stop` event).
- `run_retention_sweep` returns `(deleted_names, bytes_pruned)` — a back-compat-aware tuple OR a separate `compute_pruned_bytes` helper called pre-rmtree (decide in implementation per minimal-diff principle).
- A `_log_retention_event_to_active_recorder(...)` shim that locates the live VoiceRecorder (either via SessionLoop holding a reference OR via a small module-level registry in `audio/recorder.py`) and writes `events.jsonl` line.
- All three sweep triggers (boot, periodic, session-close) call the same logging path.
</objective>
