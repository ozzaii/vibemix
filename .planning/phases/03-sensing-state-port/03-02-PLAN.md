<objective>
Port v4's `Event` dataclass (v4:1162-1166) and `EventDetector` class (v4:1169-1325) into `src/vibemix/state/`. The EventDetector is the 7-event taxonomy with per-type cooldowns and the v4 anti-hallucination MUSIC_PRESENCE gate. This is the load-bearing logic that decides WHEN the AI talks — every later wave (AICoach, state_refresh_loop, Phase 4 LiveKit wiring) hangs off it.

**Key design choice (per D-CONTEXT + 03-PATTERNS.md):** v4's `EventDetector` had class-level constants `MUSIC_PRESENCE_MIN_SECONDS = 4.0`, `BPM_VALID_MIN = 100.0`, `BPM_VALID_MAX = 180.0` (v4:1182-1186). Phase 2 lifted these to `vibemix.audio.constants` at module scope so wave-2 can import them directly. The class-attr duplication is REMOVED in the ported version. This is non-negotiable — the source-audit constraint says EventDetector cooldowns + thresholds come from `vibemix.audio.constants`, no redefinition.

Purpose: Lock the event-firing contract before AICoach is written. AICoach reads `event.type` + `event.extra` — those shapes must be stable.

Output: 2 source files + 2 test files. ~200 LOC source, ~250 LOC tests.
</objective>
