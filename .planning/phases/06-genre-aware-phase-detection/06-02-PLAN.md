- `def crest_factor(pcm_int16: np.ndarray) -> float`:
  - If `pcm_int16.size == 0`: return 0.0.
  - `peak = float(np.abs(pcm_int16).max())`
  - If `peak == 0.0`: return 0.0 (digital silence).
  - `rms = float(np.sqrt(np.mean(pcm_int16.astype(np.float64) ** 2)))`
  - If `rms <= 1e-9`: return 0.0 (avoid div-by-zero on near-silent buffers).
  - Return `peak / rms` (a unitless ratio, typically 1.4 for sine, ~3-5 for compressed dance, 8-15 for dynamic acoustic).
  - Cast to float64 for the squared-sum to avoid int16 overflow when summing thousands of samples (Pitfall echo from Phase 2 features.py).

- `class EmaSmoother`:
  - `__init__(self, alpha: float = 0.3, initial: float = 0.0)`: store `alpha`, set `self._value = initial`, set `self._has_seen = False`.
  - `update(self, value: float) -> float`:
    - If not `self._has_seen`: `self._value = value`; `self._has_seen = True`; return `value`.
    - Else: `self._value = alpha * value + (1 - alpha) * self._value`; return `self._value`.
  - `@property value(self) -> float`: return `self._value`.

Add module docstring referencing CONTEXT §Crest Factor Detection.

def validate_bpm(raw_bpm: float, profile: GenreProfile) -> tuple[float, bool]:

Logic (verbatim from CONTEXT D-LOCKED §BPM Half/Double Validator):

1. If `raw_bpm <= 0.0`: return `(0.0, False)`. Zero is `estimate_bpm`'s "no signal" marker; never doubled. (NOT in original CONTEXT but is a defensive guard — pin via test. CONTEXT says "let EventDetector's BPM_VALID gate filter it out" for out-of-range; zero is a special case because doubling zero is zero.)
2. `lo, hi = profile.bpm_range`. Cast both to float.
3. If `lo <= raw_bpm <= hi`: return `(raw_bpm, False)` — in-range pass-through.
4. If `lo <= raw_bpm * 2 <= hi`: return `(raw_bpm * 2, True)` — half-detected, double it.
5. If `lo <= raw_bpm / 2 <= hi`: return `(raw_bpm / 2, True)` — double-detected, halve it.
6. Otherwise: return `(raw_bpm, False)` — out of range entirely; EventDetector's existing BPM_VALID_MIN/MAX gate in `vibemix.audio.constants` will filter it.

Note: prefer the half→double check order from CONTEXT (`raw_bpm * 2` first, then `/ 2`). This handles the 62→124 case before 250→125. Order matters only when BOTH would be in-range — which can't happen unless `hi/lo >= 4` (no profile has that).

```python
import time
from vibemix.state.genre.profile import GenreProfile

class VocalDetector:
    """Hysteresis-bounded vocal-section detector. Returns the CURRENT vocal_active
    bool after applying 1.5s in / 2.5s out dwell timers.

    Heuristics (CONTEXT D-LOCKED §Vocal-Section Detector — any 2 of 3 → 'above threshold'):
        1. mid_share > 0.45 AND mid sustained_for >= 2s (peek at recent_features)
        2. onsets_per_sec rising trend over last 3 snapshots (vocal phrasing)
        3. high_share > 0.20 AND sub_share < 0.30

    Hysteresis: 1.5s sustained above-threshold to flip to True; 2.5s sustained
    below-threshold to flip back to False. Implementation: track timestamps of
    last-seen-above and last-seen-below; flip when the gap exceeds the dwell.
    """

    def __init__(self, profile: GenreProfile | None = None,
                 *, in_dwell_sec: float = 1.5, out_dwell_sec: float = 2.5,
                 mid_share_threshold: float = 0.45,
                 high_share_threshold: float = 0.20,
                 sub_share_ceiling: float = 0.30):
        self.profile = profile  # not used by heuristics in v1; reserved for future per-genre tuning
        self.in_dwell_sec = in_dwell_sec
        self.out_dwell_sec = out_dwell_sec
        self.mid_thr = mid_share_threshold
        self.high_thr = high_share_threshold
        self.sub_ceil = sub_share_ceiling
        # Hysteresis state:
        self._active: bool = False
        self._above_since: float | None = None
        self._below_since: float | None = None
```

`is_vocal_section(self, features, recent_features, *, now=None) -> bool` body:
- If `now is None`: `now = time.time()`.
- Compute three rule booleans:
  - **rule1** (mid dominance): `mid_share = features.get('mid_share', 0.0)`. Then check sustained — if `recent_features` has length >= 2 (the last 2 snapshots in addition to `features`), look at `[features.get('mid_share', 0.0)] + [r.get('mid_share', 0.0) for r in recent_features[-2:]]` and require all >= self.mid_thr. (Sustained-for-2s at 10Hz ≈ 20 snapshots; v1 uses a coarser "last 2 snapshots + current all above" check because state_refresh_loop runs at 10Hz but the recent_features deque from wave 3 will be ~5 snapshots. Use `min(len(recent_features), 2)` snapshots — comment "approximated 2s sustained as 'all of last 3 snapshots ≥ threshold' because feature history depth is ~5 at 10Hz / ~2s tail").
- **rule2** (rising onsets): need at least 3 entries to check trend. `seq = [r.get('onsets_per_sec', 0.0) for r in recent_features[-3:]] + [features.get('onsets_per_sec', 0.0)]`. If `len(seq) >= 3`: `rising = seq[-1] > seq[-2] > seq[-3]`. Else: False.
- **rule3** (vocal sits above sub): `features.get('high_share', 0.0) > self.high_thr and features.get('sub_share', 0.0) < self.sub_ceil`.
- `above_threshold = (int(rule1) + int(rule2) + int(rule3)) >= 2`.

Hysteresis update (the only state mutation):
- If `above_threshold`:
  - `self._below_since = None`.
  - If `self._above_since is None`: `self._above_since = now`.
  - If `not self._active` and `now - self._above_since >= self.in_dwell_sec`: flip `self._active = True`.
- Else (below threshold):
  - `self._above_since = None`.
  - If `self._below_since is None`: `self._below_since = now`.
  - If `self._active` and `now - self._below_since >= self.out_dwell_sec`: flip `self._active = False`.

Return `self._active`.
