      acquires the lock, looks up source then key (return False on KeyError

      lifecycle; called from VoiceRecorder.close() in Plan 18-02 wiring).
    - `__len__(self) -> int`: lock-protected total observation count

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| state_refresh_loop thread → EventDetector → registry writer | Two writers (different threads) must not corrupt the dict — threading.Lock is the gate. |
| Gemini response text → parse_citations | Untrusted LLM output crosses into the parser; regex must reject malformed input without raising. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01-01 | Tampering | EvidenceRegistry concurrent writes | mitigate | threading.Lock guards every read+write; Test 3 is the gate (8-thread torn-write test). |
| T-18-01-02 | Denial of Service | EvidenceRegistry unbounded growth | accept | v1.0 ships unbounded; per-session size is bounded by detector fire count (~500 max for a 1h session per cohost_v4 cooldown gates). Plan 18-02 wires `clear()` on session-close to reset between sessions. |
| T-18-01-03 | Information Disclosure | snapshot() returning live dict reference | mitigate | snapshot() returns a deep copy with tuple-frozen inner lists; Test 4 is the gate. |
| T-18-01-04 | Tampering | Malformed Gemini citation crashing parse_citations | mitigate | regex is anchored to EVIDENCE_SOURCES; non-matches are silently skipped (no exception). Tests 9 + 10 cover whitespace + empty rejection. |
| T-18-01-05 | Spoofing | LLM emitting `[ev:fake@99]` with no backing observation | accept | v1.0 is prompt-only seeding (no enforcement); Phase 20 linter ENFORCEMENT (GROUND-04..08) closes this. Documented in module docstring. |
