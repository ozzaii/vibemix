## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-02-01 | Tampering | Two-thread concurrent registry writes | mitigate | EvidenceRegistry.write uses threading.Lock; Plan 18-01 Test 3 already gates 8-thread torn-write contract. |
| T-18-02-02 | Tampering | Registry write inside state._lock could deadlock | mitigate | EvidenceRegistry uses its OWN lock (not state._lock). Lock order: state._lock OUTER, registry._lock INNER (consistent across all writers). Documented in refresh.py module comment. |
| T-18-02-03 | DoS | Per-tick writes amplifying disk + memory growth | accept | Per-tick aud writes are 7 floats × 10Hz × 1h session ≈ 252k floats ≈ 2MB peak per session. Acceptable for v1.0; Plan 18-02 also wires `clear()` between sessions in __main__.py if needed (deferred to a real wiring plan that touches __main__). |
| T-18-02-04 | DoS | Registry write exception killing the refresh tick | mitigate | All writes wrapped in try/except: pass. The outer `_tick_once` already has top-level try/except at refresh.py:278-296. |
| T-18-02-05 | Repudiation | Phase change observation written without provenance | accept | t_session is unrounded float; matches existing events.jsonl convention. P20 linter will own provenance verification. |
