<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| webview → tauri command | Webview can invoke `set_mascot_click_through` / `set_mascot_visible` — bounded by capability allowlist |
| tray menu → app handle | Menu items emit events into the webview event channel (untrusted strings) — webview validates via existing ajv ipc validator |
| OS window event → store | Window move/resize debounce avoids store-thrash + privacy timing leak |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-02-01 | Tampering | mascot_window state | mitigate | `MascotWindowState` derives Serde with strict types (`Option<i32>` etc.); position values clamped to monitor bounds on load to prevent off-screen-spawn DoS |
| T-13-02-02 | DoS | window move debounce | mitigate | 200ms debounce on `WindowEvent::Moved` writes — prevents thousands of store.save() calls during a drag |
| T-13-02-03 | Elevation | tray-set-mood event | accept | Tray menu emits string into webview event channel; webview routes through existing ipc validator which schema-checks `ipc.settings.set`; no privilege escalation surface |
| T-13-02-04 | Spoofing | tray icon swap | accept | Plan 13-06 owns the icon-swap pump; if compromised, attacker can change the visual indicator but cannot affect audio loop or AI pipeline |
| T-13-02-05 | Info disclosure | mascot window always-on-top | accept | Mascot is intentionally visible across Spaces per requirement MASCOT-02; user can hide via tray left-click. No PII rendered in Wave 1 — Plan 13-04 ships the renderer |
</threat_model>
