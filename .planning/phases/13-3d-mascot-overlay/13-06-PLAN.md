Event → MascotState dispatch (CONTEXT.md Area 3 — verbatim):

| Event subtype           | Target state              | Trigger field           | Beat-lock? |
|-------------------------|---------------------------|-------------------------|-----------|
| TRACK_CHANGE            | react_surprised → idle_bop_to_beat | track_change | react: no; idle_bop: yes |
| PHASE → "drop"          | dance_hard                | drop                    | yes |
| PHASE → "groove"        | idle_bop_to_beat_energetic | phase_change           | yes |
| PHASE → "silent"        | idle_breathe              | phase_change            | no  |
| PHASE → "build"         | idle_bop_to_beat_energetic | phase_change           | yes |
| AI_GENERATING_REPLY     | talk_loop (mood variant)  | ai_generating_reply     | no — interrupt-class |
| AI_REPLY_DONE           | react_yes → previous idle/dance | ai_reply_done     | no  |
| MANUAL                  | react_yes                 | manual_fire             | no  |
| ipc.mascot.mood_change  | puff_particle → idle_breathe (new mood) | mood_swap | no  |

After-state for the two-step events (TRACK_CHANGE, AI_REPLY_DONE, mood_change):
- After react_surprised completes (clip duration), state machine transitions back to idle_bop_to_beat
- After react_yes completes (clip duration), return to whatever was active before
- After puff_particle completes (~500ms), settle on idle_breathe in the new mood pool

Existing ws subscription pattern (reference — `tauri/ui/src/session/ws-bridge.ts`):
- The main session UI listens for `listen<unknown>("ipc:ipc.session.snapshot", ...)` events emitted by the Rust ws_client.rs (which already forwards every inbound bus frame as `ipc:<type>` events).
- The mascot window can EITHER reuse this Tauri-event channel (preferred — single WS source-of-truth in Rust) OR open its own WebSocket directly to 127.0.0.1:8765 (per CONTEXT.md Area 6 — "Mascot overlay window connects to ws://127.0.0.1:8765").

CONTEXT.md decision: direct WebSocket (Area 6 — "Mascot overlay window connects to ws://127.0.0.1:8765" + "Reconnection: WS client reconnects with 1s/2s/4s/8s backoff"). This keeps the mascot loosely coupled to the main UI lifecycle (mascot stays alive even when main window is hidden via tray Quit-prevention from Plan 13-02).
