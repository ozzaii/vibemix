<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| state.active_genre → GenreRouter.swap | Untrusted writer (state_refresh_loop's heuristic) can flip the active genre to any string; router must safely fall back to baseline on unknown values. |
| chain detector ordering | Chain order determines pair contracts (kill before reentry, kill before phrase). A wrong order silently breaks pair detection without raising. |
| `__main__.py` audio_buf threading | If `audio_buf` isn't passed (None), genre detectors that need raw samples (KickSwap, PhraseBoundary) silently can't fire — no exception, no log. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-05-01 | Tampering | GenreRouter.swap("malicious_string") via spoofed state.active_genre | mitigate | Unknown genre → fallback to baseline + WARN log. No code execution path from the string value (registry is a dict lookup only). |
| T-17-05-02 | Repudiation | Genre chain swap mid-detect-call leaves a half-iterated chain | mitigate | Swap happens at the TOP of `.detect()` BEFORE iteration. The iteration uses `self.router.active_chain()` (single attribute read) — no concurrent swap possible inside a single detect call (single-threaded asyncio runner). Tests 4 + 5 in Task 1 pin atomicity. |
