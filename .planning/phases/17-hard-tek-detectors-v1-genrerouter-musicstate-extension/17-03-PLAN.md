<objective>
Ship the two breakdown / re-entry detectors (`BREAKDOWN_KICK_KILL`, `REENTRY_KICK_LAND`) per SENSE-12. These are paired — the re-entry detector reads the kill detector's `.last_kill_at` to know there's a "kick coming back" moment to watch for. Both honor the same silence-gate and cooldown idiom from Plan 02's kick-side detectors.

Purpose: Breakdowns and re-entries are the structurally most important "moments" a DJ ear catches — Kaan's "feels surface-level" critique points squarely at v4 missing these. Pairing them captures the WHOLE arc (kill → silence/filter → kick comes back) instead of firing two unrelated noisy events.

Output: 2 detector classes + 2 test modules; ReentryKickLandDetector takes a BreakdownKickKillDetector instance as a constructor dep (small, explicit coupling — no global state).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| state.bands["sub"] → BreakdownKickKillDetector | Spurious low sub-share readings during quiet passages could fire false kill events. |
| kill_detector.last_kill_at → ReentryKickLandDetector | The pair contract is a single mutable attribute; if Plan 05 wires a stale or wrong instance the re-entry could fire on a never-happened kill. |
| state.beat_phase → ReentryKickLandDetector | beat_phase is a mirror of downbeat_phase from Phase 13 — invalid BPM yields beat_phase=0.0, which would naively pass the downbeat alignment gate (0.0 IS the downbeat). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-03-01 | Tampering | BREAKDOWN_KICK_KILL fires on track end (rms still has tail decay above LOW_RMS for ~1s) | mitigate | `phase == "silent"` is the second silence gate — Phase 6's percentile detector commits silent immediately when the silent_rms profile threshold is breached, beating any tail-decay window. Also: 20s cooldown bounds the false-fire blast radius. |
| T-17-03-02 | Tampering | REENTRY_KICK_LAND fires when bpm_confidence == 0.0 (beat_phase forced to 0.0 by Phase 13 anti-hallucination guard) — every "no BPM lock" frame falsely satisfies the downbeat alignment gate | mitigate | Add a `state.bpm_confidence < 0.5` short-circuit to Step 7: if confidence is too low to trust beat_phase, return None. Document this gate explicitly in the action prose. |
| T-17-03-03 | Repudiation | Two ReentryKickLandDetector instances watching the same kill_detector both fire on the same kill | accept | Plan 05 wires exactly one instance per active genre; GenreRouter's atomic swap ensures only one detector chain is active at a time. Multiple-instance spawn is a wiring bug, caught by Plan 05's integration tests. |
| T-17-03-04 | Information Disclosure | Event extra dict leaks RMS values to AI prompt context | accept | RMS is a derived audio feature, non-PII. Same surface already exposed via existing v4 events. |
