<objective>
Ship `PhraseBoundaryDetector` + the band-limited (40-120Hz) autocorrelation primitives that lock the downbeat phase within ±1 bar (per SENSE-14). The detector fires on downbeats that close 8/16/32-bar phrases — the structural unit DJs use to plan blends. Self-corrects when `BREAKDOWN_KICK_KILL` fires by resetting its phrase-counter (the breakdown reset where the next phrase starts).

Purpose: PHRASE_BOUNDARY is the highest-level structural detector — it lets the AI react to "we're at the END of a phrase" not just "the kick just changed". Pairs with the kick-side detectors from Plans 02/03 to give the AI the FULL grammar of dance-music structure (groove → phrase end → breakdown → re-entry).

The DSP module is split out (`_phrase_dsp.py`) because Plan 06's `tune_detectors.py` harness needs to run band-limited autocorr against reference WAV files WITHOUT instantiating the detector class — pure functions are the right shape for a CLI tuning tool.

Output: 1 detector class + 1 DSP module (pure functions) + 2 test modules.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| audio_buf → band_limited_autocorr | Untrusted PCM — could be silent / mostly-noise / out-of-band content; lock_downbeat_phase must produce honest 0.0 confidence on bad input. |
| state.bpm + state.bpm_confidence → PhraseBoundaryDetector | Phase 13 fabricates beat_phase=0.0 on invalid BPM; this detector multiplies that error if not gated on bpm_confidence. |
| kill_detector.last_kill_at → PhraseBoundaryDetector self-correction | Stale or wrong kill_detector instance could spuriously reset the phrase counter. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-17-04-01 | Tampering | PHRASE_BOUNDARY fires on a fake downbeat (BPM autocorr noise lock) | mitigate | `state.bpm_confidence < 0.5` early-return + `lock_downbeat_phase` requires its own internal confidence ≥ 0.5 to seed. Two independent gates. |
