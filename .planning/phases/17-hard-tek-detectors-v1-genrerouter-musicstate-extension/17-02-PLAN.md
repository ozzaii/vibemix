## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| audio_buf → KickSwapDetector | Untrusted PCM (mic ambient leak via passthrough, BlackHole stale data) drives centroid shift detection. |
| state.bands / state.onset_density → SubLayerArrivalDetector / KickDensityShiftDetector | These fields come from state_refresh_loop's snapshot_features; if upstream is mistuned the detectors fire spuriously. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-02-01 | Tampering | KICK_SWAP fires on track change (centroid shift caused by completely different track, not a within-track kick swap) | mitigate | Cooldown 14s + sub-jump exclusivity (SUB_LAYER_ARRIVAL has its own BPM-stability gate). TRACK_CHANGE detector in `event_detector.py` has higher priority (fires first per existing v4 ordering) and its own 6s cooldown — it claims the moment first. |
| T-17-02-02 | Tampering | SUB_LAYER_ARRIVAL fires on track change | mitigate | Explicit BPM-stability gate (`abs(Δbpm) ≤ 4.0`) — track changes almost always shift BPM by >4 in dance music. |
| T-17-02-03 | Repudiation | Detector fires during silent / phase=silent windows produce phantom events | mitigate | Both `state.rms < LOW_RMS` and `state.phase == "silent"` gates checked first in every detector. |
| T-17-02-04 | Denial of Service | KickSwapDetector takes a 4s snapshot every tick (10Hz) — 40s of redundant FFT work per real-time second | accept | The snapshot is `audio_buf.snapshot(n)` (a memcpy, not I/O) and the FFT is on n=64000 int16 samples — ~5-10ms on the 100ms tick budget. Plan 05's GenreRouter only routes ONE detector chain active at a time, so this is the steady-state cost. |
