<objective>
Ship the SHIP-04 + SHIP-05 engineering scaffolding: a tart-based VM matrix runner driven by a JSON matrix file so adding new OS rows is data, not code. The runner walks the install wizard end-to-end in each VM, captures step screenshots + the onboarding-stopwatch timing dump, and ships a `--check-60s` sub-gate that fails the cut when any VM exceeded 60s end-to-end onboarding.

Purpose: When SHIP-01 + SHIP-02 (Apple Dev + SignPath approvals — external clock) close, Kaan needs the VM matrix to run autonomously against the first signed DMG + MSI artifacts. This plan ships everything except the actual VM images (which Kaan downloads + names per §SHIP-04 runbook) and the live invocation (which is `--live` discharge).

Output: `scripts/dist/install_vm_matrix.{sh,json}` + `tests/install/test_install_vm_matrix.py`. Dry-run by default (no tart calls), zero-network in tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| matrix.sh → tart CLI | matrix.sh executes `tart` subprocesses with arguments interpolated from `install_vm_matrix.json`. JSON is repo-tracked; tampering = code review surface. |
| matrix.sh → VM filesystem | When `--live`, the VM mounts a host dir (`dist/install-vm-runs/<run-id>/`) for screenshot + timing-JSON capture. Files written by VM are read by host (`--check-60s` reads run.json). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-45-01-01 | Tampering | install_vm_matrix.json (matrix row injection) | mitigate | JSON is repo-tracked; schema validation in matrix.sh rejects rows missing required keys; tests assert OS+version set equality against canonical 5. |
| T-45-01-02 | Injection | tart command args from JSON `tart_image` field | mitigate | Pass via array-style args (no shell-string interpolation); pytest pins that `tart_image` field never reaches `eval`/`bash -c`. |
| T-45-01-03 | DoS | matrix runner consumes large run.json | accept | Run.json bounded by 5 rows × O(10) screenshot paths each; max realistic size <50KB. |
| T-45-01-04 | Repudiation | timing dump tampering by VM user | accept | VM is a transient install-verification surface; not a trust-bearing artifact. SHIP-V1-DECISION audit (Plan 45-04) cross-references with GH telemetry for higher-trust signal. |
| T-45-01-05 | Info disclosure | screenshots contain user data | accept | Fresh-VM installs; the "user" is the VM's clean account. No personal data crosses the boundary. |
</threat_model>
