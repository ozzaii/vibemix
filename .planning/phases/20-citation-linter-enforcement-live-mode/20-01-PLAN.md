<!-- Citation atom body shapes (per Phase 18 GROUND-02). The linter MUST handle all 7. -->

  ev:TYPE@t       e.g. "KICK_SWAP@45.2"      — split on @, look up has("ev", "KICK_SWAP", 45.2, tol=1.0)
  aud:key@t       e.g. "bpm@45.2"            — split on @, look up has("aud", "bpm", 45.2, tol=1.0)
  midi:event@t    e.g. "cue_a@12.7"          — split on @, look up has("midi", "cue_a", 12.7, tol=1.0)
  track:id        e.g. "Marlon Hoffstadt - Atlas"  — no @t; look up has("track", "Marlon Hoffstadt - Atlas", t_target=ANY, tol=ANY) → True iff key exists in registry["track"]
  screen:key      e.g. "waveform_deck_a"     — no @t; existence check only
  mix:derived     e.g. "audible_deck=A"      — no @t; existence check only
  tend:fact       e.g. "user_likes_acid"     — no @t; existence check only

For non-time-keyed atoms (track/screen/mix/tend): `valid = body in registry_snapshot.get(source, {})` — presence-only check, tolerance ignored.
For time-keyed atoms (ev/aud/midi) where body lacks `@`: treat as malformed → counts as MISSING for that atom.
A response with at least one valid atom AND zero invalid atoms → LintResult.valid=True.
A response with ANY invalid atom → LintResult.valid=False (response-level strip — Pitfall-2-driven binary decision).
A response with ZERO citation atoms at all → LintResult.valid=False (uncited reply == slop in live mode; ack-bank fallback fires).
