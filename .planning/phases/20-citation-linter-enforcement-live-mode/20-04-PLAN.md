## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| coach_loop → IpcBus.emit | The loop publishes to the Tauri shell over the 127.0.0.1:8765 ws_bus. The bus validates the schema before send (Plan 11 W0 _validate_outbound contract); a malformed payload raises before reaching the wire. |
| citation_telemetry callable → coach_loop | Untrusted at the type level (any callable could be passed); coerced via float()/bool()/.get(default) before construction. A returning dict missing required keys gets safe defaults; a returning non-dict raises AttributeError caught by the try/except. |
| Tauri shell ← ipc.session.citation | Outbound-only message in v2.0; the shell consumes for diagnostics display. No shell→sidecar request/response cycle introduced — read-only telemetry. |
| Settings UI rendering ← lastUnverifiedResponse | The unverified text is Gemini-generated content; rendered with truncation + title attribute (NOT innerHTML). XSS surface limited because the existing Tauri repo uses textContent / template-literal injection patterns (recording-row.ts pattern); the spec test verifies truncation + title behavior. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-04-01 | Tampering | citation_telemetry payload | mitigate | float() / bool() coercion on every field at construction time + .get(key, default) for missing keys; jsonschema validates the [0,1] range before emit. A maliciously-crafted telemetry callable cannot inject extra fields (additionalProperties: false on the schema). |
| T-20-04-02 | Information Disclosure | last_unverified_response in IPC | accept | The unverified text is locally generated by Gemini and shown in the user's OWN Settings UI on their own machine — no cross-tenant exposure. The IPC bus binds to 127.0.0.1 only (Phase 11 invariant). Accept the local-only telemetry surface. |
