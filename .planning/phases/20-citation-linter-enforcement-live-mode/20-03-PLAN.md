<!-- Existing surfaces the replay harness consumes. -->

EvidenceRegistry write API (replay reconstructs the registry by replaying events.jsonl observation rows):
```python
class EvidenceRegistry:
    def write(self, source: str, key: str, t_session: float) -> None: ...
    def snapshot(self) -> dict[str, dict[str, tuple[float, ...]]]: ...
```

CitationLinter API (Plan 20-01):
```python
class CitationLinter:
    def check(self, text: str, registry_snapshot, *, mode: str = "live") -> LintResult: ...

@dataclass(frozen=True, slots=True)
class LintResult:
    valid: bool
    citations_found: int
    missing: tuple[tuple[str, str], ...]
    reason: str
```

Per-invocation dump folder shape (DJCoHostAgent.llm_node writes these — line ~245 + ~458):
```
<session_dir>/invocations/0001_HHMMSS_<EVENT>/
    audio.wav      # the audio Part sent to Gemini
    prompt.txt     # the full prompt (text part only)
    response.txt   # the accumulated full_text from Gemini
    meta.json      # event tag, timestamps, audible state, suppression decision
```

events.jsonl row shape (from Phase 18 EvidenceRegistry sibling-writer):
```json
{"t": 12.34, "kind": "evidence_observation", "source": "ev", "key": "KICK_SWAP@45.2", "t_session": 45.2}
{"t": 12.45, "kind": "evidence_observation", "source": "aud", "key": "bpm@45.2", "t_session": 45.2}
{"t": 12.50, "kind": "ai_text", "text": "...", "latency_s": 1.2}
```

Replay maps every "evidence_observation" row to a registry.write(source, key, t_session). Other "kind" values are skipped (replay only cares about the observation corpus + the response.txt files).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Session dir → replay_linter.py | The session dir path comes from the Phase 16 ear-test operator (Kaan) via CLI arg — fully trusted. Path traversal not a concern (no untrusted shell input). |
| events.jsonl → EvidenceRegistry replay | events.jsonl is produced by the local sidecar (VoiceRecorder) — trusted source. Malformed JSON lines raise json.JSONDecodeError which the script lets propagate (loud failure surfaces a real corruption rather than silent skip). |
| response.txt → CitationLinter | response.txt holds Gemini's recorded text output — same threat surface as Plan 20-01 T-20-01-01. The linter's parse is bounded by the EBNF whitelist; replay inherits that. |
| linter_report.csv output | Written to <session_dir>/linter_report.csv (or --out path). Writes are inside the user's chosen session dir — no traversal risk; subprocess runs with the user's own filesystem permissions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-03-01 | Tampering | events.jsonl observation rows | accept | events.jsonl is local-only, written by VoiceRecorder. Tampering would require local FS write access. Replay's job is to faithfully replay what was recorded; if the audit trail is tampered, the replay reflects the tampered state. Phase 16 ear-test operator verifies session integrity out-of-band. |
| T-20-03-02 | Information Disclosure | linter_report.csv | accept | CSV is written inside the session dir, local-only. raw response text is NOT included (only the missing atoms are exported); the response text stays in response.txt — already on disk. No new disclosure surface. |
| T-20-03-03 | Denial of Service | huge events.jsonl | mitigate | Replay reads the file linearly via for-line-in-file (no whole-file load). A pathologically huge events.jsonl would slow replay but not crash it. Phase 16 sessions are ≤2hr so events.jsonl ≤~50MB — well within the linear-read budget. |
| T-20-03-04 | Repudiation | replay results vs live results | mitigate | Replay uses the SAME CitationLinter class as the live agent (same code path, same constants); the result is byte-for-byte the live decision. Test test_synthetic_session_stripped_rate_below_0_15 pins the contract on the synthetic fixture; Phase 16 swaps in real fixtures and gates the same threshold. |
</threat_model>
