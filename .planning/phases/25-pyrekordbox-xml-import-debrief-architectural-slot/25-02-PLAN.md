```python
@dataclass(frozen=True, slots=True)
class CuePoint:
    name: str          # rekordbox cue label or "" if untitled
    type: str          # "cue" | "loop" | "fadein" | "fadeout" | "load"
    start_s: float     # seconds from start of track
    end_s: float | None  # only set for type="loop"
    number: int        # 1..8 hot-cue index, or -1 for memory cues


@dataclass(frozen=True, slots=True)
class TrackEntry:
    track_id: str                    # Rekordbox TrackID — primary key
    title: str
    artist: str
    album: str
    bpm: float                       # 0.0 when missing in XML
    key: str                         # rekordbox Tonality string (e.g., "Am") or ""
    duration_s: float                # 0.0 when missing
    cues: tuple[CuePoint, ...]
    filepath: str                    # Location (URL-decoded)


class RekordboxLibrary:
    """One-shot Rekordbox collection.xml loader. Lifecycle:

      lib = RekordboxLibrary()
      lib.load_xml(path)
      lib.tracks  # dict[str, TrackEntry] keyed by track_id

    The XML parser is `pyrekordbox.RekordboxXml` (Plan 25-01 spike-locked).
    SQLCipher path is never touched. Staleness nudge fires via logger.info
    when `mtime > 30d`. Cache pickle at `~/.cache/vibemix/library.pkl` is
    populated on successful `load_xml` and consumed by `try_load_cache` for
    warm-start; a missing or corrupted cache silently falls back to fresh
    parse (no exception bubbling). Cache versioning: stored payload is a
    `{"version": 1, "xml_path": str, "xml_mtime": float, "tracks": dict}`
    dict; v2.x bumps `version`.
    """

    SCHEMA_VERSION = 1
    CACHE_PATH = Path.home() / ".cache" / "vibemix" / "library.pkl"
    STALE_AGE_DAYS = 30

    def __init__(self) -> None: ...
    def load_xml(self, path: str | Path) -> int: ...     # returns track count
    def try_load_cache(self) -> bool: ...                 # True on cache hit, else fall through
    def lookup_by_id(self, track_id: str) -> TrackEntry | None: ...
    def __len__(self) -> int: ...
```

Implementation notes:

- Use `pyrekordbox.RekordboxXml(path)` to parse the file. Iterate `xml.get_tracks()` (the documented API). For each track, build `TrackEntry`: pull `TrackID`, `Name`, `Artist`, `Album`, `AverageBpm`, `Tonality`, `TotalTime` (seconds), `Location` (URL-decoded via `urllib.parse.unquote`).
- For cues, iterate `track.get_cuepoints()` — pyrekordbox exposes `Num`, `Name`, `Type`, `Start` (seconds), `End` (seconds, only on loops). Type code maps: `0`→"cue", `1`→"fadein", `2`→"fadeout", `3`→"load", `4`→"loop".
- Empty / None values become typed empties (`bpm = 0.0`, `key = ""`, `cues = ()`).
- After parse, write the cache: `pickle.dumps({"version": 1, "xml_path": str(path), "xml_mtime": os.path.getmtime(path), "tracks": dict_of_track_id_to_entry})` to `CACHE_PATH` (mkdir parents=True). On warm start, `try_load_cache()` reads, checks version + xml_path match + xml_mtime ≥ cached mtime, and on any mismatch returns False so the caller re-parses.
- Staleness nudge: in `load_xml`, after parse, if `(time.time() - os.path.getmtime(path)) > STALE_AGE_DAYS * 86400`, emit `logger.info("library: collection.xml is N days old — re-import via Settings → Library when ready")` where `N = int(age_days)`. Log-only, no UI surface in v2.0 (LIBRARY-06 defers UI).
- Use stdlib `logging.getLogger("vibemix.library")` — no rich console / no print.
- NO `db6` import. NO `Rekordbox6Database` use. Grep gate (Plan 25-03 task-level): `grep -r "Rekordbox6Database\|pyrekordbox.db6" src/vibemix/` must return empty.
- Tolerate non-track playlist nodes in the XML root (folder/playlist tags) — iterate ONLY via `xml.get_tracks()`, never walk the XML tree by hand.

Concurrency: `RekordboxLibrary` is single-threaded by intent — loaded once at sidecar start, then read-only. No locks needed.

```python
def register_library(self, lib, t_session: float = 0.0) -> int:
    """Register every TrackEntry in `lib` as a `track:<id>` observation.

    Phase 18 EvidenceRegistry already accepts `track` as a valid source
    (see EVIDENCE_SOURCES — line 70). The Phase 20 CitationLinter walks
    `EVIDENCE_CITATION_RE` matches and calls `has(source, key, t_target)`;
    registering library IDs here lets `[track:<id>]` citations from Gemini
    resolve against the user's real collection.

    `t_session=0.0` is the canonical "library load" timestamp — it's the
    moment of session start so any in-session citation lookup (tol=±1.0s
    live, ±2.0s debrief per GROUND-07) falls outside this window unless
    the linter explicitly skips the timestamp check for `track` source.
    Phase 25-03 wires the linter behavior; for v2.0 we just register.

    Returns the number of entries registered. Idempotent — re-registering
    the same library just appends duplicate timestamps; tolerable because
    `has()` checks any-of, not unique.
    """
    count = 0
    for track_id in lib.tracks:
        self.write("track", track_id, t_session)
        count += 1
    return count
```

Place it in the registry class between the `write()` / `clear()` block and the `snapshot()` reads — keep writers grouped.

| user filesystem → vibemix | `collection.xml` is user-supplied (untrusted XML); pyrekordbox uses lxml internally with default `resolve_entities=False` semantics on its XmlAttributeKeyError parser. |
| vibemix → ~/.cache | Pickle cache load uses `pickle.load`; trust boundary is "user's own machine" — same trust as the source XML. |
