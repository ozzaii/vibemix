# 12-05: Settings drawer + push-to-mute UX + integration + SUMMARY

## Goal

Final wave — Settings slide-over drawer with hot-reload wiring, push-to-mute UX polish, integration tests, and the phase SUMMARY. Per `12-UI-SPEC.md` §13 (Settings Drawer), §14 (Muted Banner) and `12-CONTEXT.md` §Area 2.

## Steps

1. **`settings/state.ts`** — small local state for drawer:
   - `{open: boolean, hotkeyCaptureMode: boolean, pendingGenreReload: boolean, confirmDialog: null | "re-run-calibration"}`.
   - Setters notify SettingsDrawer to re-render.

2. **`SettingsDrawer.ts`** composer:
   - `mountSettingsDrawer(root)` — appends drawer + backdrop nodes at z-index 50 (initial `translateX(100%)`).
   - `openSettings()` / `closeSettings()` toggle slide animation.
   - Reads `SessionState.settings` for current values; calls `sendSettings(field, value)` on changes.
   - Esc key listener while open → close.
   - Backdrop click → close.

3. **Group components** under `settings/components/`:
   - **`group.ts`** — generic group wrapper (panel header + body).
   - **PERSONA group**:
     - Voice picker (reuse `session/components/picker.ts`).
     - Mode rocker (reuse `session/components/rocker.ts`).
     - Genre dropdown + `pendingGenreReload` overlay (250ms `--phosphor-soft` "RELOADING PROFILE…").
   - **OUTPUT group**:
     - Device picker.
     - Profile rocker (HP / SPK).
   - **HOTKEY group** — `hotkey-capture.ts`:
     - Display chip (DM Mono 14px `--phosphor`).
     - `[ ↻ Rebind ]` button → enters capture mode.
     - On capture: swallow keydown events; require `mods.length >= 1 && key !== null`; validate against reserved set.
     - On valid capture: call `sendSettings('push_to_mute_hotkey', combo_string)` + send `rebind_hotkey` Tauri command.
     - Reserved-set: macOS `cmd+q/w/tab/space/shift+tab`, Windows `alt+f4 / ctrl+alt+del / win+l`.
   - **RECORDING group** — `retention-slider.ts`:
     - 6-stop slider (1d / 3d / 7d / 14d / 30d / ∞).
     - Knurled-knob visual: `--panel-deep` circle, 1px `--phosphor-dim` border, glow on hover.
     - DSEG7 22px readout above slider.
     - On change: call `sendSettings('retention_days', value)`.
   - **CALIBRATION group**:
     - `[ ↻ Re-run wizard ]` button → open `confirm-dialog.ts`.
     - On confirm: send `ipc.wizard.start` to sidecar (Phase 11 IPC). Tauri shell routes to wizard, tears down session.

4. **`confirm-dialog.ts`** — generic modal confirm:
   - Heading + body + 2 buttons.
   - Used for "RESTART CALIBRATION?".
   - Esc dismiss.

5. **`session/SessionLayout.ts`** update:
   - Wire gear button onClick → `openSettings()`.
   - Mount muted-banner inside cohost-panel when `state.muted === true`.

6. **Rust** (`tauri/src-tauri/src/lib.rs`):
   - `rebind_hotkey` Tauri command — unregisters old shortcut, registers new, persists via sidecar `ipc.settings.set hotkey`.

7. **Tests**:
   - `tests/settings/drawer.spec.ts` — open / close, Esc dismiss, backdrop click.
   - `tests/settings/hotkey-capture.spec.ts` — capture state machine, reserved-combo rejection.
   - `tests/settings/retention-slider.spec.ts` — 6 stops, value emission, DSEG7 update.
   - `tests/session/integration.spec.ts` — full boot path: post-wizard config → session mount → settings open → setting change → assert IPC fired → drawer close.

8. **`12-SUMMARY.md`**:
   - Phase outcome: pass/partial/deferred.
   - Component inventory (all files created + lines of code).
   - IPC message families added (count from schema diff).
   - Verification matrix per success criterion 1-5.
   - Deviations from spec (if any).
   - Deferred items:
     - Reactive mascot fill (Phase 13).
     - FL-Studio polish loop (Phase 14).
     - Recording browser (Phase 15).
     - Auto-update + signing (Phase 18).
   - Handoffs:
     - Phase 13: mascot 256×256 reserved corner + 42×42 cohost-header mascot circle = mount points.
     - Phase 14: knurled-knob shadows + scanline shimmer + screw-head polish layer.
     - Phase 15: `retention_days` setting in config — read on boot for cleanup policy.

- Manual UAT scenarios (autonomous=false — these are intentionally human-validated since they require running app + audio + hotkey):
  1. Open Settings drawer, change voice — next AI turn uses new voice.
  2. Change genre — see 250ms reload overlay; sidecar log shows profile reload.
  3. Change output device — audio resumes through new device <500ms.
  4. Capture new hotkey (e.g. `⌥⇧K`) — press it → mute fires.
  5. Re-run calibration → wizard mounts; on completion → session re-mounts at same state.
  6. Pull DDJ-FLX4 USB during session → MIDI badge flips `--rec · 0` within 2s.
