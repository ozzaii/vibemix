# 12-02: Sidecar session loop + settings dispatch

## Goal

The Python sidecar surface for Phase 12 — `SessionLoop` runs after wizard completion (Phase 11), emits 30Hz `ipc.session.snapshot` from existing `MusicState`, and routes inbound `ipc.session.*` / `ipc.settings.*` messages to the right hot-reload hook. Per `12-CONTEXT.md` §Area 5 "WS Message Schema & Hot-Reload Contract" and §Area 2 "Settings Panel UX & Persistence".

## Steps

1. **`config_store.py`** — JSON config persistence:
   - Read/write `~/Library/Application Support/vibemix/config.json` (mac) / `%APPDATA%\vibemix\config.json` (win).
   - Schema is superset of Phase 11's. Add fields: `voice` (default `"kore"`), `mode` (default `"coach"`), `genre` (default `"tech-house"`), `output_device_id` (default `null` → auto), `output_profile` (default `"hp"`), `retention_days` (default `7`), `push_to_mute_hotkey` (default `"cmd+shift+m"` on mac, `"ctrl+shift+m"` on win).
   - Round-trip safe with Phase 11 fields (don't clobber `first_run_completed`, `calibrated_at`, etc.).
   - Atomic write (`tmp` + `os.replace`).

2. **`settings.py`** — `SettingsApplier`:
   - Take refs to `cascade_agent`, `event_detector`, `audio_core`, `config_store`, `genre_loader`.
   - `apply(field, value)` switches on `field` and calls the matching hook. Returns `(success: bool, error: str|None)`.
   - Genre apply: reload `genre_profiles.{genre}.json` via existing Phase 6/10 loader; emit 250ms-tagged ack.
   - Voice apply: `cascade_agent.set_voice(value)` — next TTS turn picks up.
   - Mode apply: `event_detector.set_mode(value)` + reload corresponding prompt template.
   - Output device apply: `audio_core.restart_output_stream(device_id=value)` — must complete in <500ms; returns success only after stream resumes.
   - Output profile apply: `audio_core.set_mic_gating_profile(value)` — `hp` → existing mic gate; `spk` → force `mic.muted=True`.
   - Retention apply: persist only (Phase 15 reads at boot).
   - Hotkey apply: persist only (Tauri shell rebinds via `tauri-plugin-global-shortcut`).

3. **`session_loop.py`** — `SessionLoop`:
   - On entry: register `ipc.session.mute`, `ipc.settings.set`, `ipc.settings.get`, `ipc.status.recheck` handlers on the existing `ws_bus`.
   - Start a 30Hz background task that builds + emits `ipc.session.snapshot` from `MusicState` + `EventDetector` history + transcript ring (initially empty; Phase 4 cascade integration in 12-04).
   - Maintain transcript ring (200 entries) — append on TTS line + on user-detected-speech (Phase 3 mic chunks via Voice STT placeholder; for now just AI lines).
   - Maintain `muted: bool` — on `ipc.session.mute` toggle, call `playback_queue.clear()` and flip flag.
   - Validate every inbound message through `vibemix.ui_bus.validator` (Phase 11) before dispatch.
   - On invalid payload, emit `ipc.error` event with reason; never crash the loop.

4. **`__main__.py`** — wire `SessionLoop` to run after wizard completion (when `--wizard` not set OR wizard finished):
   ```python
   if args.wizard:
       asyncio.run(run_wizard())
   else:
       asyncio.run(run_session())
   ```

5. **Tests**:
   - `tests/runtime/test_config_store.py` — round-trip read/write; superset compatibility with Phase 11 config; atomic write under fault injection.
   - `tests/runtime/test_settings_apply.py` — mock all hooks; assert dispatch matrix for each field.
   - `tests/runtime/test_session_loop.py` — fake `MusicState` + fake `ws_bus`; assert snapshot shape + mute toggle path + invalid-payload error path.

## Verification

- `pytest tests/runtime/ -q` green for the new tests.
- `python -m vibemix` (no wizard flag, with first_run_completed=true config) emits `ipc.session.snapshot` to a connected WS client at ≥25Hz (under 30Hz target with 5Hz tolerance).
- Manual: drive an `ipc.settings.set` for each field via a WS test client, observe matching effect (voice ID logged, output device restarted, etc.).

## Notes

- DO NOT touch the audio thread / sounddevice callbacks — settings apply via existing `restart_*` methods only.
- All settings apply must complete on the asyncio loop, not block audio callbacks.
- Transcript ring lives in `SessionLoop`, not in `MusicState` — it's a UI concern.
- Phase 11's `WizardLoop` and `SessionLoop` are mutually exclusive — wizard finishes, then sidecar restarts in session mode.
