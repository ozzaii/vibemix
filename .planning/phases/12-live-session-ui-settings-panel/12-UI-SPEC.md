# Phase 12 — UI Design Contract

> Visual and interaction contract for the live session UI + Settings drawer. Generated by gsd-ui-phase orchestrator (autonomous mode); consumed by gsd-planner, gsd-executor, gsd-ui-checker, gsd-ui-auditor.

**Visual identity rule (load-bearing):** Phase 12 IS the live-session surface from `mocks/vibemix-app-ui.html` rendered as a real Tauri application. Every component is ported 1:1 from the mock — same anodised charcoal, same phosphor amber accent, same DSEG7 numerics, same WORKBENCH wordmark, same brushed-metal panels, same paper-textured phase tape + transcript, same film grain + scanlines overlays. NOT a "reinterpretation". The mock is the executable design contract.

This spec inherits the entire token system, font stack, atmospheric overlays, and component shells from Phase 11 (`.planning/phases/11-tauri-shell-calibration-wizard/11-UI-SPEC.md` §"Design System", §"Color", §"Typography", §"Spacing Scale"). All Phase 11 tokens carry forward unchanged. Phase 12 adds NO new tokens — only new component compositions on top of the existing token set.

| Theme | dark-only — NO theme switch, NO `prefers-color-scheme`, NO light mode |
| CSS scoping | `registerStyle("componentName", css)` singleton — same pattern as Phase 11 wizard |

## Color

UNCHANGED tokens from Phase 11. Phase 12 surfaces every accent-color use case from the mock:

### 80% Dominant — Anodised Charcoal Family

Panels (`--panel`, `--panel-lift`, `--panel-deep`), bezels (`--bezel-1/2/3`), brushed-metal streak overlays (`--brushed-hi`, `--brushed-lo`), grooves (`--groove`) — all unchanged. Every column gradient mirrors mock `.col` (lines 184-205): `linear-gradient(180deg, var(--panel-lift) 0%, var(--panel) 100%)` with `inset 0 1px 0 rgba(255,255,255,.05)` + brushed-metal `::before` overlay.

### Ink (text on charcoal)

`--ink`, `--ink-dim`, `--ink-deep`, `--ink-engraved` — unchanged.

### 20% Accent — Phosphor Amber

Single accent — `--phosphor` does ALL the lifting. Phase 12 expanded reserved-for list:

1. Wordmark in titlebar
2. Active mode rocker position (`pro` / `coach`)
3. Active "interaction" rocker LED (hype OR coach — exactly one lit)
4. DSEG7 timecode hero (`02:44:31` text + `--phosphor-halo`)
5. DSEG7 BPM readout in timecode-block meta
6. DSEG7 drop-countdown chip
7. Phase tape `groove` chunks (`--phosphor-soft` tint) + `build` chunks (striped `--phosphor` repeating gradient)
8. Phase tape `drop-ghost` chunk dashed border (`--phosphor-warm` ish #c8901a — vendored tint in mock, allowed)
9. Transcript "›" leading caret on each message (`--phosphor-warm`)
10. Transcript inline emphasis (`<em>` — sticker font, `#a8540a` warm-amber, allowed)
11. Transcript live-cursor "▍" pulse
12. Active meter peak segment (top 3 LEDs of music meter, top 2 of voice/mic meters)
13. Mascot cohost-name "AVERY" in transcript header
14. Status-bar `OK` LEDs (`--ok` for green, `--phosphor` allowed only for `connecting` slow-pulse)
15. Settings gear button when armed/hover
16. Settings drawer active row tinted background (`--phosphor-soft`)
17. "made by bravoh" footer signature (Caveat 14px)
18. Drop-chip glow + countdown
19. MIDI event ribbon "now" event slot (`.evt-now` from mock — `--phosphor` with glow)

Forbidden uses (unchanged from Phase 11):

- Body copy, microcopy, status descriptions
- Full panel fills (max allowed: `--phosphor-soft` tint at 18% opacity)
- Section dividers, bezel rings
- Background gradients on the main grid

### Status-only accents (LED dots only — never panel fills)

| Token | Hex | Usage |
|-------|-----|-------|
| `--rec` | `#ff3553` | REC LED dot, `NOW` marker on phase tape, MUTED banner border, status badge `down` state, hot-unplugged MIDI |
| `--cue` | `#4dc4ff` | Reserved — not used in v1 live UI (cue lines deferred to Phase 14 polish) |

### Accent-reserved enforcement (10% rule)

By inventory of the mock-driven surface, phosphor amber accent occupies < 12% of the rendered pixel area at any moment (the rest is charcoal + ink + the two paper surfaces). This passes the `frontend-enforcement` 20/80 rule with margin.

## Component Contracts

### 1. Titlebar (extends Phase 11)

Phase 11 titlebar reused. Phase 12 ADDS:

- **Live status pills cluster** (mock lines 1148-1172): three pills inline after wordmark — `● LIVE` (LED `--ok`), `● REC` (LED `--rec`, blinks 1.4s while recording), `● SYS` (LED `--ok` or `--rec` based on aggregated component health). - **Live-clock readout** (mock line 1170): DSEG7 22px `02:44:31` in `--phosphor` with `--phosphor-glow`. Updates every 250ms (rAF-throttled).
- **Settings gear button** (NEW — mock has no exact analogue; bespoke): inline-SVG gear glyph 18×18px, Workbench 11px "SETTINGS" label hidden visually but exposed to AT. Idle: glyph `--ink-dim`. Hover: `--phosphor` + `--phosphor-glow`. Active (drawer open): `--phosphor` + `--phosphor-soft` background pill. Click opens drawer.

### 2. Persona Panel (left column — mock lines 1190-1234)

Panel shell: standard `--panel-lift → --panel` gradient + brushed-metal `::before` + `inset 0 1px 0 rgba(255,255,255,.05)`. 1px `--bezel-1` border, 8px border-radius.

Panel header (mock `.panel-h` lines 295-310): "PERSONA" Workbench 9px UPPERCASE 0.32em + optional `--phosphor-soft` badge "CFG" on right.

Inner stack:

- **Skill mode rocker** (mock `.mode-rocker` lines 351-405) — 3-state segmented rocker `[ BEG | INT | PRO ]`. Rocker bezel: `inset 0 2px 4px rgba(0,0,0,.5)` to look pressed-in. Single-row, 32px tall.
- **Interaction rocker** (mock `.interaction` lines 408-438) — 2-state radio `[ ● HYPE | ● COACH ]` with LED indicators. Same rocker visual language. Active LED: `--phosphor` with `--phosphor-glow`. Inactive LED: `--ink-engraved`.

- **Genre dropdown row** — same component shape as voice picker. Options pulled from `genre_profiles.list()`. Selected option: DM Mono 14px weight 500 `--ink`. Click-open animates 200ms ease-out (slide down 8px + opacity 0→1).

### 3. Output Panel (left column — mock lines 1238-1285)

Panel header: "OUTPUT".

Inner stack:

- **Device row** — same voice-picker visual; left glyph swaps to inline-SVG headphones/speakers SVG (24×24, `--phosphor` stroke 1.5px). Name DM Mono 14px weight 500. Auto-detected device shows `--phosphor-soft` "AUTO" pill.

### 4. Meter Strip (left column — mock lines 1289-1334, `.meter` class lines 530-571)

Three vertical-stack LED meters in a row: `music`, `voice` (AI), `mic` (user).

Per meter:
- Width 56px, height 200px
- Frame: `--panel-deep` background, `--bezel-1` 1px border, 4px border-radius, `inset 0 2px 6px rgba(0,0,0,.7)`
- 16 LED segments stacked top-to-bottom, each 10px tall × 36px wide, 2px gap
- Segment LED colour mapping (peak hold):
  - Top 3 segments: `--phosphor` lit at peaks (clip warning)
  - Middle 8: `--phosphor` (warm, dimmed to `--phosphor-warm`)
  - Bottom 5: `--ok` (safe headroom)
  - Unlit segments: `--ink-engraved` (visible-but-dead)
- LED segment shape: rounded rectangle, `box-shadow: 0 0 4px var(--phosphor)` when lit
- Label below frame: Workbench 9px UPPERCASE 0.22em `MUSIC` / `VOICE` / `MIC` in `--ink-dim`
- Peak-hold dot above current level: single `--phosphor` segment held 1.2s before decaying (lifts mock peak behaviour)

Update path: WS `ipc.session.snapshot.{music,voice,mic}.rms` → CSS `--meter-{music,voice,mic}-rms: 0.0..1.0` → segments rendered conditionally based on `var(--meter-music-rms) > segment_threshold`. JS only touches the CSS variable; segments self-render — no layout thrash.

### 7. Drop Countdown Chip (center column — mock lines 871-928, `.drop-chip`)

Renders only when `phase == "BUILDUP"` AND `drop_pred != null` AND `drop_pred <= 16` bars.

Inline-flex chip:
- Padding: 10px 16px
- Background: `linear-gradient(180deg, #1a1e25 0%, #0e1014 100%)`
- Border: 1px solid `--phosphor-dim`, 6px border-radius
- Box-shadow: `inset 0 1px 0 rgba(255,255,255,.06), var(--phosphor-glow)`
- `::before` halo: 8px-blurred radial of `--phosphor-soft`

Internals (horizontal flex, gap 12px):
- `⟶` DSEG7 14px `--phosphor` arrow
- DSEG7 22px `--phosphor` countdown digits — e.g. `00:12` (bars remaining, formatted as `{bars}:{eighths}`)
- Workbench 9px UPPERCASE 0.28em "BARS TO DROP" label in `--ink-dim`

Chip pulses 1× per beat (BPM-synced via `--bpm-period-ms` CSS variable from WS).

When `drop_pred == 0`, chip flashes `--rec` once then unmounts.

### 8. Event Ribbon (center column — mock lines 1042-1098, `.event-ribbon`)

Horizontal scrolling chip strip below drop-chip / above mascot reserve.

Container:
- Padding: 10px
- Background: `--panel-deep`
- Border: 1px solid `--bezel-1`, 5px border-radius
- Box-shadow: `inset 0 2px 4px rgba(0,0,0,.6)`

Chips (`.evt` class — mock lines 1058-1086):

- Default: `--ink-engraved` background, `--ink-deep` text

Order: newest right, scrolls left as new events arrive. Max 12 visible — older chips trimmed.
