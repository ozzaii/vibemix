# 12-04: Live session state + rAF render loop + push-to-mute hotkey

## Goal

Glue the Wave 2 components to the Wave 2 sidecar IPC. Single SessionState singleton, single rAF render loop, hot-path mutations via CSS custom properties, push-to-mute system-wide hotkey through `tauri-plugin-global-shortcut`. Per `12-CONTEXT.md` §Area 1, §Area 3 and §"Performance budget".

## Steps

1. **`state.ts`** — `SessionState` shape:
   ```ts
   type SessionState = {
     meters: { music: {rms,peak}, voice: {rms,peak}, mic: {rms,peak} },
     phase: PhaseChunk[],
     phaseNowPct: number,
     bpm: number | null,
     bpmPeriodMs: number | null,
     dropPredBars: number | null,
     transcript: TranscriptLine[],
     midiEvents: MidiEvent[],
     track: TrackInfo | null,
     status: { livekit, gemini, midi, screen },
     settings: { voice, mode, genre, output_device_id, output_profile, retention_days, push_to_mute_hotkey },
     muted: boolean,
     cohostStatus: "LISTENING" | "TALKING" | "IDLE",
     latencyMs: number | null,
     grounded: boolean,
   };
   ```
   Plus `getSessionState()` / `setSessionState(patch)` — shallow merge.

2. **`ws-bridge.ts`**:
   - On boot: send `ipc.settings.get` → populate `state.settings`.
   - Subscribe to `ipc.session.snapshot` → merge into `state.meters`/`phase`/`bpm`/`dropPredBars`/`transcript` (append, capped at 200)/`midiEvents` (append, capped at 12)/`track`.
   - Subscribe to `ipc.status.tick` → write `state.status`.
   - Subscribe to `ipc.settings.state` → merge into `state.settings`.
   - Subscribe to `ipc.session.mute` (sidecar acks) → write `state.muted`.
   - Export `sendSettings(field, value)` / `sendMute(toggle)` helpers wrapping `sendIpcRequest`.

3. **`render-loop.ts`** — single rAF loop:
   - On each frame: read `SessionState`, call `renderSessionFrame(state)` from `SessionLayout`.
   - `renderSessionFrame` mutates CSS variables for hot paths (`--meter-music-rms`, `--meter-voice-rms`, `--meter-mic-rms`, `--phase-now-pct`, `--bpm-period-ms`, `--clock-text`), rebuilds the transcript / event-ribbon / phase-tape only when their array refs change.
   - Dev mode: track frame durations; if >33ms p95 over a 1s window, log warning to console.

4. **`SessionLayout.ts`** update:
   - Implement `renderSessionFrame(state)` as the diff path described.
   - Sticky-bottom transcript: track `userScrolledUp` flag (set true if user scrolled >40px from bottom); auto-scroll only when false.
   - Mute banner: mount/unmount based on `state.muted`.

5. **`router.ts`** — Phase 11 router gets `route.session()`:
   - Tears down wizard DOM (if mounted), mounts session DOM via `mountSessionLayout`.
   - Boot `initSessionBridge()` + `startRenderLoop()`.

6. **`main.ts`** — boot decision:
   - Read config via existing config IPC (Phase 11 first_run check pattern).
   - If `first_run_completed === false` → wizard. Else → session.
   - Preserve Phase 11 dev surface, sidecar-error / crash banner / status subscribers.

7. **Rust: push-to-mute** (`tauri/src-tauri/src/lib.rs`):
   - `Cargo.toml`: add `tauri-plugin-global-shortcut = "2.x"`.
   - `capabilities/default.json`: add `"core:event:default", "global-shortcut:allow-register", "global-shortcut:allow-unregister"`.
   - On startup: register default hotkey from config (`config.push_to_mute_hotkey`).
   - On shortcut press: forward `ipc.session.mute {toggle: true}` to sidecar via existing `forward_ipc_to_sidecar` command.
   - Expose `rebind_hotkey(new_combo: string)` Tauri command — called from webview when user rebinds.
   - On combo validation: reject macOS-reserved (`cmd+q`, `cmd+w`, `cmd+tab`, `cmd+space`) and Windows-reserved (`alt+f4`, `ctrl+alt+del`, `win+l`).

8. **Tests**:
   - `tests/session/state.spec.ts` — merge semantics, ring cap behaviour (transcript 200 / midi 12), shallow-merge ref preservation.
   - `tests/session/render-loop.spec.ts` — fake rAF; assert single loop, CSS var mutations match state, transcript scroll behaviour.

## Verification

- `cd tauri/ui && npm run typecheck && npm run test` green.
- Manual smoke: launch full app post-wizard, watch meters move under live audio, fire `⌘⇧M` → AI voice cuts mid-utterance (Phase 4 cascade in playback), `MUTED` banner appears, status bar shows `● MUTED · ⌘⇧M`. Press again → resume.
- Manual MIDI hot-unplug: unplug DDJ-FLX4 during playback — `● MIDI · 0` flips red within 2s.
- Dev console: no `[frame slow]` warnings during a 5-minute idle session.

## Notes

- Threat: `tauri-plugin-global-shortcut` shortcuts persist across window blur; we ONLY want them active while window has focus. Use the plugin's `with_handler` window-focus check (RESEARCH item — assume `window.is_focused()` is callable from handler).
- The render loop is the ONLY rAF — components do not start their own rAF/setInterval.
- Transcript scroll uses `behavior: 'auto'` (not 'smooth') for instant snap — smooth scroll fights the rAF loop.
- ZERO hardcoded hex still enforced — paper colours from UI-SPEC §Color/Paper Family are the only exceptions and stay scoped to phase-tape + transcript.
