# 12-03: Live session components (presentation layer)

## Goal

Build the pure-function presentation components for the live session UI per `12-UI-SPEC.md` §Component Contracts §§1-12 + §14 (muted banner). NO IPC wiring in this plan — components accept props and return HTMLElement. Wave 3 (plan 12-04) glues these into the live state pipeline.

## Steps

1. **`_style-registry.ts`** — copy/adapt from `tauri/ui/src/wizard/components/_style-registry.ts`. Same `registerStyle(name, css)` singleton pattern.

2. **Icon modules** — inline SVG TS modules:
   - `gear.svg.ts` — 18×18 8-tooth gear, stroke 1.5px currentColor.
   - `headphones.svg.ts` — 24×24 headphones glyph, stroke 1.5px currentColor.
   - `speakers.svg.ts` — 24×24 speaker glyph (cone + waves), stroke 1.5px currentColor.
   - `screw.svg.ts` — 8×8 cross-slot screw glyph for shell corners.
   - `mascot-placeholder.svg.ts` — 32×32 stylized headphones for the cohost-header mascot circle (Phase 13 replaces).

3. **`panel.ts`** — generic panel shell helper: returns `<div class="vmx-panel">` with brushed-metal `::before` overlay + `--panel-lift → --panel` gradient + 1px `--bezel-1` border + inset shadows + optional header strip (Workbench 9px label + optional pill badge).

4. **`titlebar.ts`** — `renderTitlebar({live, rec, sys, clock, onSettingsClick})`:
   - Traffic-light placeholders left.
   - "vibemix" Workbench 28px wordmark with `--phosphor-glow`.
   - 3 status pills (LIVE / REC / SYS) — each with 6×6 LED.
   - DSEG7 22px live clock.
   - Gear button right (Settings) — fires `onSettingsClick`.

5. **`rocker.ts`** — generic segmented rocker switch:
   - `renderRocker({options, active, onChange, variant: "rocker" | "interaction"})`.
   - `rocker` variant: simple segmented (BEG/INT/PRO).
   - `interaction` variant: LED-prefixed buttons (HYPE/COACH).

6. **`picker.ts`** — dropdown row component:
   - `renderPicker({icon, label, value, options, autoPill, onChange})`.
   - 48px row, opens 240px dropdown with hover `--phosphor-soft`.

7. **`meter.ts`** — vertical 16-segment LED meter:
   - `renderMeter({label, palette})` returns element. Caller sets `--meter-{label}-rms: <0..1>` on the element to update.
   - Each segment is a div with `background: var(--phosphor)` lit / `--ink-engraved` unlit.
   - CSS uses `:nth-child(N) { opacity: var(--meter-X-rms) > 0.{threshold} ? 1 : 0.15 }`-equivalent via CSS calc + custom property.
   - Peak-hold dot via separate CSS variable `--meter-{name}-peak`.

8. **`timecode.ts`** — center column timecode block:
   - `renderTimecode({clock, bpm, key, deck})` — DSEG7 88px hero + meta row.

9. **`phase-tape.ts`** — paper-textured phase timeline:
   - `renderPhaseTape({chunks: PhaseChunk[], nowPct: number})`.
   - Chunks `{kind: "silent"|"groove"|"build"|"drop-ghost", weight, label}`.
   - Paper-grain SVG fractalNoise CSS variable.
   - Drop-ghost chunks use Caveat italic label.

10. **`drop-chip.ts`** — beat-pulsed drop countdown:
    - `renderDropChip({bars: number, bpmPeriodMs: number})`.
    - Hides when `bars == null`. Flashes `--rec` on `bars == 0`.

11. **`event-ribbon.ts`** — MIDI event chip strip:
    - `renderEventRibbon({events: MidiEvent[]})` — max 12 visible, newest right.
    - Per-chip CSS class swap on 4s / 12s for `--phosphor` → `--phosphor-warm` → `--ink-deep` fade.

12. **`cohost.ts`** — full right column:
    - `renderCohostPanel({mascotState, transcript: TranscriptLine[], latencyMs, grounded})`.
    - Mascot 42×42 circle header with name "AVERY" + LED status.
    - Receipt-paper transcript with `.msg` per line + `.now` last + `.faded` next 5 + `.old` rest.
    - Sticky-bottom scroll behaviour.
    - Foot: `● {grounded ? "GROUNDED ON AUDIO + SCREEN" : "WARMING UP"}` + DSEG7 latency.

13. **`status-bar.ts`** — bottom live status bar:
    - `renderStatusBar({livekit, gemini, midi, screen, muted, hotkey})`.
    - 4 LED badges + muted indicator + Caveat "made by bravoh" signature.
    - Click handler on each badge: opens `--rec`-tinted tooltip with `[ ↻ Recheck ]` button when in `down` state.

14. **`muted-banner.ts`** — mute UX banner:
    - `renderMutedBanner({hotkey})` — `--rec`-tinted strip pinned above transcript.

15. **`SessionLayout.ts`** — composer:
    - `mountSessionLayout(root: HTMLElement)` — builds full DOM tree (titlebar + 3-col grid + status bar + panel screws + body atmospheric overlays carried from Phase 11 tokens.css).
    - `renderSessionFrame(state: SessionState)` — diffs targeted regions via CSS variable mutations + child-list updates. Initial mount + idempotent re-render.

16. **Tests** — `tests/session/components.spec.ts` (vitest):
    - Meter: rms=0.5 → segments 1-8 lit; rms=1.0 → all 16 lit + peak segment lit.
    - PhaseTape: 3 chunks render in correct order with correct flex weights; nowPct positions NOW marker.
    - DropChip: bars=null → returns null; bars=8 → renders; bars=0 → renders with rec-flash class.
    - EventRibbon: 15 events input → 12 rendered (newest 12, oldest 3 trimmed).
    - Cohost: 200 lines input → last 200 rendered (no overflow); last line has `.now` class; lines 6-10 have `.faded`.
    - StatusBar: livekit=down → click opens tooltip with Recheck button.
