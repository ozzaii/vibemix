# 12-01: IPC schema for session + settings messages

## Goal

Lock the cross-boundary contract for Phase 12 before any UI or sidecar code lands. Extends Phase 11's `messages.schema.json` with the session/settings/status-recheck message families per `12-UI-SPEC.md` §"WS Message Schema & Hot-Reload Contract" and §12 "Status Badges".

## Steps

1. Extend `tauri/ui/src/ipc/messages.schema.json` `oneOf` array + `definitions` with:
   - `IpcSessionSnapshot` (sidecar → shell, 30Hz): payload as defined in must_haves.
   - `IpcSessionMute` (bidirectional): `{toggle: bool, muted: bool}` — `toggle` for command, `muted` for state ack.
   - `IpcSettingsSet` (shell → sidecar): `{field: enum, value: <field-typed>}` discriminated by `field`.
   - `IpcSettingsGet` (shell → sidecar, no payload).
   - `IpcSettingsState` (sidecar → shell): full settings snapshot `{voice, mode, genre, output_device_id, output_profile, retention_days, push_to_mute_hotkey, muted}`.
   - `IpcStatusRecheck` (shell → sidecar): `{component: enum livekit|gemini|midi|screen}`.
2. Mirror in `src/vibemix/ui_bus/messages.py` — dataclasses with `from_dict` / `to_dict` and JSON-Schema-aligned defaults.
3. Mirror in `tauri/ui/src/ipc/messages.ts` — discriminated unions tagged on `type`.
4. Extend `tauri/ui/src/ipc/validator.ts` ajv compile list.
5. Add round-trip tests in `tauri/ui/src/ipc/validator.spec.ts` (green-path + reject-on-extra-field for each new type).
6. Add `tests/ipc/test_session_messages.py` — Python jsonschema round-trip per dataclass.
7. Re-run `scripts/check_ipc_schema.py` — must stay green.

## Verification

- `python scripts/check_ipc_schema.py` passes (count parity + per-type roundtrip).
- `cd tauri/ui && npm run typecheck && npm run check:ipc` passes.
- `pytest tests/ipc/test_session_messages.py -q` passes.
