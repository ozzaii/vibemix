# Phase 11 — UI Design Contract

> Visual and interaction contract for the Tauri shell + 3-step calibration wizard. Generated by gsd-ui-researcher; consumed by gsd-planner, gsd-executor, gsd-ui-checker, gsd-ui-auditor.

**Visual identity rule (load-bearing):** the wizard MUST feel like `mocks/vibemix-app-ui.html` is loading — same anodised charcoal, same phosphor amber, same DSEG7 numerals, same WORKBENCH wordmark, same film grain + scanlines. NOT a "wizard skin" that diverges from the live session aesthetic. Every token below is lifted verbatim from the mock so the visual handoff into Phase 12 is seamless.

## Typography

Type roles (3 sizes, 2 weights per role — strict; refused to exceed):

| Role | Family | Size | Weight | Line Height | Letter Spacing | Usage |
|------|--------|------|--------|-------------|----------------|-------|
| Wordmark | Workbench | 28px | 400 (only weight ships) | 1.0 | 0.04em | "vibemix" titlebar wordmark, glows with `--phosphor-glow` |
| Step Heading (H1) | Workbench | 22px | 400 | 1.1 | 0.04em | "STEP 1 / 3 — PERMISSIONS" |
| Section Label | Workbench | 9px | 400 | 1.0 | 0.32em (UPPERCASE) | Panel headers ("AUDIO IN", "MIDI", "OUTPUT DEVICE") |
| Body | DM Mono | 14px | 400 | 1.5 | 0.01em | Wizard prose, descriptions, microcopy |
| Label | DM Mono | 11px | 400 | 1.35 | 0.18em (UPPERCASE) | Form field labels, pill text, status pills |
| Numeric / LCD | DSEG7-Classic | 22px (small) / 48px (hero readout) | 400 | 1.0 | 0.06em | 1kHz, sample-rate values, 10s controller countdown |
| Sticker | Caveat | 14px | 700 | 1.1 | 0.02em | Reserved for Phase 12 stickers — wizard MUST NOT introduce stickers (would break "step focus") |
| Button (display) | Workbench | 11px | 400 | 1.0 | 0.22em (UPPERCASE) | Primary CTA buttons, rocker buttons |

**Two weight rule:** every typeface ships at most two weights. Workbench is single-weight. DM Mono uses 400 (body/label) + 500 (selected option / picked device name only — like mock `.voice-pick .name`).

**Display ↔ body pairing rationale (documented per `frontend-enforcement` rule 5):** Workbench's chunky LED-display character pairs with DM Mono's industrial monospace for the "Pioneer hardware front-panel" feel. DSEG7 isolates numerics so they read as "instrument readouts" not prose. Caveat sticker is the one breath of humanity. No Inter, no Roboto, no system-ui, no Arial anywhere.

### Accent reserved-for list (10% rule enforcement)

Phosphor amber is ALLOWED on:

1. The "vibemix" wordmark in the titlebar (text + glow)
2. The currently-active step indicator (LED + numeral)
3. The primary "Continue" / "Recheck" CTA button (lit fill + glow when armed)
4. DSEG7 numeric readouts (1kHz, sample rate, 10s countdown)
5. LED dots in status pills when state = ok
6. Active rocker/dropdown selection (lit border + soft fill)
7. Audio-test sine-wave "playing" indicator (animated ring around speaker icon)
8. Mascot wordmark in placeholder corner (text only, no full-fill)

Phosphor is FORBIDDEN on:
- Step descriptions, body copy, microcopy
- Inactive buttons, secondary CTAs
- Background panels (max allowed: `--phosphor-soft` for tinted overlays, no solid fills)
- Section dividers, bezel rings
- Mascot placeholder rectangle (Phase 11 = empty `--ink-engraved` outline only)

Destructive color (`--rec`) reserved for:
- "Skip — use generic mapping" button (NOT destructive but action-of-last-resort; warm-warning tone)
- "Permission denied" inline error
- "1kHz test failed — recheck" inline error
- "Sidecar stopped — restart?" crash banner

No second semantic color in Phase 11. No "warning yellow", no "info blue".

## Interaction Contracts

### Focus Order (keyboard accessibility)

Wizard MUST be fully keyboard-traversable (tab/shift-tab/enter/escape). Tab order per step:

**Step 1 (Permissions):** Permission card 1 → [Grant 1] → Permission card 2 (if macOS) → [Grant 2] → [Continue].
**Step 2:** [Open install page] (if BlackHole missing) → [Recheck] → Audio test button → [Yes] → [Retry] → Device dropdown → Window picker → [Continue].
**Step 3:** [Listen again] → [Skip — generic mapping] → [Continue] (after caught/timeout).
**Smoke test:** [Replay] → [Open vibemix].

Focus ring: 2px solid `--phosphor` with `--phosphor-glow`, offset 2px from element. Visible on `:focus-visible` only (no focus ring on mouse click).

`Esc` key behavior: noop in Step 1-3 (no escape during calibration; force-quit only via window close button which triggers confirm dialog).

### Motion Budget

Per CONTEXT 90-second constraint and `frontend-enforcement` rule 6:

| Element | Duration | Easing | Rationale |
|---------|----------|--------|-----------|
| Button state transitions | 150ms | ease-out | Snap, not float |
