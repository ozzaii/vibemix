   ```python
   from __future__ import annotations
   import argparse, os, platform, shutil, subprocess, sys
   from pathlib import Path

   def detect_target_triple() -> str:
       """Returns the Rust target triple by invoking `rustc -vV` and parsing `host: <triple>`.
       Raises RuntimeError if rustc is not on PATH (caller must install it)."""

   def run_pyinstaller(spec: Path, *, clean: bool = True) -> Path:
       """Invokes `uv run pyinstaller <spec> --clean --noconfirm --distpath dist`.
       Returns the resulting dist/vibemix-core directory. Raises CalledProcessError on failure."""

   def install_into_tauri_binaries(onedir: Path, triple: str, *, exe_suffix: str) -> Path:
       """Moves dist/vibemix-core/ → tauri/src-tauri/binaries/vibemix-core-{triple}/.
       Renames the inner binary to vibemix-core-{triple}{exe_suffix}.
       Returns the path to the renamed binary."""

   def assert_no_aiza_leak(bundle_dir: Path) -> None:
       """CI gate: runs `strings` recursively over bundle_dir and greps for AIza[A-Za-z0-9_-]{35}.
       Raises RuntimeError if any match found. macOS has `strings` builtin; Windows may need a fallback
       using `pathlib.Path.read_bytes()` + regex scan over .pyc/.so/.dylib files."""

   def main() -> int:
       parser = argparse.ArgumentParser()
       parser.add_argument("--spec", required=True, help="Path to .spec file (vibemix.spec.macos | vibemix.spec.windows)")
       parser.add_argument("--triple", default=None, help="Override target triple (default: detect from rustc)")
       parser.add_argument("--no-aiza-check", action="store_true", help="Skip the AIza key-leak gate (dangerous; CI must NOT pass this)")
       # ... orchestrates the four steps above and prints the final install path.
   ```

   Implementation specifics:
   - `detect_target_triple`: `subprocess.check_output(["rustc", "-vV"], text=True).split("host: ")[1].split("\n")[0].strip()`. If rustc absent, raise with actionable message ("Install Rust via https://rustup.rs/ to build the Tauri sidecar").
   - `run_pyinstaller`: `subprocess.run(["uv", "run", "pyinstaller", str(spec), "--clean", "--noconfirm", "--distpath", "dist"], check=True)`. Returns `Path("dist/vibemix-core")`.
   - `install_into_tauri_binaries`: target dir `Path("tauri/src-tauri/binaries") / f"vibemix-core-{triple}"`. If exists, `shutil.rmtree`. `shutil.copytree(onedir, target_dir)`. Inside target_dir, rename `vibemix-core{exe_suffix}` → `vibemix-core-{triple}{exe_suffix}`. Return the renamed path. **Sanity check:** assert renamed binary exists and is executable (`os.access(..., os.X_OK)` on POSIX; check stat on Windows).
   - `assert_no_aiza_leak`: On macOS/Linux, use `subprocess.run(["strings", "-a", str(p)], capture_output=True, text=True)` for each `.so/.dylib/.bin` + the main exe, run `re.findall(r"AIza[A-Za-z0-9_-]{35}", out)`. On Windows where `strings` is absent: fall back to reading file bytes in 4MB chunks and `re.findall(rb"AIza[A-Za-z0-9_-]{35}", chunk)`. If any matches: raise `RuntimeError(f"API KEY LEAK: found AIza-pattern in {p} — bundle is unsafe to ship.")`.
   - `main` returns 0 on success, 1 on failure. Prints the final path (`tauri/src-tauri/binaries/vibemix-core-{triple}/vibemix-core-{triple}{exe_suffix}`) so the executor can verify visually.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Source tree → PyInstaller bundle | Anything in src/ + collect_data_files can land in the shipping binary; `.env` files are the highest-risk surface |
| PyInstaller bundle → shipped artifact | Once built, the bundle is what users run; strings extraction is trivial (RESEARCH Pitfall 5) |
| rustc subprocess → triple detection | If a hostile cargo workspace overrides the host triple, the binary lands in the wrong externalBin slot — but build host is the developer machine; trust is implicit |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-W1-01 | Information Disclosure | `.env` accidentally bundled by `collect_data_files('vibemix', includes=['**/*'])` | mitigate | Explicit `excludes=['*.env', '.env*', '*credentials*', '*.key', '*.pem']` in both spec files + `assert_no_aiza_leak()` CI gate runs on every build (RESEARCH Pitfall 5) |
| T-11-W1-02 | Tampering | `--upx` flag or `--onefile` flag accidentally added later — triggers AV false positives | mitigate | Both spec files lock `upx=False` + `--onedir`; plan-checker rejects future PRs that flip these (RESEARCH Pitfall 1) |
| T-11-W1-03 | Spoofing | Sidecar binary impersonated by another process listening on 127.0.0.1:8765 | accept | Localhost binding is single-user; Phase 18 codesigns the bundle so user knows it's the real binary |
| T-11-W1-04 | Denial of Service | PyInstaller hidden_imports miss → sidecar crashes on first audio call → blank wizard (RESEARCH Pitfall 6) | mitigate | Spec uses `collect_submodules` + `collect_all` for every dynamic-import-heavy package per Pitfall 2; Wave 2 watchdog catches crashes; Wave 4 ipc.* timeouts catch hangs |
| T-11-W1-05 | Information Disclosure | Sidecar log accidentally captures `GEMINI_API_KEY` from env at process start | mitigate | sidecar.log capture is stdout/stderr only; `__main__.py` does NOT print env vars; assert in Wave 2 log-capture test |
</threat_model>
