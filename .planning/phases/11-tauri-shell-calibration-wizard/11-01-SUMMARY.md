## Decisions Made

- **Single source-of-truth = schema JSON file**, not the dataclasses or the TS types. Both sides validate against the same file at build time and runtime.
- **Codegen output committed alongside schema** — RESEARCH alternatives-considered. Downstream Tauri builds don't depend on Node being available; CI regenerates and diffs.
- **Two CI gates, not one** — Python-side (`check_ipc_schema.py`) catches schema-vs-wrapper drift; TS-side (`npm run check:ipc`) catches schema-vs-codegen drift. Either failing exits non-zero.
- **Count-parity assertion via `__dataclass_fields__` introspection** — survives wrapper renames; finds "wrapper added without schema entry" and "schema entry added without wrapper" symmetrically.
- **Wrappers are frozen + slots = hashable** — payload arrays are `tuple[T, ...]`; recursive `_tuples_to_lists` converts before jsonschema validate because Draft-07's type checker rejects tuples for `type: "array"`. JSON itself has no tuple — fine after `json.dumps`.
- **ajv `strict: false`** — schema uses `$comment` (Draft-07 supported, not in ajv strict whitelist) and oneOf-with-const discriminator (rather than ajv discriminator extension).

**1. [Rule 3 — Blocking] json2ts CLI binary name + no banner-comment option**
- **Found during:** Task 2 (codegen scripting)
- **Issue:** Plan's `codegen:ipc` invocation `json-schema-to-typescript` is the package name, not the CLI name (`json2ts`). Then `json2ts --help` confirmed `--bannerComment` is not exposed in CLI mode (only programmatic API).

- **Committed in:** `11f3eb7` (Task 2 commit).

**2. [Rule 1 — Bug] dataclasses.asdict preserves tuples; jsonschema rejects tuples for type:array**
- **Found during:** Task 1 (first roundtrip smoke test)
- **Issue:** `CalibrationWindowList.make(windows=[...]).to_json()` raised `ValidationError: (...,) is not of type 'array'`. Root cause: payload structs use `tuple[T, ...]` for hashability, but `dataclasses.asdict` recursively preserves tuples, and jsonschema's Draft-07 `type` checker rejects tuples (only accepts `list`).
- **Fix:** Added `_tuples_to_lists(value)` helper that recursively converts tuple/list/dict trees to list-only. New shared `_serialize(self)` body runs `_tuples_to_lists(asdict(self))` before validate + dumps. All 19 wrapper `to_json` methods reduced to a single call.
- **Files modified:** `src/vibemix/ui_bus/messages.py`.
- **Verification:** 35 tests pass; `scripts/check_ipc_schema.py` exits 0.
- **Committed in:** `4f1d879` (Task 1 commit).

**3. [Rule 1 — Bug] Generated TS union type name mismatch**
- **Found during:** Task 2 (after first codegen run)
- **Issue:** json2ts PascalCases the schema `$id` — `vibemix.ipc.messages` becomes `VibemixIPCMessages`, not `VibemixIpcMessages` as the validator.ts initial import expected. tsc would fail.
- **Fix:** Updated `validator.ts` import to `VibemixIPCMessages as IpcMessage`.
- **Files modified:** `tauri/ui/src/ipc/validator.ts`.
- **Verification:** `npm run check:ipc` (codegen + tsc --noEmit) exits 0.
