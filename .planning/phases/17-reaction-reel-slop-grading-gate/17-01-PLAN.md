Locked schema for per-reaction grade record (Plan 17-02's grade.py writes one
JSON object per line into recordings/<session>/grades/<rater>.jsonl):

- reaction_id: string (SHA8 of voice.wav clip filename, anonymized)
- score: integer in [1, 2, 3, 4, 5]
- rater: string in ["kaan", "francesco", "dj1", "dj2"]
- grounded: bool
- timely: bool
- unique: bool
- personality_fit: bool
- slop_flag: enum string in ["none", "late", "generic", "hallucination", "repetition", "cringe"]
- comment: string (free text, may be empty)
- would_clip: bool

Pass thresholds (from ROADMAP success criterion #3 verbatim):
- Average score ≥ 4.0 across (all reactions × all raters)
- ZERO ratings of 1 or 2 from any rater on any reaction

Tie-breaker:
- If avg == 4.00 ± 0.05 AND any 3-score appears in >25% of reactions → escalate to Kaan.

Anti-slop dictionary already exists at src/vibemix/prompts/negative_dict.py.
The rubric MUST reference (not duplicate) NEGATIVE_PHRASES for the slop_flag=generic
detection guidance. Importable as:
    from vibemix.prompts import NEGATIVE_PHRASES, NEGATIVE_REGEX
