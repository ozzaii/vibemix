Verdict logic (compute_verdict):
    Inputs: list of all grade records across all raters.
    PASS iff:
      - len(records) > 0
      - All raters expected (raters_present == 4) — if fewer, verdict is "INCOMPLETE" not PASS/FAIL.
      - All reactions have been graded by all present raters — if any reaction missing a rater, verdict is "INCOMPLETE".
      - avg(score for record in records) >= 4.0
      - count(score in {1, 2}) == 0
    FAIL iff:
      - avg < 4.0 OR count(score in {1, 2}) > 0
    INCOMPLETE iff:
      - Missing raters OR partial grading (some reactions un-graded by some present raters).
    Tie-breaker case (per Plan 17-01 rubric):
      - If 4.00 - 0.05 <= avg <= 4.00 + 0.05 AND count(score==3) / count(records) > 0.25:
        verdict = "PASS_TIE_BREAKER_NEEDED" — flags Kaan's manual ship-vs-iterate decision.

Aggregation entry points:

    - `load_grades_key(grades_dir: Path) -> dict`: same as Plan 17-02's loader; returns sha8 → reaction-record map.
    - `compute_verdict(records: list[dict], expected_raters: int = 4) -> tuple[str, dict]`: returns (verdict_string, metrics_dict).

Exit codes (operational signal for CI / shell scripts):
    0 = PASS
    1 = FAIL
    2 = INCOMPLETE
    3 = PASS_TIE_BREAKER_NEEDED
