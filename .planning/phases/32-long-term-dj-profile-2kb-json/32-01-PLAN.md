## Builder API

```python
def build_profile(
    prior_profile: dict | None,
    session_events: list[Event],
    evidence_snapshot: dict[str, dict[str, tuple[float, ...]]],
    *,
    consent: bool = False,
) -> dict | None:
    """Regenerate profile from session evidence + prior profile.

    Returns None if consent is False OR if no field can be regenerated AND
    no prior profile exists. Each tendency field requires ≥2 distinct
    EvidenceRegistry citations to (re)generate; otherwise retains prior
    value (PROFILE-06 drift prevention).
    """
```

Aggregation rules per field:
- `preferred_genre`: majority vote across event-types tagged with genre hints in evidence_snapshot["genre"]. ≥2 citations required (or retain prior).
- `avg_session_duration`: arithmetic mean of session durations from `session_events[-1].state.session_t` (clamped 0-720).
- `mix_style_tags`: tags emitted by EventDetector that survived; intersect with allowlist; cap 8.
- `tempo_preference_bin`: histogram of `session_events[*].state.bpm` → modal bin. ≥2 distinct BPM observations required.
- `event_type_response_preferences`: per event type, count distinct citations from `evidence_snapshot["event"][type]`. Map: 0 → "never", 1-2 → "rarely", 3-5 → "sometimes", ≥6 → "always". Field only regenerated if total citations ≥2 (PROFILE-06).

Serializer:
```python
def serialize_profile(profile: dict) -> bytes:
    validate_profile(profile)
    raw = json.dumps(profile, separators=(",", ":"), sort_keys=True).encode("utf-8")
    if len(raw) > MAX_PROFILE_BYTES:
        raise ProfileError(f"profile exceeds {MAX_PROFILE_BYTES}-byte cap (got {len(raw)})")
    return raw
```

def render_profile_for_cache(profile: dict | None) -> str:
    """Compact flat-key form for cache injection. Empty string if profile is None.

    Sample output:
        '\\n\\n# DJ profile (allowlist v1)\\n- preferred_genre: hard_tek\\n- ...'

    NEVER includes track titles, artist names, library hashes."""

## Tests (Gate)

1. `test_profile_size_cap_2048_bytes`: builder output ≤ 2048 UTF-8 bytes; oversize synthetic profile → `ProfileError`.
2. `test_profile_additional_properties_false`: schema rejects unknown field (e.g. `recent_tracks`).
3. `test_no_track_titles_in_profile`: grep-style `re.search(r"\\b[A-Z][a-z]+ - [A-Z]", serialized)` returns None on 100-track synthetic library.
4. `test_tendency_requires_2_citations`: synthetic session with 1 citation → tendency retains prior (or null if new).
5. `test_consent_off_returns_none`: `build_profile(..., consent=False)` returns None.
6. `test_cache_render_token_budget`: rendered profile char-len // 4 ≤ 300.
