<objective>
Wire the components from Plan 15-04 into the drawer + connect them to the IPC families from Plan 15-01 + listen for the pushes from Plan 15-03. This is the integration layer — no new components, no new IPC, just glue.

Three deliverables:

1. **SettingsDrawer.ts RECORDING group extension:** append the disk usage line + recording browser as `recordingBody` children, below the existing `renderRetentionSlider` mount. Hook drawer open lifecycle to fire `sendIpcRequest('recordings.list', {})` with a 10s timeout. Render loading + error states per UI-SPEC §Copywriting.

2. **State slice + subscriber wiring:** add a `RecordingsSlice` to the existing settings UI state singleton in `tauri/ui/src/settings/state.ts` (use Phase 12's `SettingsUIState` extension pattern). Subscribe to `ipc.recordings.usage` pushes via `subscribeIpc(...)` from `client.ts` — bus router lives in `tauri/ui/src/session/ws-bridge.ts` per Phase 12; mirror that file's existing `ipc.settings.state` subscriber.

3. **Delete optimistic update:** delete callback fires `sendIpcRequest('recordings.delete', { session_dir })`, on `ok=true` removes the row from the local slice without waiting for a re-list; on `ok=false` the confirm dialog body swaps to show the error.

Purpose: closes the user-facing surface for REC-05 + REC-06. After this plan + Plan 15-06 (tests), Phase 15 is feature-complete pending Kaan's visual review.
