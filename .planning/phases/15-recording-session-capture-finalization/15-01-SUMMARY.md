## Open-Extensibility Design Note: events_result.events

The `recordings.events_result.payload.events` array has `additionalProperties: true` on its per-event element schema â€” deliberately NOT enumerated. Why:

- POC `events.jsonl` (from `cohost_v4.py:771-850`) records ~10 kinds today (`session_start`, `trigger`, `trigger_fired`, `ai_text`, `controller_move`, `midi_event`, `turn_error`, `session_error`, plus mood-change additions from Phase 13-05).
- Per-kind extras differ widely: `ai_text` has `text`; `trigger` has `reason`; `controller_move` has `control` + `value` + `channel`; `midi_event` has the raw bytes string; `turn_error` / `session_error` have `error` + `traceback`.
- Phase 16 (Hallucination Verification Gate) will add verifier-side records (`grounding_score`, `evidence_keys`, etc.).

Enumerating every kind in the schema would force a schema bump on every new event class â€” and the consumer (the row-expanded transcript overlay in Plan 15-04) renders events with a per-kind formatter that already tolerates unknown fields. The schema requires `t` (non-negative number) + `kind` (string) as the only invariants; everything else is free-form per the kind-specific contract.

This matches Phase 13-05's `mascot.mood_change` precedent where `previous_mood` / `at` are optional + nullable to keep emit paths free.

## V12 Path-Traversal Gate Coverage

The `^[0-9]{8}-[0-9]{6}$` regex is enforced on `session_dir` at the schema layer in 5 places:

1. `RecordingsDelete.payload.session_dir`
2. `RecordingsDeleteAck.payload.session_dir`
3. `RecordingsEvents.payload.session_dir`
4. `RecordingsEventsResult.payload.session_dir`
5. Every `RecordingSummary` element inside `RecordingsListResult.payload.sessions[]`

This means any frame carrying `../../etc/passwd`, `..\\Windows\\System32`, `/tmp/anything`, or any string that doesn't match the basename pattern is rejected before reaching Python `shutil.rmtree` (Plan 15-03 handler) or Tauri's `recording://` asset-protocol resolver (Plan 15-04 consumer). Both Python (`jsonschema`) and TS (`ajv`) sides enforce identically.
