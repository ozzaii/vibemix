# Phase 15 — UI Design Contract

> **Narrow scope.** Phase 15 adds a Recording Browser UI **inside the existing Phase 12 Settings drawer `RECORDING` group**, directly below the existing 6-stop retention slider. No new windows, no new routes, no new design tokens. Every visual is a re-use of the CDJ Whisper v5 contract (Phase 14) and the drawer pattern (Phase 12). CONTEXT.md locked the row layout, copywriting, expansion behavior, delete confirm, and empty state — this spec formalizes those into the design contract.

> Generated by gsd-ui-researcher. Verified by gsd-ui-checker; final approval by gsd-ui-auditor against the live drawer.

## Glass + Glow + Atmospherics

Phase 15 inherits — does not modify — the Phase 14 glass anatomy. Notes for the executor:

- The Recording Browser list **does NOT introduce a new glass surface.** It is content rendered INSIDE the drawer's existing `--glass-1` body. Rows themselves are NOT glass panels — they are `--glass-2` fills with `--glass-edge` 1px dividers.
- **No `.border-anim` instance inside the recording browser.** The drawer's existing border-sweep (Phase 14 Surface 3, `--motion-border-sweep`) is the single sweep for this surface. Nesting would violate the "one sweep per visible region" rule.
- **No `.vmx-glass-streak` inside rows.** Streaks are reserved for primary panel surfaces; recording rows are content-level.
- **No backdrop-filter on rows.** The drawer body's `var(--blur-glass)` already provides the glass effect; rows would re-blur and stack costs.
- **Expanded panel uses `--glass-3` recessed feel** (matching the Phase 14 display-window recipe) WITHOUT its own backdrop-filter — semantic depth via color + inset shadow only.

### `prefers-reduced-motion`

The row expand/collapse height transition (250ms ease-out per CONTEXT.md "no animation beyond a height transition") MUST honor the Phase 14 reduced-motion override:

```css
@media (prefers-reduced-motion: reduce) {
  .vmx-rec-row__expand { transition: none; }
  .vmx-rec-row__expand[data-open="false"] { display: none; }
  .vmx-rec-row__expand[data-open="true"] { display: block; }
}
```

The animation is height-only — no opacity sweep, no color shift, no transform. Cheap on every GPU.

**Order inside the RECORDING group (top to bottom):**

2. `renderRetentionSlider(...)` (existing) — UNCHANGED

4. **NEW: Recording Browser** — virtualized list of rows OR empty state

### Row anatomy (44px, three cells, action cluster)

```
┌─ Row (44px min-height, glass-2 bg) ──────────────────────────────────────┐
│  [TS mono 12px silk-65]    [Body 14px silk]               [▶] [🗑]      │
│   2026-05-13 21:04         1h 24m · 38 events             icon icon      │
│   ← left cell, 140px       ← center cell, flex-grow       ← right, 64px  │
└──────────────────────────────────────────────────────────────────────────┘
```

- **Left cell:** timestamp in JetBrains Mono 12px wght 500, tabular-nums, `--silk-65`. Format `YYYY-MM-DD HH:MM` (matches dir name format, no seconds in the UI for density). Fixed width 140px.
- **Center cell:** flex-grow. `{duration_human} · {event_count} events`. Saira 14px wght 400 `--silk` body color. Middle dot character U+00B7 separator (matches Phase 14 mock convention). Duration format:
  - `< 1h`: `{M}m` (e.g. `48m`)
  - `>= 1h`: `{H}h {MM}m` (e.g. `1h 24m`)
- **Right cell:** action cluster — 64px wide, flex row, `gap: var(--sp-2)`, right-aligned.
  - **Replay button** (▶): 24×24 icon-only button, transparent background, idle stroke `--silk-65`, hover stroke `--amber` with `--glow-faint` text-shadow. Toggles row expansion (CONTEXT Area 2: "Click row → expand inline").
  - **Delete button** (🗑): 24×24 icon-only button, transparent background, idle stroke `--silk-40`, hover stroke `var(--rec)` with `box-shadow: inset 0 0 8px rgba(212,65,58,0.18)`. Opens the Phase 12 `confirmDialog`.

**Row click target:** the whole row (excluding the right action cluster) toggles expansion. Action cluster events stop propagation so clicking ▶ explicitly expands and clicking 🗑 explicitly opens the confirm — neither stacks two actions in one click.

**Row hover state:** `background: rgba(214,207,199,0.06)` (inline, derived from `--silk-22` at lower alpha — matches Phase 12 row hover density). Transition: `background var(--motion-snap) ease-out`.

**Row expanded state:** `data-open="true"` on the row root. Inline child `.vmx-rec-row__expand` flips from `height: 0` to `height: auto` via a height-transition. Inside the expanded panel:

- **Audio player** — native HTML5 `<audio controls preload="metadata" src="recording://{session_dir}/voice.wav">` with `accent-color: var(--amber)` and `width: 100%`. The `recording://` custom protocol is registered in `tauri.conf.json` per CONTEXT Area 2 (scope: recordings dir per OS).
- **Transcript overlay** — static (not synced; sync deferred to v2 per CONTEXT). Lines rendered from `events.jsonl`:
  - **AI text + trigger reason events** → `<div class="vmx-rec-evt vmx-rec-evt--bold">`, `border-left: 1px solid var(--amber-22)`, `padding-left: var(--sp-3)`, `color: var(--silk)`, line spacing 1.45.
  - **Controller moves + other ambient events** → `<div class="vmx-rec-evt vmx-rec-evt--dim">`, no border, `color: var(--silk-40)`, same line spacing.
  - Each line prefixed with relative timestamp `[+M:SS]` in JetBrains Mono 11px `--silk-40` `tabular-nums`.
- **Expanded panel background** `--glass-3` with inset shadow `inset 0 1px 0 var(--glass-top)` and `border-top: 1px solid var(--amber-22)` (the only amber edge in this surface — signals "open").
- Padding `var(--sp-4)` on the expanded panel inner; max-height `40vh` with `overflow-y: auto` for long transcripts.

### Disk usage line

Single row above the recording browser list. NOT a row of the list — it's a header for the list.

- **Font:** Saira 9–10px wght 500 wdth 85, `letter-spacing: 0.22em`, UPPERCASE, color `--silk-40`, engraved text-shadow.
- **Format:** `RECORDINGS · {N} SESSIONS · {SIZE_DISPLAY}` where `{SIZE_DISPLAY}` is human-readable bytes:
  - `< 1 GB`: `{N} MB USED`
  - `>= 1 GB`: `{N.N} GB USED` (one decimal, e.g. `3.4 GB USED`)
- **Updates:** on drawer open + on `recordings.usage` push from the sidecar.
- **Bullet separators:** U+00B7 middle dot, identical to Phase 14 cross-surface convention.

### Empty state

When `recordings.length === 0`, the browser renders a single centered block instead of the list:

```
┌─────────────────────────────────────┐
│                                     │
│   No recordings yet. Sessions       │  ← Saira 14px wght 400 silk-40, centered
│   appear here after they end.       │
│                                     │
└─────────────────────────────────────┘
```

- **Container:** padding `var(--sp-4) var(--sp-3)`, `text-align: center`, `color: var(--silk-40)`.
- **No icon, no illustration, no CTA.** Pioneer-grade restraint.
- **Disk usage line still renders** above, with `RECORDINGS · 0 SESSIONS · 0 MB USED`.

### Virtualization

Per CONTEXT.md ("virtualized list of rows; lazy-rendered rows; only mounts when the drawer opens"):

- Rows only render when within the visible viewport of the drawer body's overflow region.
- Implementation freedom: simple IntersectionObserver on a 12-row chunk window is acceptable; full `tanstack/virtual`-style libraries are NOT introduced (no new third-party UI deps).
- Initial list mount: defer until the drawer opens at least once. Subsequent re-opens re-use the mounted list (refresh by replaying `recordings.list` on each open per CONTEXT).
- When the drawer is closed, no `<audio>` elements are alive — `data-open="false"` rows have their `<audio>` torn down on collapse to free decoder resources. Re-expand re-creates.

## Component Contracts

### `renderRecordingBrowser(opts)` — new module

```ts
// tauri/ui/src/settings/components/recording-browser.ts

interface RecordingSummary {
  session_dir: string;       // e.g. "20260513-210410"
  started_at_iso: string;    // "2026-05-13T21:04:10+02:00"
  duration_s: number;        // 5040 (1h 24m)
  event_count: number;       // 38
  bytes_total: number;       // sum of input.wav + voice.wav + events.jsonl + session.json
  crashed: boolean;          // from session.json; row gets a `data-crashed="true"` flag
}

interface RecordingBrowserHandle {
  root: HTMLElement;
  setSessions(sessions: RecordingSummary[]): void;
  setUsage(usage: { sessions: number; bytes_total: number }): void;
}

export function renderRecordingBrowser(opts: {
  initialSessions: RecordingSummary[];
  initialUsage: { sessions: number; bytes_total: number };
  onReplay: (session_dir: string) => void;          // toggle expansion
  onDelete: (session_dir: string, timestamp: string) => void;  // open confirm
}): RecordingBrowserHandle;
```

Internals:
- Calls `registerStyle("vmx-rec-browser", CSS)` once per process.
- Renders the disk usage line + either the list (sessions.length > 0) or the empty state.
- Stores row state internally (which rows are expanded) — no external state singleton needed for this v1.
- Crashed sessions render with `data-crashed="true"` and a small `--led-warn` 5×5 LED dot prefixed to the center cell text — exposes the `crashed: true` field from `session.json` without a separate column.

### `renderRecordingRow(opts)` — new module

```ts
// tauri/ui/src/settings/components/recording-row.ts

interface RecordingRowHandle {
  root: HTMLElement;
  setExpanded(open: boolean): void;
  setAudioSrc(src: string): void;  // sets <audio src> on expand; clears on collapse
}

export function renderRecordingRow(opts: {
  summary: RecordingSummary;
  onToggle: () => void;
  onDelete: () => void;
}): RecordingRowHandle;
```

Internals:
- Lazy-creates the `<audio>` element + transcript overlay only on first expand.
- On collapse, sets `<audio>` `src=""` and `pause()` to free decoder resources.
- Renders the inline expansion via height transition. `prefers-reduced-motion: reduce` disables the transition (display-toggle instead, per `@media` override above).

### Delete confirmation (re-use Phase 12)

The Phase 12 `confirmDialog` modal is re-used VERBATIM. No new dialog component.

Invocation:

```ts
renderConfirmDialog({
  heading: "Delete session 2026-05-13 21:04?",
  body: "This cannot be undone.",
  confirmLabel: "DELETE",
  confirmVariant: "destructive",  // primary CTA renders in --rec
  cancelLabel: "CANCEL",
  onCancel: () => { /* close */ },
  onConfirm: () => { emitIpc("ipc.recordings.delete", { session_dir }); },
});
```

If the existing `confirmDialog` does not yet support a `confirmVariant: "destructive"` flag, Phase 15 adds it as a minimal one-prop extension (primary CTA color flips from `--amber` to `var(--rec)`). This is the only structural change to a Phase 12 component, and it's already implied by Phase 14 §Color (which lists `--rec` as destructive variant on confirm dialogs).

## IPC Contract Surface

Three new families (codegen + drift gate per Phase 11 W0 pattern). NO UI changes needed beyond consuming these:

| Family                      | Direction         | Payload (TS shape)                                                | When fired                                                     |
|-----------------------------|-------------------|-------------------------------------------------------------------|----------------------------------------------------------------|
| `ipc.recordings.list`       | UI → sidecar      | `{}` (no body)                                                    | On drawer open                                                  |
| `ipc.recordings.list_result`| sidecar → UI      | `{ sessions: RecordingSummary[]; bytes_total: number }`           | Reply to `list` request                                         |
| `ipc.recordings.delete`     | UI → sidecar      | `{ session_dir: string }`                                         | Confirm modal `onConfirm`                                       |
| `ipc.recordings.delete_ack` | sidecar → UI      | `{ session_dir: string; ok: boolean; error?: string }`            | Reply to `delete` request                                       |
| `ipc.recordings.usage`      | sidecar → UI push | `{ sessions: number; bytes_total: number }`                       | On any sweep — startup, retention-change, session-close         |

Total IPC family count: 26 (Phase 12 close) → 29 (Phase 15 close). The drift gate (`scripts/check_ipc_schema.py` + `npm run check:ipc`) must remain green.

UI consumption pattern (all via existing `sendIpcRequest` / `subscribeIpc` from Phase 11 Wave 4 `tauri/ui/src/ipc/client.ts`):

- On drawer `openSettings()` call: `sendIpcRequest('recordings.list', {})` with 10s timeout; populate the browser via `handle.setSessions(...)`.
- On delete confirm: `sendIpcRequest('recordings.delete', { session_dir })` with 10s timeout; on `ok: true` ack, optimistically remove the row and refresh disk usage.
- On `recordings.usage` push: call `handle.setUsage(...)` to re-render the disk usage line.

## State Management

New sub-slice on `SessionState`:

```ts
// tauri/ui/src/session/state.ts (extension)

interface RecordingsSlice {
  sessions: RecordingSummary[];           // last response of recordings.list
  usage: { sessions: number; bytes_total: number };
  loading: boolean;                       // true while list request in flight
  error: string | null;                   // last error message, if any
}

// Added to SessionState:
recordings: RecordingsSlice;
```

Lifecycle:
- Default `{ sessions: [], usage: { sessions: 0, bytes_total: 0 }, loading: false, error: null }`.
- On drawer `openSettings()` call: set `loading: true`, dispatch `recordings.list`, on response set sessions + usage + `loading: false`, on timeout/error set `error` + `loading: false`.
- On `ipc.recordings.usage` push (sidecar): update `usage` only — sessions list is NOT refreshed by usage pushes (avoids list-flicker mid-interaction).
- On delete success ack: optimistically remove the deleted session from `sessions` and re-fetch usage via the next sidecar push.

UI re-render: same `subscribeSettingsUI` pattern Phase 12 uses; the drawer's `handle.refresh()` re-builds the body on any state diff.

**No new state singleton.** Reuses the existing `SessionState` store.

## Registry Safety

| Registry          | Blocks Used | Safety Gate                                       |
|-------------------|-------------|---------------------------------------------------|
| shadcn official   | none        | not applicable — project does not use shadcn      |
| third-party       | none        | not applicable                                    |
| inline SVG icons  | hand-authored ▶ play / 🗑 delete / ▾ chevron icons under `tauri/ui/src/settings/icons/` (NEW or re-used from session/icons/) | no third-party dependency, no network fetch |
| Tauri plugins     | re-uses `tauri-plugin-fs` (already enabled) + adds `app.security.assetProtocol` for `recording://` scoped to recordings dir | OS-keyed asset protocol allowlist — capability allowlist updated per Phase 11 W4 convention |

**Network gate:** zero remote fetches at runtime. The `recording://` custom protocol is local-disk-only; the asset protocol allowlist scopes it to the recordings root (`~/Library/Application Support/vibemix/recordings` on macOS, `%APPDATA%\vibemix\recordings` on Windows). No symlink escape — Tauri's asset protocol resolves real paths.

**Privacy gate:** the recording rows expose timestamps + durations + event counts only. No track titles, no MIDI controller names — those would leak set context to anyone with momentary screen access. The expanded panel exposes `events.jsonl` content, which IS the full session timeline (intentional — the user opens it themselves). Tauri's asset protocol scope ensures no remote-page can `fetch('recording://...')`.

## Performance

The recording browser is a content surface, not a perf-critical real-time render. Targets:

- Drawer open → list rendered in < 200ms with ≤ 200 sessions.
- Row expand → audio element mounted + transcript rendered in < 100ms.
- Disk usage update on push → DOM mutation under 5ms.

**Virtualization triggers** at > 50 rows. Below 50 the entire list mounts immediately (per CONTEXT, this matches "virtualized list" intent without over-engineering — lazy mounting kicks in when needed). Above 50, IntersectionObserver chunked rendering.

`prefers-reduced-motion: reduce` and the runtime `data-blur-perf="on"` toggle (Phase 14 Surface 3 §Performance group) both apply to the drawer body; no additional Phase 15 fallback needed.

## Tauri Configuration Delta

`tauri.conf.json` requires one addition — registering the `recording://` asset protocol:

```jsonc
{
  "app": {
    "security": {
      "assetProtocol": {
        "enable": true,
        "scope": [
          "$APPDATA/vibemix/recordings/**",
          "$APPLOCALDATA/vibemix/recordings/**"  // Windows variant
        ]
      }
    }
  }
}
```

Capability allowlist (`tauri/src-tauri/capabilities/default.json`) inherits the asset protocol — no additional command identifier needed since the asset protocol is plugin-less. The Phase 11 W4 capability description should be amended to enumerate this addition for future-reader contract continuity.

**Concrete OS paths** resolved at runtime (Tauri `path` plugin):
- macOS: `~/Library/Application Support/vibemix/recordings/{session_dir}/voice.wav`
- Windows: `%APPDATA%\vibemix\recordings\{session_dir}\voice.wav`

The `recording://` URL pattern is `recording://{session_dir}/voice.wav` — Tauri maps this against the scope above.
