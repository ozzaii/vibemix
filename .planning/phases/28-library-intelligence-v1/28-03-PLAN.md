  <action>
Create `src/vibemix/library/search.py` per RESEARCH Pattern 4.

**Module-level constants:**
- `QUERY_CACHE_TTL = 86400` (24h, per CONTEXT decision).
- `QUERY_CACHE_TABLE = "query_cache"` — DDL: `CREATE TABLE IF NOT EXISTS query_cache (key TEXT PRIMARY KEY, query_text TEXT NOT NULL, result_json TEXT NOT NULL, ts REAL NOT NULL)`. Co-locates in `embeddings.db` (same DB as content-hash cache from Plan 01).

**Dataclass `VibeSearchResult`** (frozen, slots):
- `track_id: str`
- `title: str`
- `artist: str`
- `bpm: float | None`
- `confidence: float` (cosine similarity in [0, 1], rounded to 4 decimals)
- `snippet: str` (short human-readable: `f"{title} — {artist} @ {bpm:.0f} BPM"` truncated to 80 chars; `bpm` may be None → omit segment)

`VibeSearchResult.to_dict()` returns the asdict form for JSON serialization.

**Function `vibe_search(embedder, store, library, query: str, k: int = 10) -> tuple[list[VibeSearchResult], bool]`:**
Signature returns `(matches, cache_hit)` — cache_hit boolean used by Plan 09 IPC payload (LibrarySearchResultPayload.cache_hit). `library` = the RekordboxLibrary for title/artist lookup.

Algorithm:
1. Compute `library_snapshot = store.snapshot_hash()` (Plan 02).
2. Compute `cache_key = sha256(f"{query}|{library_snapshot}".encode()).hexdigest()`.
3. Open `embeddings.db` (the same DB Plan 01 uses; if no connection passed, open fresh). Look up cache_key.
4. **Cache hit AND `(time.time() - row.ts) < QUERY_CACHE_TTL`:** parse `result_json` (list of dicts) → reconstruct VibeSearchResult list → return `(results, cache_hit=True)`. No embed call. No store search.
5. **Cache miss / expired:** `qvec = embedder.embed_query(query)`, `top = store.search(qvec, k=k)`.
6. Resolve `track_id → TrackEntry` via `library.tracks` (build a dict once; per call). Build VibeSearchResult per (track_id, sim) tuple. Tracks not found in library (defensive, shouldn't happen) are skipped with a log line.
7. Serialize results → `INSERT OR REPLACE INTO query_cache`. `commit()`.
8. Return `(results, cache_hit=False)`.

**Privacy / proxy note:** All embedding calls go through `embedder` which is built via `build_proxy_genai_client` upstream — no direct AIza key path.

**Empty library handling:** if `library.tracks == []` → return `([], False)` immediately (don't embed; nothing to match against).

**Update `src/vibemix/library/__init__.py`:** re-export `vibe_search`, `VibeSearchResult`, `QUERY_CACHE_TTL`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI stdin → vibe-search → Gemini Embedding 2 (via Bravoh proxy) | NL query text leaves the user's machine. |
| CLI stdout → user terminal | JSON results visible to whoever runs the CLI. |
| Query cache DB | Stored locally; persists query history. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-03-01 | Information Disclosure | Query text sent to Gemini | accept | Telemetry consent (Phase 34) covers; CLAUDE.md disclosure surface lives in Settings → Privacy (Phase 34). Plan 09 IPC schema adds renderer-side awareness. |
| T-28-03-02 | DoS (cost amplification) | Repeated identical queries | mitigate | 24h cache with library-snapshot-keyed invalidation. Test asserts cache hit. |
| T-28-03-03 | Spoofing | CLI invoked with poisoned `~/.cache/vibemix/library.pkl` | accept | User-owned cache path; trust boundary is the user's own filesystem. |
| T-28-03-04 | Information Disclosure | Query log in `query_cache` table | accept | Local-only; no remote sync. Cache can be cleared via `rm ~/.cache/vibemix/embeddings.db` (document in Phase 34 privacy README). |
| T-28-03-05 | Injection | Query string passed to SQLite | mitigate | Parameterized statements only (`db.execute("... WHERE key = ?", (cache_key,))`). Verified by grep — no string-concatenated SQL. |
| T-28-03-06 | Tampering | CLI argparse — could a crafted query break the parser? | mitigate | argparse handles quoting; query is a single positional `str`. Length cap = 4096 chars at the embedder boundary (Gemini Embedding 2 8192-token context). |
</threat_model>
