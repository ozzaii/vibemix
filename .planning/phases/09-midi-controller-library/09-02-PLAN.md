<objective>
Wave 2 ships the rest of the controller library + the generic fallback + hot-plug:

1. Author 9 controller JSONs (DDJ-400, FLX6, FLX10, 1000, SX3; XDJ-RX3; Numark Party Mix Live; Hercules Inpulse 300, 500) using the schema seeded by Wave 1's `pioneer_ddj_flx4.json`. Every JSON must encode at minimum: per-deck volume + 3-band EQ, crossfader, play/cue/sync per deck, tempo per deck. Filter knob if the controller has one. Hotcue pads where standard.
2. Implement `vibemix.midi.generic` — a synthesized "GENERIC_MIDI" ControllerProfile factory + `find_mapping_or_generic(port_name)` helper. Generic decode emits positional events (`generic_cc_<channel>_<cc>` field names; no semantic deck assignment) with magnitude in [0.0, 1.0]. Coach prompt context (Phase 10's territory) will surface "controller is unmapped — magnitude semantics not available; reactions limited to track audio + screen".
3. Implement `vibemix.midi.watcher.port_watcher_task` — async loop polling `mido.get_input_names()` every 2 seconds; on diff against the previous sweep, invoke a user-supplied `on_change` callback with `('connected', port_name, profile)` or `('disconnected', port_name)` tuples. The watcher does NOT directly restart the listener; it informs callers (production callback lives in `_midi_common.handle_port_change`).
4. Wire the production hot-plug callback in `_midi_common.py`: when a NEW known controller (or generic) appears at a port not currently bound, signal the existing listener thread to stop + spawn a new listener with the new profile, and reassign `controller_state` to a fresh ControllerState built from the new profile (or — simpler implementation — keep the existing ControllerState but swap its `_profile` + rebuild internal lookup tables atomically under the lock; pick the simpler path during execution and document the choice).

Purpose: closes the user-visible MIDI feature surface — every CONTEXT-locked controller works on plug-in, generic controllers degrade gracefully, hot-plug surfaces "controller connected" within 2 seconds of the user plugging the FLX4 mid-set (success criterion #4).

Output: 9 new JSON files; `generic.py` + `watcher.py`; updated `registry.py` (`find_mapping_or_generic`); updated `_midi_common.py` (port-change handler + watcher integration); updated `_midi_macos.py` + `_midi_windows.py` `start_listener_thread` to also spawn the watcher; 4 new test files; updated `tests/test_midi_common.py`.

NOTE on JSON authorship — Kaan only owns the FLX4. The 9 additional JSONs are authored from public-record MIDI maps (manufacturer documentation, mido community contributions, the MixxxDJ controller-mapping repository, DJTechTools forum threads). 09-CONTEXT.md §Deferred Ideas explicitly accepts that live verification of these 9 ships in Phase 16 + 20 + community PRs. Wave 2 ships them as best-effort JSON-validated mappings; the generic fallback ensures unmapped controllers still degrade gracefully.

    # SPDX-License-Identifier: Apache-2.0
    """port_watcher_task — async polling loop for MIDI port hot-plug detection.

    Polls mido.get_input_names() every `poll_seconds` seconds (default 2.0 per
    success criterion #4 in 09-CONTEXT.md). On diff vs the previous sweep,
    invokes a user-supplied async callback with one of:

        ('connected', port_name: str, profile: ControllerProfile)
        ('disconnected', port_name: str)

    The watcher does NOT directly mutate ControllerState or restart the
    listener thread — those are caller concerns (production wiring lives in
    `_midi_common.handle_port_change`). This separation lets unit tests inject
    a synchronous callback without dragging in the threading + mido stack.
    """
    from __future__ import annotations
    import asyncio
    import sys
    from collections.abc import Awaitable, Callable
    from typing import Any
    from vibemix.midi.registry import find_mapping_or_generic

    OnChange = Callable[[tuple[str, ...]], Awaitable[None] | None]

    async def port_watcher_task(
        stop_event: asyncio.Event,
        on_change: OnChange,
        mido_module: Any,
        *,
        poll_seconds: float = 2.0,
    ) -> None:
        last_seen: set[str] = set()
        first_sweep = True
        while not stop_event.is_set():
            try:
                current = set(mido_module.get_input_names())
            except Exception as e:
                print(f"[port watcher err] {e}", file=sys.stderr)
                await _wait(stop_event, poll_seconds)
                continue

            connected = current - last_seen
            disconnected = last_seen - current

            for port in sorted(disconnected):
                await _safe_invoke(on_change, ("disconnected", port))
            for port in sorted(connected):
                profile = find_mapping_or_generic(port)
                await _safe_invoke(on_change, ("connected", port, profile))

            last_seen = current
            first_sweep = False
            await _wait(stop_event, poll_seconds)


    async def _wait(stop_event: asyncio.Event, seconds: float) -> None:
        """Sleep `seconds`, returning early if stop_event is set."""
        try:
            await asyncio.wait_for(stop_event.wait(), timeout=seconds)
        except asyncio.TimeoutError:
            pass


    async def _safe_invoke(callback: OnChange, payload: tuple[str, ...]) -> None:
        try:
            result = callback(payload)
            if asyncio.iscoroutine(result):
                await result
        except Exception as e:
            print(f"[port watcher callback err] {e}", file=sys.stderr)
    ```

    @dataclass
    class ListenerHolder:
        """Mutable holder for the currently-active listener thread + ControllerState.
        The watcher's on_change callback mutates this on hot-plug events."""
        controller_state: "ControllerState"
        listener_thread: threading.Thread | None
        listener_stop: threading.Event | None
        mido_module: Any
        bound_port: str | None = None

    def handle_port_change(holder: ListenerHolder, event: tuple) -> None:
        """Production on_change callback for port_watcher_task.

        On ('connected', port, profile): if the port is already bound, no-op. Otherwise
            stop the existing listener (if any), build a fresh ControllerState from the
            new profile (or rebuild internal lookups in-place under lock — pick during
            execution and document), and spawn a new listener bound to the new port +
            profile.
        On ('disconnected', port): if the disconnected port matches `holder.bound_port`,
            stop the listener + mark controller_state disconnected.
        """
        kind = event[0]
        if kind == "connected":
            _, port, profile = event
            if holder.bound_port == port:
                return  # no-op — already bound to this port
            # Stop existing listener
            if holder.listener_stop is not None:
                holder.listener_stop.set()
            if holder.listener_thread is not None:
                holder.listener_thread.join(timeout=1.0)
            # Rebuild ControllerState (fresh — old state was for a different controller)
            from vibemix.midi.state import ControllerState
            holder.controller_state = ControllerState(profile=profile)
            # Spawn fresh listener
            holder.listener_stop = threading.Event()
            holder.listener_thread = spawn_listener(
                holder.controller_state, holder.listener_stop, profile, holder.mido_module
            )
            holder.bound_port = port
        elif kind == "disconnected":
            _, port = event
            if holder.bound_port == port:
                if holder.listener_stop is not None:
                    holder.listener_stop.set()
                # mark_disconnected method on ControllerState — add it: thin lock-protected
                # `self._connected = False` setter (mirrors mark_connected).
                holder.controller_state.mark_disconnected()
                holder.bound_port = None
    ```
