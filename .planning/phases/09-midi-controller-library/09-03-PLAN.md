    5. **Magnitude semantics** — explain MIDI-14:
       - **Unipolar controls** (volume, EQ low/mid/hi): `magnitude` is the SIGNED delta from the previous CC value, normalized to [-1.0, 1.0] (positive = twisted up, negative = twisted down). Phase 10's prompt rendering treats `abs(magnitude) > 0.7` as "killed" or "max", `< 0.2` as "slight", and the middle as "twist".
       - **Bipolar controls** (crossfader, tempo, filter): `magnitude` is the SIGNED absolute position from center (64), normalized to [-1.0, 1.0] (negative = A-side / cut / bass-cut, positive = B-side / boost / hi-cut).
       - **Buttons** (play, cue, sync, hotcue, jog_touch, loop_in, loop_out): no magnitude — `magnitude` is `None`. Buttons are events not levels.
       - **Generic fallback**: any unmapped CC emits magnitude `value / 127.0` (unipolar default — no axis info available without a profile).

    6. **Generic-MIDI fallback** — paragraph: "Plug in a controller vibemix doesn't recognize and you'll see something like 'Generic MIDI Controller (unmapped) connected on port {name}'. Knob and fader moves are still detected — they just lack semantic context (the Coach knows you twisted *something*, but not whether it was an EQ or a filter). The Coach prompt context flags this and reactions stay grounded — limited to track audio + screen + 'controller is unmapped' notice. To add proper magnitude semantics, contribute a mapping JSON (see Contributing)."

    7. **Hot-plug behavior** — paragraph: "vibemix polls connected MIDI ports every 2 seconds. Plug a controller in mid-session and the status badge flips to '✅ MIDI: <controller> connected' within ~2 seconds (max ~3s end-to-end including listener-thread startup). Unplug it and you'll see the disconnect within the same window. Swapping controllers (unplug FLX4, plug DDJ-1000) restarts the listener with the new mapping automatically — no app restart needed."
