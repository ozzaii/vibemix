<objective>
Port the three asyncio runtime loops from v4 into a new `vibemix.runtime` package: `coach_loop` (the event-driven AI reaction pump), `diag_loop` (terminal meter), and `ws_broadcast` (the mascot WebSocket bus). These are pure-asyncio code with no audio/sounddevice touchpoints — they run alongside `state_refresh_loop` (Phase 3) and the audio I/O streams (Phase 2) inside `__main__.py` (plan 04-04). All three are verbatim ports with one structural change: the `_HAS_WS` feature flag is dropped because `websockets` is now an explicit pyproject dep (Phase 2 already declared it).

This plan runs in parallel with plan 04-02 (DJCoHostAgent + PlaybackQueueAudioOutput) — same wave, zero `files_modified` overlap. It depends on 04-01 only for type-only imports (`DJCoHostAgent` for type hints if needed — but we can use `TYPE_CHECKING` to avoid the runtime dep on 04-02). The coach_loop import surface needs `Event`, `EventDetector`, `MusicState`, `Levels`, `VoiceRecorder` (Phases 2-3, already shipped), plus LiveKit `AgentSession` from `livekit.agents`.

Purpose: lock the three runtime loops as importable, testable async coroutines. Plan 04-04's `__main__.py` spawns each as an `asyncio.create_task`.

Output: a `vibemix.runtime` package with coach.py / diag.py / ws_bus.py and ~25 tests pinning the in-flight guard, mic detection, manual trigger flow, terminal meter format, WS server lifecycle, and inbound trigger action.
</objective>
