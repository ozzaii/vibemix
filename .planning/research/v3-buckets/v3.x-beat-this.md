## Why deferred

Beat This! is a high-accuracy Rust BPM + beat-grid detector. Adding it as a sidecar would supplement the current numpy autocorr approach with a CPU-side DSP path — a non-Gemini grounding axis. The cost is a Rust crate added to the bundle plus a separate phase for install-size budget analysis per memory `project_v2_open_candidates`. v3.1 is hardening, not stack widening.

## Re-eval trigger

- A v3.x phase pinpoints a hallucination class that the current numpy autocorr BPM cannot close.
- Bundle-size budget for Rust sidecars is set in a v3.x install phase.

## Context

The numpy autocorr BPM works for the v3.1 grounding stack but has known accuracy ceilings on tracks with heavy syncopation or off-grid edits. Beat This! claims state-of-the-art accuracy on the MIREX beat-tracking benchmark; the question is whether vibemix's listener path benefits more from BPM accuracy or from other grounding axes the same engineering window could land.
