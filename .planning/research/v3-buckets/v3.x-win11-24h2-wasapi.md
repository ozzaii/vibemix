## Why deferred

Win11 24H2 has not hit stable as of 2026-05. The existing PyAudioWPatch pin handles WASAPI loopback per Phase 7; defer verification to a v3.x phase once 24H2 is broadly deployed. Speculative bumps without a live OS image to test against are wasted cycles.

## Re-eval trigger

- Win11 24H2 hits stable channel.
- A vibemix user reports loopback issues on 24H2 in the GitHub issue tracker.

## Context

WASAPI loopback shapes have historically shifted in Win11 feature updates (22H2 introduced process-loopback APIs; 23H2 tightened permissions). The v3.x phase that lands this surface should run the existing Phase 7 audio smoke harness against a clean 24H2 VM and patch any drift surgically.
