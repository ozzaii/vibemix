## Why deferred

Mixxx upstream PR #14388 (OSC adapter) is still unmerged in the Mixxx mainline as of 2026-05-17 research. Adopting an unmerged upstream surface would either pin vibemix to a fork or carry a maintenance burden that the v3.1 distribution polish window cannot absorb.

## Re-eval trigger

- Mixxx PR #14388 merges to mainline.
- A vibemix v3.x phase explicitly takes on "Mixxx live-set integration" as a goal.
- User demand from the Mixxx community surfaces in the GitHub issue tracker.

## Context

Mixxx is open-source DJ software with a substantial controller-map library (the Mixxx-corpus-transpile path the controller-map-transpiler stub covers). An OSC adapter would let vibemix receive Mixxx state (deck loads, tempo, fader positions) without screen capture or audio analysis — a higher-fidelity grounding signal for the AI co-host. The pairing with `v3.x-controller-map-transpiler.md` is intentional: both surfaces enable Mixxx-native vibemix sessions in tandem.
