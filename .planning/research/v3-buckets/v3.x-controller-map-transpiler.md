## Why deferred

The Mixxx project ships hundreds of community-contributed controller maps. Transpiling them into the vibemix controller library is gated on DEP-OPP-17 (Mixxx OSC) landing first, plus a v3.x phase that scopes which subset of the corpus to import. v3.1 keeps the controller library frozen at the baseline 10 per memory `project_v2_open_candidates`.

## Re-eval trigger

- DEP-OPP-17 (Mixxx OSC) is adopted in a v3.x phase.
- A v3.x phase explicitly takes on "controller library expansion" as a goal.

## Context

A working transpiler converts Mixxx XML controller maps into vibemix JSON profiles, expanding hardware coverage without per-device manual work. The corpus is large enough that a curation pass (popular controllers first, edge-case mappings second) is part of the phase scope.
