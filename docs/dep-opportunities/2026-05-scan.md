| `green-adopt` | Lands in v3.1. Carries an ADR sidecar at `.planning/decisions/DEP-OPP-<N>-<slug>.md`. |

## 4. Per-Candidate Rationale

### 4.1 Green-Adopt

**DEP-OPP-01 — OBS browser-source mascot path** (per OPP-06)

Tauri webview already serves the mascot bus at `ws://127.0.0.1:8765` per v3.0 baseline. OBS Studio's built-in browser-source plugin subscribes to the same surface and captures the rendered Three.js canvas as an overlay — zero new runtime code required, zero new IPC wrappers, zero new dependencies. Adoption is docs-only:

- ADR: `.planning/decisions/DEP-OPP-01-obs-browser-source.md` (decision + rationale + integration plan + rollback path)
- Integration doc: `docs/integrations/obs-browser-source.md` (step-by-step OBS setup + screenshots + troubleshooting)
- README cross-link under the existing Streaming / Integrations section

Per memory `project_mascot_as_vtuber_personality_surface`, this lets streamers route the existing VTuber-style mascot into an OBS scene without vibemix-side work.

### 4.2 Red-Constraint

**DEP-OPP-02 to DEP-OPP-04 — CLAP / MERT / OpenL3.** All three violate the Gemini-only constraint per memory `feedback_no_clap_use_gemini_embedding`. Audio embedding for library vibe search, "what's playing" grounding, track-to-track similarity is served by Gemini Embedding 2 (multimodal, ~180s audio cap) per memory `project_gemini_embedding_2`. The exclusion is absolute regardless of any research recommending these libraries.

**DEP-OPP-05 to DEP-OPP-06 — OpenAI / Anthropic direct API.** Violates `feedback_no_scope_creep_clean_utility` ("OUT: ... multi-provider AI"). Note that `livekit-plugins-openai` already in the lockfile is a TRANSITIVE TTS-proxy fallback dependency, NOT direct OpenAI use; Phase 46 logged it as `cull-blocked` because it cannot be removed without rewiring the TTS chain. The exclusion here covers ADOPTING new direct AI providers — not the existing transitive paper trail.

**DEP-OPP-07 to DEP-OPP-08 — Demucs / Spleeter.** Stem separation is explicitly out per `feedback_no_scope_creep_clean_utility` ("OUT: stem separation"). The vibemix value proposition is the AI co-host reaction to a live set — not source-separation analysis. Mixing-engineer use cases are out of scope.

**DEP-OPP-09 — Ableton Link / DAW APIs.** Same memory: "OUT: ... DAW". vibemix targets bedroom DJs running djay Pro / Mixxx / Rekordbox / Serato, not DAW producers. DAW timing-clock integration is a different product category.

**DEP-OPP-10 — Linux-only audio libs.** Per `/Users/ozai/CLAUDE.md` Platforms line: "macOS + Windows in v1. Linux explicitly excluded." Any candidate that ships Linux-only wheels or requires PipeWire / ALSA-only chains is auto-Red.

### 4.3 Red-Risk

**DEP-OPP-11 to DEP-OPP-12 — ProDJ Link / cdj-link-py.** Wrong market (Pioneer CDJ hardware, not bedroom DJ controllers); Java runtime + LAN config violates one-click install per `project_one_click_install_hard_req`. Already rejected per memory `project_v2_open_candidates`.

**DEP-OPP-13 to DEP-OPP-15 — Dante Via / Loopback Audio / Soundflower.** Commercial alternatives ($60 / $99) or abandonware (Soundflower, 2014 last release, unsigned). BlackHole 2ch covers the macOS virtual-audio use case free + open-source + actively maintained.

**DEP-OPP-16 — Auto-Rig Pro.** Paid ($40) Blender plugin; Mixamo auto-rig (free) is already scaffolded in `scripts/mascot/` (Phase 43-05) per `.planning/research/STACK.md` § Bucket 5. No incremental aesthetic value to justify the cost + license friction.

### 4.4 Yellow-Defer

All eight yellow rows carry forward to `.planning/research/v3-buckets/` via Plan 48-05 stubs. Brief summaries:

- **DEP-OPP-17 Mixxx OSC** — upstream PR 14388 still unmerged in Mixxx mainline. Re-eval when upstream lands.
- **DEP-OPP-18 Controller map transpiler** — Mixxx-corpus-transpile path for library expansion. Gated on DEP-OPP-17 landing + a phase-scoped corpus selection.
- **DEP-OPP-19 pyrekordbox depth** — current pin is shallow XML parsing only (Phase 25 Plan 25-01 `--no-deps` recipe + dormant SQLCipher). Re-eval if v3.x scope adds db6 reads.
- **DEP-OPP-20 Beat This!** — Rust beat-grid verifier sidecar. Non-Gemini path; needs separate phase for install-size budget per memory `project_v2_open_candidates`.
- **DEP-OPP-21 Voicemeeter Banana** — donationware advanced-user wizard tier. Documented as optional path, NOT bundled in installer per OPP scan.
- **DEP-OPP-22 Numark / Hercules controllers** — library expansion via DEP-OPP-18 transpile path; not a new dep, just JSON files.
- **DEP-OPP-23 macOS 26+** — verify when Apple ships the next-major OS post-v3.1.
- **DEP-OPP-24 Win11 24H2 WASAPI** — spot-check after Win11 24H2 hits stable; existing PyAudioWPatch handles WASAPI loopback.
